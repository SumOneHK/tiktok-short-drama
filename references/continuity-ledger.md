## 目标

`分集追踪.md` 不是流水账，而是四本账的合并：

1. 事实账
2. 关系账
3. 悬念债务账
4. 制作资产账

## 一、为什么必须有 `分集追踪.md`

只靠 `故事设定.md` 不够。`故事设定.md` 维护稳定事实，`分集追踪.md` 维护逐集变化。

没有 `分集追踪.md`，最容易出现：

- 上一集埋的钩子被下一集忘掉
- 感情状态乱跳
- 角色明明没见过某物却突然认识
- AI 制作资产前后不一致

## 二、`分集追踪.md` 模板

`分集追踪.md` 的固定模板以 `references/templates/episode-tracker.md` 为唯一权威源，直接套用，不在本文件重复定义字段。

生成时须包含以下核心字段（完整字段见 `references/templates/episode-tracker.md`）：

- 登场角色与标准称谓
- 本集推进、开场钩子承接、本集情绪曲线、本集爽点、本集兑现
- 尾钩、新增悬念、已兑现悬念
- 角色状态变化、关系温度变化
- 新增功能性角色（含是否必要）
- 场景/道具新增或损坏、伤势/造型连续性
- 时间标记、与大纲偏差、下一集必须承接

## 三、悬念债务维护

对每个重要悬念，至少记录：

- 什么时候抛出
- 谁知道
- 观众知道多少
- 角色知道多少
- 什么时候部分兑现
- 什么时候完全兑现
- 当前已挂起集数（每次写新集后自动+1，兑现后归零）
- 是否触发预警（5集/8集/12集档位）

悬念追踪推荐表格格式：

| 悬念 | 抛出集 | 挂起集数 | 部分兑现 | 完全兑现 | 当前状态 |
|------|-------|---------|---------|---------|---------|

`挂起集数` 计算方式：当前最新集 - 抛出集（已兑现则记录兑现时的集数差）

悬念债务预警规则：
- 挂起集数达到 5 集：在下一次质检时必须列为关注项
- 挂起集数达到 8 集且无部分兑现：强制升级为 P1，必须在后续 3 集内部分兑现
- 挂起集数超过 12 集且仍未部分兑现：升级为 P0，禁止继续推进直到处理
- 核心悬念（影响主线的）上限比普通悬念低 2 集

## 四、情绪债务维护

当你让观众承受以下内容时，就产生了情绪债务：

- 长时间羞辱主角
- 长时间误会不解
- 长时间感情压抑
- 长时间低谷或连败

情绪债务必须回收。否则观众会脱离，不会继续等待。

## 五、资产连续性维护

AI 制作尤其要记录：

- 角色造型是否变化
- 衣着是否连续
- 场景是否重复出现
- 道具是否已经损坏、遗失、转手
- 角色伤势是否延续

### 双时间线/闪回连续性账（含闪回、失忆、穿越、平行时空类项目必维护）

含闪回或双时间线的项目，资产连续性必须按"时间线"分轨记录，否则极易在 AI 生成时漂移：

| 时间线 | 角色造型 | 关键道具 | 场景版本 | 伤势/年龄状态 | 备注 |
|--------|---------|---------|---------|--------------|------|
| 现实   |         |         |         |              |      |
| 闪回-1 |         |         |         |              |      |
| 失忆前 |         |         |         |              |      |
| 失忆后 |         |         |         |              |      |

- 每条时间线必须有独立的造型/道具/场景版本
- 跨时间线的同一角色，必须在 `角色外观变化记录`（`故事设定.md` §1.16）登记两套版本
- 分镜稿引用的"时间线归属"字段（见 `references/storyboard-script-templates.md` §统一硬字段）必须能在本表中找到对应轨
- `/总检` 时若发现同一镜头的造型/道具与所属时间线轨道不符 = P0

### 行为微码追踪（Behavioral Microcode）

除了造型/衣着/道具/伤势等"可见状态"，AI 视频制作还需要追踪角色的**亚秒级潜意识动作**——这些动作在剧本中作为角色标签出现，在分镜中作为视觉锚点反复出现，在总检中作为跨集连续性硬依赖。

| 角色 | 微码动作 | 触发条件 | 首次出现集 | 硬锚点集 | 当前状态 |
|------|---------|---------|-----------|---------|---------|
| （示例）Alex | 右手拇指摩擦左手婚戒内侧 | 压力/被揭穿时 | E001 1-3-04 | E008 | 已记录未解释 |
| （示例）Richard | 越生气越慢：摘眼镜 → 五指按桌 → 拍桌一次 | 愤怒递进 | E003 3-1-02 | E015 | 首登场序列完成 |

**维护规则：**

- 每个微码动作在 `故事设定.md` 或 `AI资产圣经.md` 的角色档案中定义
- 首次被镜头记录时，分镜稿的 `连续性锁定` 必须标注"首次被记录"
- 后续出现时标注"第 N 次"并回溯首次出现集
- 微码的含义不在首次出现时解释——只有到达硬锚点集才揭示
- `/分镜质检` 检查：同一角色的微码动作在不同集中的物理描述必须一致（同一只手、同一个方向、同一个触发条件）；不一致 = P0
- `/总检` 时在本表中核对所有微码的首次→硬锚点路径是否完整

## 六、写剧本后的回填规则

每次完成一批分集剧本后：

1. 回填本批次每集的情绪曲线
2. 标记哪些悬念被兑现
3. 更新关系温度
4. 更新角色、场景、道具状态
5. 记录是否新增了功能性角色以及是否必要
6. 记录任何与大纲偏离的实际写法

若某次修订影响稳定事实、命名、关系基线或长期资产状态，必须同步回填 `故事设定.md`，不能只改 tracker。

## 七、总检必查

1. 是否存在挂了很久却无人提及的悬念
2. 是否存在关系突变但 tracker 没有支撑事件
3. 是否存在资产状态穿帮
4. 是否存在情绪债务积压过久
5. 是否存在角色名字、称谓飘移
6. 是否存在功能性角色冗余

## 八、支线追踪（60集以上必维护）

支线不追踪会悄悄消亡，破坏世界可信度。每条支线独立维护一个追踪块。

### 支线追踪表

| 支线名称 | 开始集 | 当前状态 | 最近推进集 | 与主线交织集 | 预计收束集 |
|---------|-------|---------|----------|-----------|---------|
|         |       |         |          |           |         |

状态说明：`推进中` / `暂停` / `已收束` / `预警（超期未推进）`

### 支线健康检查（每次大纲质检时执行）

- 是否有支线已超过预计收束集数但仍未收束（P1）
- 是否有支线最近8集以上没有推进记录（可能已悄悄失去追踪，P1）
- 是否有支线角色开始承担主线戏剧功能（需评估是否合并处理）
- 支线高潮是否与主线高潮撞车（P0）

### 支线回填规则
每写完一批分集剧本后，如支线在该批次有推进，必须：
1. 更新支线追踪表的"当前状态"和"最近推进集"
2. 若支线与主线有交织，在主线对应集的 `分集追踪.md` 中标注"支线交织：[支线名]"
3. 若支线在该批次收束，将状态改为"已收束"并记录收束方式
