# AI 制作规则（统一文档）

本文件统一定义本 skill 的 AI 生产口径：`AI资产圣经.md` 全局资产、artStyle、角色/场景/道具锚点、情绪色彩、导出包、AI 就绪检查。字段清单类内容全部指针化，本文件只保留规则与口径。

> **按阶段分章读取**（权威以 SKILL.md 为准，**禁止 Read 整文件**）：
>
> | 阶段 | 必读段 |
> |------|-------|
> | `/设定` | §二、§三 |
> | `/分集剧本` | §四 |
> | `/分镜脚本` 首次 | §一、§二、§三、§九 |
> | `/分镜脚本` 常规 | §三、§五、§七 |
> | `/总检` | §十、§十一、§十二 |
> | `/导出` | §十 |
>
> 段号定位用 `grep ^##`，不要跨阶段混读。

## 目标

本 skill 的单集目标交付区间是 **60-90 秒竖屏 AI 视频**，但 90 秒不是硬性上限；根据源剧本和分镜实际需要，允许进入 **90-108 秒弹性区间**（+20% 以内），前提是镜头级时长理由充分且三层闭合。AI 视频由模型（可灵 / Sora / Veo / Runway / 海螺 等）按分镜稿直接生成。为此 AI 生产信息必须同时满足：

- 编剧可读
- 制作可执行
- 生成模型可理解

AI 生产信息按"全局层 + 单集增量层"组织，目标不是增加文档，而是**减少重复、降低漂移、稳定复用**。

```
全局稳定资产  →  AI资产圣经.md（一份，跨集继承）
单集叙事事实  →  分集剧本/第NNN集.md 本集调用资产清单（纯叙事级，无 prompt）
单集镜头 prompt →  分镜脚本/第NNN集.md（首次出现 AI 画面/动作提示、连续性锁定、负面约束）
```

> **强约束**：分集剧本不承载任何镜头级 prompt 和风格关键词；所有 AI 画面提示、AI 动作提示、机位/运镜、连续性锁定词、负面约束、时长闭合一律由 `/分镜脚本` 阶段首次生成。详见 `references/episode-script-templates.md` 与 `references/storyboard-conversion.md`。

## 一、全局资产圣经（`AI资产圣经.md`）

### 1. 什么时候创建

**唯一首次创建点**：首次进入 `/分镜脚本 {起止集}`。

不允许的场景（全部硬禁止）：

- `/开始`、`/立项`、`/设定`、`/结构`、`/分集大纲` 任何阶段**禁止**提前创建
- `/分集剧本 {起止集}` **禁止**创建或"降级创建最小版本"
- 任何过程质检命令**禁止**创建

创建前提：

- 项目已完成 `故事设定.md`（artStyle、全局视觉基调字段已在 `故事设定.md` 预埋）
- 已进入 `/分镜脚本 {起止集}` 且 `AI资产圣经.md` 缺失

**口径对齐**：以 `SKILL.md` 的"按阶段延迟创建规则"表为唯一权威。如本文件与 SKILL.md 有任何冲突，以 SKILL.md 为准。

### 2. 放什么

只放"跨集稳定、可继承、值得复用"的内容：角色基础资产、角色外观状态、场景基础资产、道具基础资产、全局生成规则（artStyle / 色彩 / 光线 / 镜头语言 / 负面约束 / 风格关键词模板 / 目标模型偏好）。

角色资产必须采用**单角色实体 + 多外观状态**结构：

- 一个叙事角色在 `AI资产圣经.md` 中只能有一个 `### 角色：{角色名}` 资产卡
- 角色卡内先定义 `Identity Anchor`（全剧不可变的脸、身形、固定识别点、禁止变化）
- 现在线、过去线、失忆前、复仇后、长期伤势、长期换装等差异，必须写入同一角色卡下的 `Visual States / Life Stages`，用 `S01/S02/...` 标识
- 禁止把同一角色的不同时间线拆成多个角色卡，禁止为每条时间线生成互不继承的完整角色设定
- 只有 `S01` 可以作为初始身份基线；`S02+` 必须显式写明“从阶段继承”和“必须保持不变”，用于制作阶段以图生图/参考图继承身份

> **字段清单以 `references/templates/ai-asset-bible.md` 为唯一权威源**（`全局生成规则`、`角色资产卡`、`场景资产卡`、`道具资产卡` 四块）。本文件不重复列举字段。

### 3. 什么时候必须回写全局资产圣经

满足以下任一条就必须回写：

- 变化会持续影响后续 2 集以上
- 会改变角色/场景/道具的识别方式
- 会影响后续 prompt 的稳定继承
- 会让未来单集继续重复同样说明

典型必须回写：角色长期换造型、年代跳跃导致年龄状态变化、固定持续伤势/疤痕、固定场景被毁或封锁、关键道具损坏/转手/污染进入新常态、全局画风/色彩/镜头语言改口径。

典型不必回写：某集一次性哭红眼、某场临时站位、某一镜头慢推或甩镜、一次性字幕强化。

角色回写时只允许新增或更新同一角色卡下的 `Sxx` 外观状态；除非剧情确实新增了另一个人，不得新增同名/近似名角色卡。一次性哭痕、湿发、血迹、灰尘、临时外套、临时配饰不建 `Sxx`，只写入分镜稿的连续性锁定或单镜增量。

## 二、制作锚点的最低字段

核心角色 / 核心场景 / 关键道具每张资产卡的必填字段，以 `references/templates/ai-asset-bible.md` 的 `角色资产卡` / `场景资产卡` / `道具资产卡` 块为准，本文件不重复列举。

任一资产卡缺失必填字段，`/分镜质检` 判定为全局资产未达最低就绪标准。

角色资产卡的最低可制作标准额外要求：

1. `characterId` 唯一且稳定
2. `Identity Anchor` 必须覆盖面部几何、身形比例、不可变特征、禁止变化
3. 至少存在 `S01` 外观状态
4. 每个 `Sxx` 必须有时间线归属、有效范围、触发原因、继承来源、必须保持不变、允许变化、服装/发型/妆造、基础 AI 画面提示
5. 同一角色多状态之间必须能看出继承关系；若 `S02+` 像另一个人，视为资产契约失败

## 三、artStyle 与统一交付管线

本 skill 的单集目标交付区间是 60-90 秒竖屏 AI 视频，并允许有叙事理由的 90-108 秒弹性区间。剧本和分镜模板**不硬编码任何美术风格**；所有艺术风格都在这里统一定义为 `artStyle`，由 AI 视频模型在生成时按 `artStyle` 关键词渲染。

### 1. artStyle 取值

artStyle 取值是开放列表，常见参考如下（任选其一，也可补充其他风格）：

- **写实系**：写实电影感 / 都市冷调 / 港风复古 / 日系清冷 / 欧美大片 / 胶片颗粒 / 半写实 CG
- **东方系**：国潮水墨 / 仙侠工笔 / 古风工笔
- **科幻系**：赛博朋克 / 蒸汽朋克 / 未来科幻
- **3D 系**：3D 卡通 / 皮克斯 3D / 半写实 3D
- **插画系**：厚涂插画 / 油画 / 水彩 / 韩漫风插画
- **特殊**：像素艺术 / 拼贴

### 2. `AI资产圣经.md` 全局生成规则必填字段

字段清单以 `references/templates/ai-asset-bible.md` 的 `全局生成规则` 块为准。本章只给出两个示例帮助理解填写口径：

- `风格关键词模板` 示例：`cinematic realism, soft natural light, shallow depth of field, vertical 9:16, urban cool grading`（可直接拼进镜头 `AI画面提示`）
- `色彩基调 + 光线语言` 示例：`高对比冷灰 + 关键场金爆` / `自然硬光 + 侧逆光`

### 3. artStyle 与 AI 模型选择的关系

不同 AI 视频模型对不同 artStyle 的支持力度不同。`目标 AI 视频模型偏好` 必须写明本剧的主力模型和备用模型：

- 写实电影感 / 港风复古 / 都市冷调 → 视频模型成熟度高，主流可用
- 国潮水墨 / 厚涂插画 / 3D 卡通 → 部分视频模型支持较好，需要在风格关键词模板里强化
- 赛博朋克 / 像素艺术 等强风格 → 需在分镜阶段对关键镜头单独验证

`/分镜质检` 会检查 artStyle 与目标模型的匹配度，必要时建议切换 artStyle 或模型。

### 4. artStyle 在分镜稿中的注入方式

`AI画面提示` 字段必须按以下结构组织：

```
{主体描述} + {动作瞬间} + {场景/光线} + {景别构图} + {artStyle 关键词模板}
```

其中 `artStyle 关键词模板` 来自 `AI资产圣经.md` 的全局生成规则，全集统一注入，**不允许逐镜手写风格关键词**。

### 5. 切换 artStyle 的影响

切换 artStyle 时只需更新 `AI资产圣经.md` 中的：

1. `artStyle` 字段
2. 风格关键词模板
3. 目标 AI 视频模型偏好
4. 通用负面约束

分集剧本和分镜脚本文件本身**不需要任何字段层修改**，因为字段结构与风格无关。

## 四、分集剧本阶段如何使用（叙事事实级）

进入 `/分集剧本` 时：

1. 先读 `故事设定.md`（artStyle、角色、场景、关系等全部取自这里）
2. 若 `AI资产圣经.md` 已存在（说明某批次已经跑过 `/分镜脚本`），读取当前集相关资产卡作为叙事事实参考
3. 若 `AI资产圣经.md` 不存在（首轮剧本生产的正常状态），**不创建、不降级**，直接以 `故事设定.md` 作为叙事事实唯一来源继续写
4. 按 `references/episode-script-templates.md` 写单集正文（纯叙事中间稿）
5. 在 **本集调用资产清单** 中只登记：
   - 调用的全局角色（名称、基础身份）
   - 调用的全局场景（名称、基础地点）
   - 调用的全局道具（名称、基础功能）
   - 本集新增/变化的叙事事实（伤势、换装、道具损坏、场景状态变化等）
6. 若本集引入会持续影响后续 2 集以上的长期变化，**先写入 `分集追踪.md` 的资产账**，等到 `/分镜脚本` 阶段再按 `references/document-sync.md` 的规则回写 `AI资产圣经.md`

**分集剧本阶段绝对不允许**：
- 创建或修改 `AI资产圣经.md`
- 出现镜头号、时长求和、`AI画面提示`、`AI动作提示`、机位/角度/运镜、景别、连续性锁定词、负面约束、风格关键词片段

这些一律由 `/分镜脚本` 阶段首次生成。

## 五、分镜脚本阶段如何使用（镜头 prompt 级）

进入 `/分镜脚本` 时：

1. 先读 `AI资产圣经.md`
2. 把全局角色/场景/道具资产继承到镜头层；涉及角色时必须先按镜头的 `时间线归属`、集数、剧情状态匹配该角色卡下唯一一个 `Sxx` 外观状态
3. 按 `references/storyboard-script-templates.md` 生成镜头稿
4. 每个关键镜头的字段清单以 `references/storyboard-script-templates.md` 为准（镜头号/时长、景别/机位/运镜、AI画面提示、AI动作提示、连续性锁定、负面约束等）。本文件只强调：`AI画面提示` 必须按 `§三 §4 artStyle 注入结构` 组织，**不允许**逐镜手写风格关键词；角色描述必须引用 `角色名 + Sxx + 本镜增量`，不得在镜头里重新生成另一套完整长相。
5. 本集附录只补：
   - 本集新增锁定词
   - 本集状态变化
   - 本集增量负面约束
   - 高风险镜头
6. 若拆镜中发现全局资产不够稳定，先补全 `AI资产圣经.md`，再继续拆镜；如果缺的是角色长期状态，补同一角色卡的 `Sxx`，不要新增同名角色卡

镜头 `AI画面提示` 的角色部分推荐格式：

```text
角色：{角色名}（{characterId}，{Sxx 状态语义}）
身份继承：继承 Identity Anchor 的脸型、眼型、鼻型、唇形、身形比例和固定识别点
本状态外观：{Sxx 的发型/服装/妆造/伤势}
本镜增量：{一次性哭痕/湿发/血迹/站位等；没有则写 无}
```

禁止格式：

```text
过去线林晚：18岁少女，圆脸大眼，甜美校服...
现在线林晚：28岁医生，冷脸单眼皮...
```

这种写法等价于让 AI 重抽两个人，必须改为同一 `characterId` 下的 `S01/S02` 状态继承。

完整的镜头拆解流程、时长守恒规则和镜头就绪要求，见 `references/storyboard-conversion.md`。

### 序列化视觉系统（Serialized Visual System）

许多短剧项目需要一个**跨集反复出现的签名视觉元素**，同时承载叙事推进、品牌识别和结构节奏三重功能。常见模式：

- 法律/复仇剧：编号证据字卡（EXHIBIT 1→2→3...）
- 甜宠/倒追剧：倒计时字卡（Day 30→Day 29...）
- 悬疑/解谜剧：线索板/红线交叉图
- 家庭/逆袭剧：对比分屏（过去 vs 现在）

**若项目需要序列化视觉系统，必须在 `AI资产圣经.md` 中建卡，最低字段：**

| 字段 | 说明 |
|------|------|
| 系统名称 | 如 "EXHIBIT 编号字卡" |
| 字体/字号/颜色 | 如 "血红 Serif · 大号" |
| 安全区位置 | 竖屏中的固定位置（如 "中上部 30%-50% 居中"） |
| 出现动画 | 淡入速度 / 持续时间 / 退出方式（如 "0.3s 锐利淡入 · 持续 0.5-1s · 硬切出"） |
| 叙事功能 | 该系统除了信息标注外还承担什么（如 "时间线跳转信号 + 尾钩锚点"） |
| 编号规则 | 序列递增方向 / 箭头方向含义 |
| 禁止事项 | 如 "不允许在字卡存在时同时出现底部字幕" |

**在分镜稿中的使用规范：**

- 每次字卡出现都作为独立镜头（不与对白镜头合并）
- `字卡/字幕` 字段写明完整字卡内容和安全区位置
- `AI画面提示` 中复制字卡的完整视觉规范
- `连续性锁定` 中标注与前几集同系统字卡的体系一致性
- 尾钩镜头使用字卡时，字卡的编号方向（→/←）可暗示下一集的推进方向

## 六、对白与配音要求

为了适配 AI 配音和 lip-sync：

1. 单句尽量短
2. 避免复杂从句
3. 避免一口气塞入解释信息
4. 情绪词直接
5. 重要停顿在说明中标出

### 倍速观看适配（TikTok 特有要求）

TikTok 大量用户以 1.5x-2x 倍速观看，对白和视觉设计需同时适配：

1. **关键信息视觉化**：核心剧情转折不能只靠台词传递，必须有对应的视觉动作（表情特写、道具出现、字卡强化）
2. **禁止仅靠语气传情**：倍速下声音变调，"轻声说/哽咽说"等语气在倍速下效果消失——必须用视觉动作配合
3. **字卡作为信息保险**：每集最关键的 1-2 个信息点，在分镜阶段标注"建议字卡强化"，确保倍速用户也能接收
4. **OS/VO 谨慎使用**：倍速下 OS/VO 的语速感知会失真，重要内心独白应缩短并配合表情特写

## 七、情绪色彩规范

AI 视频制作中，色彩是情绪的第一语言。具体色彩口径由 `AI资产圣经.md` 全局生成规则的 `artStyle`、`色彩基调` 和 `光线语言` 字段统一定义；单集镜头稿附录只写增量。

### 情绪-色彩对照表

| 情绪类型 | 色彩基调 | 光线特征 | 适用场景 |
|---------|---------|---------|---------|
| 对峙/冲突 | 高对比冷色调（蓝灰/黑白） | 硬光、侧逆光 | 权力博弈、争吵、揭露 |
| 心动/暧昧 | 暖色低饱和（蜜桃/米白） | 柔光、漫反射 | 近距离接触、眼神交汇 |
| 恐惧/悬疑 | 去饱和冷色（蓝绿/灰） | 暗调、局部阴影 | 跟踪、秘密发现、威胁 |
| 爽点/反杀 | 高饱和暖色爆发（金/红） | 强光从下打、高对比 | 身份曝光、公开反杀、大获全胜 |
| 低谷/绝望 | 去饱和灰调 | 顶光下压、无辅光 | 失败、被抛弃、最低点 |
| 治愈/和解 | 暖白色调 | 柔和均匀光 | 关系修复、支撑时刻 |
| 回忆/闪回 | 泛黄/去饱和 | 曝光过度或欠曝 | 闪回场景 |

### 全剧色彩基调

在 `/设定` 阶段完成时，`故事设定.md` 的"项目概览"应补充：

- 全剧主色调关键词（如"冷灰+金爆"）
- 情绪转折时的色彩切换规则

### 分镜稿中的色彩标注

在 `分镜脚本/*.md` 的关键镜头 `AI画面提示` 或镜头附注中，对关键情绪场景标注色彩：

```text
- 分镜提示：特写，女主眼眶泛红，【色彩：去饱和冷调，侧逆光】
- 分镜提示：全幅爽点分镜，男主站定，光从下打，【色彩：金色暴击光】
```

### 多时间线色彩互斥协议（含闪回/穿越/双时间线项目必执行）

当项目包含 ≥2 条时间线时，仅靠"情绪-色彩对照表"不够——必须建立**时间线级色彩互斥协议**，确保 AI 视频模型在任何一帧都不会混淆时间线归属。

**协议必须在 `AI资产圣经.md` 的全局生成规则中定义，最低字段：**

| 字段 | 说明 | 示例 |
|------|------|------|
| 时间线名称 | 对应 `分集追踪.md §五` 的时间线轨道 | 现在线 / 过去线（闪回-1） |
| 专属色盘 | 该时间线的主色调 + 辅色 | 冷钢蓝 + 炭灰 / 暖金蜜糖 + 酒红 + 珠光 |
| 专属光线 | 该时间线的光线语言 | 冷白 LED 顶光 / 过饱和水晶吊灯暖光 |
| 角色状态映射 | 每个出场角色在本时间线使用哪个 `Sxx` | 林晚=S01 / 林晚=S03-过去线 |
| 互斥禁令 | 该时间线**绝对禁止**出现的对方元素 | 禁止暖金色 / 禁止冷钢蓝 LED |
| 跨线过渡机制 | 从本时间线切到另一条线时必须使用什么 | 血红字卡 + 色温瞬变闷响 / 硬切无过渡 |

**在分镜稿中的执行规范：**

- 每个场次的 `时间线归属` 字段决定该场次使用哪套色盘，不允许混用
- 每个场次的 `时间线归属` 同时决定角色的 `Sxx` 状态映射；同一角色跨时间线仍然继承同一个 `Identity Anchor`
- 每次时间线跳转的前一镜和后一镜必须各有独立镜头，且至少一个镜头的 `画面内容` 中描述过渡机制（字卡/色温瞬变/硬切）
- `AI画面提示` 中的色彩关键词必须与所属时间线的专属色盘一致——`/分镜质检` 检查到冷蓝出现在暖金时间线 = P0
- 全集共用负面约束中必须写明双向互斥（如 `no warm lighting in present-line scenes, no cold LED in flashback-line scenes`）
- 过渡镜头使用 `dissolve`（渐隐渐出）还是 `hard cut`（硬切）必须在全局生成规则中统一，不允许逐集自行决定

**不适用条件：** 纯单时间线项目无需执行本协议。

## 八、字幕、字卡、BGM、SFX

这些不是后期随便补的，关键落点必须在分镜阶段锁定：

- 哪句对白需要屏幕字卡强化
- 哪个爽点需要冲击音效
- 哪个钩子需要硬切 BGM
- 哪段情感戏需要留呼吸空间

分集剧本阶段只允许以**叙事事实**形式出现字卡/SFX 意图（例如"结尾需要一次信息强化"），**不允许**直接写字卡文案的屏幕坐标、安全区尺寸或具体音效 ID，这些一律推迟到分镜阶段。

## 九、TikTok 竖屏安全区规范

竖屏安全区（UI 覆盖区域、字幕落位、构图安全检查）以 `references/tiktok-platform-constraints.md` §一·附 为唯一权威源，本文件不重复。

## 十、导出包最少包含什么

`导出/` 目录默认至少包含三个文件：

1. `导出/{剧名}-策划包.md`
   - 立项与市场判断
   - 世界与角色设定
   - 全局结构与情绪/卡点设计
   - 分集大纲
   - 连续性账本
   - 质检结论
   - 合规结论
2. `导出/{剧名}-分集剧本合集.md`
   - 通过脚本合并全部 `分集剧本/第NNN集.md`
   - 保留海外模式下的英文对白、旁白、`OS`、`VO`
3. `导出/{剧名}-分镜脚本合集.md`
   - 通过脚本合并全部 `分镜脚本/第NNN集.md`
   - 保留镜头级 `AI画面提示`、`AI动作提示`、连续性锁定与负面约束

默认"制作包导出"要求第 2、3 项都覆盖全剧集数；若缺少分镜脚本，只能定义为"编剧包"，不能定义为完整制作包。

若用户明确要求"单文件全量包"，再额外生成第四个整合文件；默认导出定义不把完整分集正文并入策划包。

## 十一、AI 就绪检查

在 `/总检` 阶段至少确认：

1. 角色外观锚点是否稳定
2. 核心场景是否足够复用
3. 道具是否可追踪
4. 分集剧本是否以纯叙事中间稿形态完整（无镜头级越界字段）
5. 对白是否适合配音与口型
6. 若存在 `分镜脚本/`，关键镜头是否具备可直接投喂模型的 prompt 完整度
7. `AI资产圣经.md` 的全局生成规则（artStyle、色彩基调、风格关键词模板、目标模型）是否已锁定且与已生成的镜头稿一致

## 十二、质检联动

### `/剧本质检`（叙事事实级）

- `AI资产圣经.md` 是否已建立并覆盖本批次涉及的核心资产
- 本集调用资产清单是否只写叙事事实级登记（无 prompt 增量）
- 角色/场景/道具命名是否与 `故事设定.md` 一致
- **反向检查**：本批次剧本是否误写了镜头级字段；若有，一律移除

### `/分镜质检`（镜头 prompt 级）

- 镜头层是否正确继承全局角色/场景/道具资产
- 单集增量是否写清状态变化与特例
- artStyle 关键词是否按统一结构注入，未逐镜手写
- 字幕/字卡是否落在竖屏安全区内
- 是否因缺少全局资产而导致角色/场景漂移风险

## 最小执行标准

若要保持 skill 简洁，最低要求是：

- 有 1 份 `AI资产圣经.md`（全局层）
- `分集剧本/*.md` 的本集调用资产清单只写叙事事实级登记
- `分镜脚本/*.md` 镜头层继承全局资产，只写本集增量 prompt
- `/剧本质检` 和 `/分镜质检` 明确检查资产继承与增量边界

做到这四点，重复和漂移就会明显下降。
