# 藏线与发现感设计

## 目标

“藏”不是少写信息，也不是让角色当谜语人，而是把真相拆成观众能看见、能误读、能回看确认的证据链。观众当下必须看懂当前危机，后续揭露时又能意识到：原来线索早就摆在画面里。

## 一、藏而不糊的边界

不能藏：
- 主角现在想要什么
- 当前危机是什么
- 本集发生了什么状态变化
- 观众为什么要继续看下一集

可以藏：
- 事件真正原因
- 角色真实动机
- 道具真正用途
- 世界规则的底层代价
- 反派真正害怕的东西
- 某句台词、某个动作、某个画面的第二层含义

判定原则：
- 观众当下看不懂“发生了什么” = 糊，P0/P1
- 观众当下看懂危机，但误读了原因 = 藏，成立
- 揭露时没有早期可回看的证据 = 硬编，P1

## 二、藏点设计表

结构阶段必须为核心藏线建表。每条藏线至少填写：

| 字段 | 填写要求 |
|------|---------|
| 藏线ID | 固定 ID，如 `R-01`，后续大纲/剧本引用 |
| 藏的是什么 | 真相本体，不写给观众看的假解释 |
| 首次可见异常 | 观众第一次“看见它”的动作、道具、画面或反应 |
| 当时误读 | 观众/角色当下最可能以为它是什么 |
| 小回报 | 5-8 集内给一次部分解释、功能展示或反向验证 |
| 反向理解点 | 观众开始意识到旧解释不够的节点 |
| 终局兑现 | 真相完整揭开的位置和方式 |
| 回看奖励 | 兑现后，观众回看哪一个早期画面会觉得“早就给过” |

## 三、分层使用

### `/设定`

`故事设定.md` 的 `秘密与信息差` 不能只写“秘密/真相/兑现区间”。含多重秘密、身份反转、调查、悬疑、精品漫剧发现感的项目，要补：
- 首次可见异常
- 当时误读
- 部分兑现
- 终局兑现

### `/结构`

`故事结构.md` 必须建立“藏线与发现感总表”。核心藏线不宜太多：
- 20 集以下：1-3 条
- 20-50 集：3-6 条
- 50 集以上：5-8 条

结构阶段必须安排“显点/藏点/误读/部分兑现/反向理解/终局兑现”，不要把所有真相一次性写成说明段。

### `/分集大纲`

每集需要区分：
- 本集显性推进：观众必须当下看懂的事件
- 本集暗藏信息：观众可以暂时误读的证据
- 本集发现证据：道具、动作、画面、表情、措辞、系统字卡等
- 本集误读方向：观众或角色当下会怎么错解
- 本集兑现责任：它对应哪条藏线 ID，最迟何时给小回报

### `/分集剧本`

藏线优先通过画面、动作、道具、反应来写，不优先用解释台词。

推荐写法：
- 让道具短暂异常，但角色先按普通故障处理
- 让角色听见某个名字后微停顿，但不解释
- 让系统字卡闪过半秒旧编码
- 让反派避开某个看似普通的问题
- 让一张照片、签字页、衣物、旧伤在后文改变意义

禁止写法：
- 角色直接说“这个东西以后很重要”
- 反派突然独白解释自己的计划
- 揭露前完全没有画面证据
- 为了藏而让本集没有明确状态变化

## 四、兑现节奏

藏线不是无限拖延。推荐节奏：
- 首次异常后 1-3 集内必须再次出现或产生后果
- 普通藏线 5-8 集内给小回报
- 核心藏线 8-12 集内必须部分兑现或反向理解
- 超过 12 集无任何回报，默认 P1；若是主线核心悬念，升 P0

## 五、质检口径

P0：
- 为了藏而让主角目标、当前危机或本集状态变化不清楚
- 核心主线真相在揭露前没有任何可回看的证据
- 关键集/付费卡点靠“突然解释”硬揭露，前文无支撑

P1：
- 藏线只有文件名、编号、代号，没有具体兴趣点
- 藏线超过 8-12 集无小回报或反向理解
- 揭露主要靠长篇解释台词，而不是动作、证据、反应和可见后果
- 同类藏法连续重复，例如每条秘密都靠“突然文件曝光”

P2：
- 藏点存在，但回看奖励不够清楚
- 误读方向单一，观众很早就能猜完
- 发现证据过于隐蔽，分镜阶段需用构图/字卡微强化

## 六、示例

错误：
- 第25集突然说“母亲其实是地基资产”，前24集没有任何异常、编码、旧权限或他人反应。

正确：
- 第1集工具手环在下沉通知出现时闪旧红光。
- 第2集电梯识别时短暂闪过 `M. VALE ACCESS`。
- 第3集 Rising Slot 绑定 `FOUNDATION ASSET F-019`。
- 第8集反派说 “some foundations must remain invisible”。
- 第17集男主发现 `F-019` 不是死亡编号，而是活体维护编号。
- 第25集揭示母亲进入地基层。
- 第45集工具手环真正用途兑现。
