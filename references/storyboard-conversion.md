## 目标

`/分镜脚本` 阶段的目标不是把 `分集剧本` 换个排版，而是把"纯叙事中间稿"转成"可执行制作稿"。

**上下游边界（强约束）**：

- `分集剧本/*.md` 是**纯叙事中间稿**，不含任何镜头级字段（无镜头号、无时长求和、无 `AI画面提示`、无 `AI动作提示`、无机位/运镜/景别）
- `分镜脚本/*.md` 是**唯一的镜头级权威源**——镜头号、镜头级时长、场次/全集时长闭合、AI 画面/动作提示、机位/运镜、连续性锁定、负面约束全部在本阶段首次出现
- 若在 `分集剧本/*.md` 中发现分镜级字段，属于上游越界，先按 `references/episode-script-templates.md` 的反向检查清除，再进入 `/分镜脚本`

这里的"可执行"至少同时满足：

- 导演/分镜师能直接理解镜头意图
- AI 视频模型能直接拿镜头级提示生成画面
- 剪辑、配音、字幕、特效能拿到明确落点

默认标准：**每个关键镜头都要达到可单独投喂模型的完整度**。

## 一、何时启动

推荐时机：

1. 对应批次 `分集剧本/*.md` 已完成
2. 对应批次已通过 `/剧本质检 {起止集}`
3. 关键卡点集、揭露集、关系翻页集已通过 `/衔接质检 {起止集}`

建议流程：

- 小批次生产：`/分集剧本 001-005` -> `/剧本质检 001-005` -> `/分镜脚本 001-005` -> `/分镜质检 001-005`
- 全剧完成后：至少再做一次面向导出的 `/分镜脚本 {全剧范围}` 收口检查

## 二、转换前必须读取什么

转换 `第{NNN}集` 前至少读取：

1. `故事设定.md`
2. `AI资产圣经.md`（若缺失，先基于 `故事设定.md` 创建最小版本）
3. `分集大纲.md` 中上一集、本集、下一集块
4. `分集追踪.md` 中最近 3-5 集
5. `分集剧本/第{NNN}集.md`
6. 若 `分镜脚本/` 已存在，再额外读取最近 1-2 集已完成的 `分镜脚本/第{NNN}集.md`
7. `references/storyboard-script-templates.md`
8. `references/ai-production.md`

读取后先整理一份“本集分镜事实底稿”，至少包含：

- 本集必须成立的剧情事实
- 本集承接上一集尾钩的落点
- 本集需要埋下或兑现的悬念/道具/伤势/关系变化
- 本集最重要的 1-2 个情绪高点
- 本集尾钩为下一集留下的动作或信息切口

若这些事实没有先锁清，分镜很容易从“根据剧本拍”滑到“为了镜头表好看而改剧本”。

以下情况额外回看“首次埋设集”：

- 关键道具回收
- 重大身份揭露
- 旧伤势或旧造型延续
- 旧秘密兑现

若已存在历史分镜稿，还要额外确认以下信息不要漂移：

- 角色锁定词
- 常用场景锁定词
- 全局道具锁定词
- 常用负面约束
- 同一角色的运镜习惯
- 同类高潮镜头的放大方式

## 三、分镜脚本的交付标准

专业分镜脚本至少要把以下层级拆开：

1. **场次层**：本场承担什么叙事和情绪功能
2. **节拍层**：本场内部的动作节点、信息节点、情绪翻点
3. **镜头层**：每个镜头怎么拍、给模型什么提示、音视频怎么落

每个镜头都必须至少有：

- 镜头号
- 来源场次
- 预计时长
- 景别
- 机位/角度
- 运镜
- 画面主体
- 主动作
- 情绪目标
- 台词/OS/VO/音效
- 字卡/字幕
- AI 画面提示
- AI 动作提示
- 连续性锁定
- 负面约束

缺任一项，都不算可直接投喂模型的完成稿。

## 四、标准转换流程

### 1. 先锁场次目标，不先写 prompt

每场先回答：

1. 这场要推进什么
2. 这场要改变什么
3. 这场最值钱的镜头是什么
4. 这场尾部要怎样把观众推入下一镜或下一场
5. 这场有哪些不能改动的事实锚点
6. 这场的时长为什么值得是这个数字

若这四个问题答不清，不要开始拆镜。

这里的“事实锚点”通常包括：

- 人物此刻知道/不知道什么
- 人物站位、伤势、服装、道具状态
- 动作是否已经发生、正在发生、还是即将发生
- 这一场是否必须接住上一集尾钩
- 这一场是否承担本集中段升压、高潮放大或尾钩切断

### 2. 把场次拆成节拍

每场先拆成 2-5 个节拍，再从节拍拆成镜头。

常见节拍类型：

- 钩子落地
- 信息揭露
- 权力变化
- 情绪压迫
- 爽点兑现
- 尾钩硬切

原则：

- 一个节拍只负责一个主变化
- 一个镜头尽量只承载一个主动作
- 信息变化和情绪变化最好不要都塞在同一长镜里
- 节拍时长必须来自动作事实和情绪需要，不能只为把场次数字凑平

### 3. 先定镜头功能，再定镜头语言

每个镜头先写清“它为什么存在”，再补镜头语言。

镜头常见功能：

- 建立空间
- 放大表情
- 交代动作
- 强化信息
- 留白停顿
- 爽点暴击
- 尾钩切断

### 4. 最后补 AI 生成字段

prompt 字段不是文学描述，要服务生成。

推荐顺序：

1. 主体是谁
2. 当前动作是什么
3. 情绪状态是什么
4. 场景和光线是什么
5. 镜头距离和角度是什么
6. 是否有运镜
7. 有什么绝对不能变

### 5. 保持跨批次分镜语言稳定

新批次分镜稿默认继承最近 1-2 集已通过质检的分镜语言。

重点保持稳定：

- 主角/反派的锁定词写法
- 常驻场景的光线和构图描述
- 字卡安全区口径
- 常用负面约束
- 尾钩镜头的切断方式

若必须改变镜头语言，必须在本集 `AI 视频生成附录` 中写明“变化原因”。

## 五、镜头拆解规则

### 时长建议

- 60-90 秒 AI 视频：单集建议 10-20 个镜头
- 节奏偏快的爽点/反杀集：可上调至 18-24 个镜头
- 节奏偏慢的情感高潮集：可下调至 8-14 个镜头
- 单镜头超过 6 秒时，必须确认它承担的是停顿放大、高潮定格或连续动作，不是节奏拖慢
- artStyle 差异只影响 `AI画面提示` 关键词注入，不改变单集镜头数量基准

### 时长守恒规则（权威定义）

> **本节为时长闭合的唯一权威源**。`/分镜脚本`、`/分镜质检`、`/总检`、`/导出` 引用时长容差规则一律回到本节，不得在其他文件硬编码不一致的数字。

分镜时长不能靠感觉填，必须按"集 -> 场 -> 镜头"逐层守恒：

1. 先确定本集目标时长，再把时长预算分配到每个场次。
2. **镜头 → 场次容差（权威值）**：每场先写 `本场预计时长`，再拆镜；场内所有镜头/分镜时长之和必须等于该场预计时长，**允许误差 ±0.5s**。超出 = P0。
3. **场次 → 全集容差（权威值）**：全集所有场次时长之和必须等于 `本集目标时长` 或落在允许浮动区间内；60-90 秒项目默认**允许浮动 ±2s**。超出 = P0。
4. 若拆镜后发现时长对不上，不允许保留"双轨数字"；必须回到该场重分配镜头时长，直到场次和全集都闭合。
5. 场次时长是制作预算，不是叙事备注；镜头时长是剪辑预算，不是随手估数。两者必须是一套数字系统。

**容差速查表**：

| 闭合层级 | 等式 | 容差 | 超出判级 |
|---------|------|------|---------|
| 镜头 → 场次 | Σ镜头时长 = 本场预计时长 | ±0.5s | P0 |
| 场次 → 全集 | Σ场次时长 = 本集目标时长 | ±2s | P0 |
| 单镜头上限 | 单镜头时长 ≤ 6s（除非显式标注停留理由） | — | P1 |

### 外置算术验证（E3 P0 复盘沉淀 · 2026-04-12）

> **背景**：E3 分镜稿落盘前的内建自检靠"自然语言逐行审阅"判时长，漏掉了两处 P0——场次 3-2 镜头求和实际 48s 但声明 45s，全集 total shots 实际 46 但声明 31。如果没有随后的 Python 外置求和，这两处 P0 会一路炸到 `/分镜质检` 甚至 AI 生成环节。分镜稿的"时长声明"和"镜头求和"必须是 owner 亲自跑一遍算术，不能靠自然语言审阅。

**硬约束（写入 `/分镜脚本` 标准流程）**：

1. **场次 shot ≥ 15 的，落盘前必须外置算术求和**：任何一场镜头数 ≥ 15 的场次，禁止只靠 LLM 自审。落盘前必须用 Python / bc / 手算逐镜求和 `时长` 字段，把结果写入该场次 `场次头` 的"时长校验"一行，格式为：`- 时长校验：镜头求和 {N}s / 场次目标 {M}s / 状态：{一致|偏差 ±Xs}`。
2. **全集 total shots ≥ 30 的，落盘前必须外置算术求和**：任何一集 total shots ≥ 30 的，除每场场次校验外，必须再做一次"全集场次求和 = 本集目标时长"的独立校验，写入 `## 本集分镜统计` 块的"预计总时长"一行，格式同上。
3. **校验行缺失 = 落盘不通过**：没有"时长校验"行的分镜稿视为未完成，禁止进入 `/分镜质检`。`/分镜质检` 扫描到校验行缺失 = P0 直接阻塞。
4. **外置工具强制**：Python / bc / 手算三选一，不允许"在对话里让主 agent 心算"——主 agent 心算和自然语言审阅是同一类漏网模式。
5. **校验行必须留在源文件里**：不是写在 `质检检查点.md` 里，是写在每个场次头和本集统计段里。这样下游 `/分镜质检`、`/总检`、AI 生成环节都能直接读到。

**为什么这一条是 P0 不是 P1**：镜头求和错一次 = AI 视频生成环节按错误时长出素材 = 单集总时长炸穿单集目标 = 整集返工。返工成本远大于一次 `python -c "print(sum([...]))"` 的成本。

推荐做法：

- 先给每场列一个“节拍时长草表”，例如 `钩子 4s + 对抗 8s + 反杀 6s + 尾钩 5s = 23s`
- 再把每个节拍拆到镜头，确保节拍和镜头两层都能回算
- 关键镜头优先吃时长，功能重复的反应镜头和空镜优先压缩
- 生成完成后，逐场复核一次“场次预计时长 vs 镜头求和”，不一致则视为未完成

### 镜头时长的决定依据

专业分镜里，镜头时长不是“平均分配”，而是由以下四类依据共同决定：

1. **动作事实**：这个动作真实完成需要多久，例如抬手、转身、拔刀、对视、坠落、撕毁契约，不同动作的物理时长不同。
2. **信息落点**：观众需要多长时间看清关键信息，例如戒指显现、血迹出现、来电名字、银器伤痕、身份露出。
3. **情绪停留**：角色是否需要一个足够长的反应镜头让情绪落地，例如震住、心动、羞辱、反杀前压迫。
4. **全局节奏规划**：该镜头处在开场快切、中段升压、高潮放大还是尾钩硬切，不同段落的平均镜头时长本来就不一样。

禁止做法：

- 因为场次还差 `2s`，就把无动作、无新信息、无情绪变化的镜头硬拉长
- 因为要凑总时长，就额外塞入功能重复的反应镜头
- 为了让镜头表看起来整齐，忽略动作物理和表演反应时间
- 让镜头时长背离全局节奏，例如开场钩子区连续使用拖慢节奏的长停顿

判断标准：

- 若删掉该镜头的 `1-2s` 不影响事实理解、情绪落地和节奏推进，这段时长大概率是灌水
- 若该镜头的存在理由说不清“它让观众多看见了什么/多感受了什么”，就不该保留

### 竖屏拆镜原则

1. 关键信息尽量居中或偏左，避开右侧 UI 区
2. 爽点、揭露、尾钩优先用特写/近景放大
3. 对话镜头不要平均分配，要让权力更强的一方占据更稳定的构图
4. 结尾钩子优先使用“动作未完成”或“结果刚落地”的硬切，不用平淡收尾

### 镜头数量控制

- 同一功能的镜头不要机械重复
- 同一场连续 3 个镜头若都只是在“说话”，必须补动作变化、视角变化或信息变化
- 若一场拆出大量镜头仍无状态变化，优先回退修源剧本，不要靠多镜头假装节奏快

### 节拍拆镜顺序

为了避免“场景时长”和“镜头时长”各写各的，`/分镜脚本` 必须按以下顺序生成，而不是直接铺表：

1. 先写场次目标、事实锚点与本场预计时长
2. 再写 2-5 个节拍，并给每个节拍分配时长
3. 再把节拍拆成镜头/分镜，并说明关键镜头为什么值得停留
4. 最后回算场次总时长与全集总时长，并确认没有“为凑时长而存在”的镜头

若模型无法在脑内稳定回算，宁可减少 1-2 个镜头，也不要保留无法闭合的时长表。

## 六、AI 视频模型就绪要求

每个关键镜头都要形成一组最小可执行输入。字段清单（`AI画面提示` / `AI动作提示` / `连续性锁定` / `负面约束` 的内部组成）以 `references/storyboard-script-templates.md` 为唯一权威源，本文件不重复列举。

本章只追加 `storyboard-script-templates.md` 没覆盖的两项口径：

1. **写法规约**：`AI画面提示` 和 `AI动作提示` 必须短、准、可复用，不写长篇散文；运镜字段写"慢推 / 甩镜 / 跟拍 / 定镜"等可执行词，不写形容词。
2. **音视频同步信息**：每个镜头至少标注——对白或 OS/VO 落在哪一镜、哪个镜头需要字卡强化、哪个镜头需要 SFX/BGM 爆点、哪个镜头是转场切点。

## 七、回写规则

分镜阶段发现以下问题时，不能只在分镜稿里偷偷修：

- 源剧本缺动作
- 转场逻辑不成立
- 角色站位与上一场冲突
- 道具状态不连续
- 尾钩无法视觉落地
- 关键情绪只靠对白，没有画面支撑

处理规则：

1. 先最小修改 `分集剧本/*.md`
2. 再同步 `分集追踪.md`
3. 若动到稳定事实，再同步 `故事设定.md` 或 `分集大纲.md`
4. 最后继续更新 `分镜脚本/*.md`

## 八、阻塞条件

出现以下任一情况，`/分镜脚本` 应判阻塞：

- 源剧本尚未通过剧本质检
- 本场叙事目标不清
- 关键动作无法视觉化
- 镜头总时长明显超出单集时长
- 主要镜头缺少 AI 画面提示或动作提示
- 连续性锁定缺失，模型容易跑偏
- 关键尾钩只能靠说明成立，无法拍成画面
