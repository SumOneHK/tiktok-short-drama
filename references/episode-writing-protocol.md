## 目标

当整部剧的策划已经完成，后续生成第 N 集时，不能只看“当前集大纲”。要读取最小但足够的上下文包，否则越写到后面越容易出现：

- 连贯性断裂
- 人名和称谓飘移
- 功能性角色越加越多
- 台词和人设冲突
- 情节逻辑脱离现实或脱离设定
- 上下集承接断裂或重复

## 一、写新集前必须读取什么

### 最小上下文包

写 `第{NNN}集` 前至少读取：

1. `故事设定.md`（角色档案、世界规则、能力边界表必须在当前上下文中）
2. `故事结构.md` 中当前阶段相关段落
3. `分集大纲.md` 中当前集的上一集、本集、下一集三个块
4. `分集追踪.md` 最近 3-5 集（重点看悬念债务表中是否有挂起超过 5 集的未兑现悬念）
5. 已完成剧本中的前 2 集
6. 若存在 `AI资产圣经.md`（说明某批次已跑过 `/分镜脚本`），必须读取当前集涉及的角色/场景/道具**叙事事实层**资产卡；若不存在（首轮剧本生产的正常状态），**不创建、不降级**，直接以 `故事设定.md` 为叙事事实唯一来源。`AI资产圣经.md` 首次创建时机**统一收敛到** `/分镜脚本` 阶段，本命令下禁止提前创建

以下情况额外回看”首次埋设集”：

- 当前集要兑现旧悬念
- 当前集要回收旧道具
- 当前集要翻转旧关系
- 当前集要揭露旧秘密

以下情况额外检查角色档案：

- 当前集包含某角色的重大决定或立场转变
- 当前集是某角色成长弧线锚点事件所在集（见 `故事设定.md` 弧线锚点字段）
- 当前集涉及超自然能力使用（回查能力边界表）

原则：

- 不需要每次重读全剧
- 但绝不能只看当前集标题和一句话推进
- 单集 AI 附录优先写“增量状态”，不要每集重写整套角色/场景基础提示词

## 二、写前检查

**前置检查（写 14 问之前先判断）：**

本集是否为关键集？判定标准——满足以下任一条：
- 首个强卡点集
- 重大身份/秘密揭露集
- 终极反杀/公开清算集
- 感情线确认或决裂集
- 结局集

若是关键集，**必须先完成 6 节拍规划，不得跳过直接写场景正文**。6 节拍模板见 `references/story-engine.md`。6 节拍未完成 = P0，不允许进入正文。

**特殊说明：第 1 集处理规则**

第 1 集没有"上一集"，问题 1 改为：
> 前 30 秒是否完成了题材承诺的首次落地？主情绪是否已在第 1 集内有过至少一次明确释放或强烈暗示？

第 1 集不回答原始问题 1，改用上述替代检查。

---

**A 组 · 状态与连接（必答 8 项）**

1. 上一集最后一个钩子，这一集前半段怎么落地？（第 1 集见上方特殊说明）
2. 这一集真正要改变的状态是什么？（写出"起点状态 → 终点状态"，差量必须落在身份/关系/信息/筹码/情绪至少一项）
3. 这一集哪些角色必须在场，哪些角色不该出现？
4. 这一集有没有新增角色；如果有，为什么现有角色不能承担？
5. 这一集兑现什么旧承诺，新增什么新债务？
6. 这一集情绪起点和终点分别是什么？
7. 哪些名字、称谓、伤势、场景、道具必须连续？当前集是故事内第几天/哪个时段？与上一集的时间衔接是否合理？
8. 下一集最自然的承接点是什么？

**B 组 · 高质量自检（必答 5 项）**

9. 本集结尾最后一句台词/最后一个动作是否必要？删掉它，尾钩是否仍然成立？若不成立，这句话就是真正的钩子，确认它有"必看下一集"的拉力。
10. 把本集任意一场戏整段删掉，剧情是否仍能推进？若是，该场戏属于"灌水场"，必须改成有差量的事件或合并删除。
11. 本集每一个关键事件（爽点/反转/告白/揭露），是否都有明确的"见证人/反应方"？无人见证的爽点没有传播力，必须补反应镜头或反应台词。
12. 观众看完本集后，最可能记住的一个画面/一句台词/一个动作是什么？把它写下来；如果想不出，本集没有记忆点 = P1，重写关键场。
13. **本集信息差移动**：观众/主角/反派三方的"知识增量"分别是什么？本集收场时三方信息距离是"拉大 / 不变 / 缩小 / 反转"中的哪一种？谜题/秘密/失忆/双重身份类剧本，连续 3 集信息差"不变" = P1，必须推进或反转。
14. **时长外置算术闭合**（E7-E9 沉淀 · 2026-04-12 · 写完即答）：若本集给了场次时长字段（可选），把每个场次的时长区间拆开做外置加法，下限和、上限和、中位值各写一遍。中位 ∉ [73, 77] 或目标 75s ∉ [下限总计, 上限总计] = 本集结构失衡，必须调整场次时长分配直到闭合。若本集**未写**场次时长（全权交给分镜阶段），本问免答但需在集信息头明确标注"本集不给场次级时长预算，由分镜阶段全权分配"。详细规则见 §十一。

如果这 14 个问题答不清，不要开始写正文。

## 三、名字和称谓统一

名字统一靠 `故事设定.md` 的角色总表，不靠记忆。

每个角色至少固定：

- 标准中文名
- 英文展示名或别名
- 常用称谓
- 谁会怎么叫他
- 禁止混用的称呼

规则：

- 同一角色只能有一套主称谓体系
- 改称谓必须有剧情原因
- 功能性群众角色若无长期作用，用通用标签，不要随手起新名字

## 四、功能性角色不冗余

新增角色前先问：

1. 这个角色的戏剧功能是什么？
2. 现有角色能否承担？
3. 他会不会在 3 集内再次出现？

若答案是“现有角色能承担”，就不要新增。

高风险冗余角色：

- 只负责传一句消息的人
- 只负责当场见证的人
- 只负责制造一次误会的人

这些角色优先复用已有配角、家人、同事、保镖、闺蜜、律师等。

## 五、台词和人设一致

写台词前先看角色的：

- 当前情绪状态
- 当前关系温度
- 当前知道什么
- 说话风格和禁语

### 台词个性化检查（区分角色声音）

每个主要角色必须在 `故事设定.md` 中记录**说话声音档案**，写台词前必须核对：

| 字段 | 说明 | 示例 |
|------|------|------|
| 句式偏好 | 惯用长句/短句/反问/省略 | "她从不把话说完，经常突然停顿" |
| 词汇禁区 | 哪些词这个角色不会说 | "强势男主不说'可能''也许''对不对'" |
| 情绪表达方式 | 愤怒时变冷还是爆发？紧张时是否话变少？ | "他越愤怒说话越慢、越客气" |
| 信息处理方式 | 直接说意图？还是迂回暗示？ | "她从不正面回应攻击，而是反问" |

**台词个性化自检（每写完一场必做）：**

1. 把本场对白全部盖住角色名，能凭台词内容分辨出是谁在说吗？
2. 两个主角之间的对话，能感受到明显的节奏和语气差异吗？
3. 有没有"功能性台词"（主要是为了传递信息而不是角色说话）？若有，改成有角色声音的表达方式。

**禁止：同质化台词**——主角和配角说话节奏完全一样，观众靠角色名区分而不是靠声音区分，视为 P1。

台词长度约束（TikTok 短剧硬性要求）：

- 单行台词不超过 **25 个中文字 / 60 个英文字符**；长台词必须拆成多句
- 连续 3 句以上对白必须插入动作行（`△`）或反应镜头，不允许纯对话推进
- 内心独白（`OS`）不超过 2 句，超过则改为外化行动

### 按题材类型设计台词密度（2026-04-13 沉淀）

不同题材的台词策略截然不同。写台词前先确认本项目的题材类型，按对应策略执行：

| 题材类型 | 台词在叙事中的角色 | 台词密度策略 | 核心口径 |
|---------|-------------------|------------|---------|
| 法律/商战/悬疑 | 台词即武器，每句改变权力格局 | 高密度·精确·信息量大 | 一句话翻一次盘，台词=动作 |
| 甜宠/暧昧 | 台词制造暧昧张力 | 低密度·留白多·停顿>词句 | 说得越少观众脑补越多 |
| 复仇/爽剧 | 台词承载"打脸"爽感 | 短句暴击·单向输出·高密度 | 一句话让对手闭嘴 |
| 悬疑/惊悚 | 台词隐藏信息，一句多义 | 中密度·双关·伏笔 | 复看时台词里有线索 |
| 家庭/伦理 | 台词承载关系温度变化 | 中密度·生活化·潜台词 | 表面说A实际说B |
| 动作/冒险 | 台词辅助画面，不抢主导 | 低密度·命令式·短促 | 画面能说的台词不说 |

**跨题材通用原则**：

1. **台词必须改变状态**：每句台词说完后，场上的权力/信息/关系/情绪至少有一项发生变化。如果一句台词删掉不影响任何状态，它就是灌水。
2. **台词服从画面优先**：TikTok 是视觉平台——能用一个动作表达的，不用台词解释。台词的职责是做画面做不到的事（比如揭露信息、改变关系、建立权威）。
3. **台词设计时考虑字幕呈现**：TikTok 大量用户静音观看，台词同时也是字幕——短句+有力+可截图传播。长复合句在字幕上阅读体验差。

### TikTok 平台台词设计原则（2026-04-13 沉淀）

TikTok 短剧台词与传统影视剧台词的根本差异：

**1. 静音友好（50%+ 用户静音或低音量观看）**

- 关键剧情推进不能**只**靠台词——必须有对应的视觉动作或字卡强化
- 台词设计时想象"字幕版"：如果观众只看字幕不听声音，能理解发生了什么吗？
- 情绪性台词（呢喃、叹息、哽咽）必须配合表情特写，不能只靠语气传达

**2. 倍速友好（大量用户 1.5x-2x 倍速）**

- 倍速下语气消失——"轻声说"和"大声说"听起来差别不大
- 关键信息点用**字卡+台词双保险**：台词传递信息，字卡锁定关键词
- 停顿效果在倍速下缩短 50%——设计停顿时按"实际停顿 × 0.5"来估算倍速体验

**3. 金句传播性（好台词 = 免费流量）**

- 每集至少设计 1 句"可截图传播"的台词：短、有力、脱离上下文仍有意义
- 金句类型：反杀句（"Seven years. Case number stays open for ten. Do the math."）、定义句（角色身份宣言）、钩子句（让人想知道后面发生什么）
- 金句不能是功能性信息句——"打开那个文件"不会被传播，"I don't need you to believe me. I need the court to." 会

**4. 开场 3 秒台词钩子**

- 每集第一句台词必须在 3 秒内出现（或由视觉钩子替代）
- 开场台词钩子模式：
  - **悬念式**："They said she wouldn't show up."（谁？为什么不来？）
  - **冲突式**："That's not what the contract says."（什么合同？什么问题？）
  - **反转式**："I quit. — No. You're fired."（打断+反转权力）
  - **信息式**："Exhibit 7. The one they didn't want you to see."（直接抛信息钩子）
- 禁止开场台词：寒暄、背景说明、前情提要、情绪铺垫

### 分镜阶段台词-画面协同规则（2026-04-13 沉淀）

分镜稿中台词和画面共存于同一镜头时，必须考虑协同关系：

**1. 台词-动作同步性**

| 情况 | 处理方式 |
|------|---------|
| 角色边说边做（如边签字边说） | 动作不能需要双手+注意力集中，否则说话不自然。语速适当放慢 |
| 角色说完再做（如宣告后起身） | 台词时长 + 动作时长 = 总镜头时长，不能压缩 |
| 角色做完再说（如放下证物后开口） | 动作完成后留 0.3-0.5s 静默再开口，否则衔接突兀 |

**2. 台词与景别匹配**

| 景别 | 适合的台词类型 | 原因 |
|------|--------------|------|
| 特写 | 短句、金句、停顿后的关键词 | 特写放大情绪，长句分散注意力 |
| 近景 | 对话、反应、情绪表达 | 标准对话距离 |
| 中景 | 带动作的台词、多人对话 | 能同时看到肢体语言 |
| 全景 | 命令式短句、宣告 | 远距离靠声量和气势 |

**3. 镜头给谁（说话者 vs 反应者）**

- **默认给说话者**：观众需要看到谁在说
- **给反应者的情况**：(a) 台词内容是攻击/揭露，观众更想看被攻击者的反应 (b) 台词是权力宣告，镜头给屈服者更有张力 (c) 台词透露秘密，需要看到某人"听到后的变化"
- 分镜稿中明确标注"镜头给 X（说话者）"或"镜头给 Y（反应者）"，不能含糊

**4. 停顿期间画面内容**

- 台词中标注的停顿（半拍、满拍）期间画面不能空：必须有微动作（攥拳、移开视线、吞咽、抿唇）或切到反应者
- 戏剧性长停顿（>1s）期间建议切镜（从说话者切到反应者再切回）

禁止：

- 角色说出他不该知道的信息
- 上一集刚决裂，下一集说话像没事人
- 强势角色突然变成解释型工具人
- 高智角色突然为推动剧情而犯低级错误
- 用大段 OS/VO 代替可视化场景（台词能看见的不要说出来）
- 台词标注"从容/戏剧化"语速但分镜分配的时长只够快速语速——表演风格与时长分配必须一致

## 六、逻辑合理性检查

每个关键桥段都要回答：

1. 为什么现在发生？
2. 为什么是这个角色来做？
3. 为什么别人没用更简单的方法解决？
4. 代价是什么？
5. 这个结果会留下什么后果？

“不脱离现实”的意思不是只能写现实题材，而是：

- 现实题材遵守现实常识
- 奇幻题材遵守自己设定过的规则
- 不允许前面有限制，后面为了爽点直接无代价突破

### 角色重大决定的动机溯源检查

角色做出重大决定时（如：关键背叛、信任翻转、主动暴露身份、重大牺牲），必须回查 `故事设定.md` 对应角色档案的以下字段：

- 隐性伤口：这个决定是否与伤口方向一致？
- 情绪触发器：触发动机的事件是否属于该角色的触发器范围？
- 失控点：若角色行为看似”失控”，是否在设定的失控点范围内？

若决定与上述字段明显矛盾且无剧情铺垫支撑，判定为”动机断层”，必须在正文前补写铺垫或回退修设定。

### 超自然/奇幻题材的能力边界约束

凡本集涉及超常能力使用（主角/反派/能力突破或失效/角色绕过明确限制），**必须先回查 `故事设定.md` 的"能力边界"表**，对照 `references/world-character-scene.md` §能力边界设计框架的六项（正常上限/触发条件/触发代价/突破上限成本/反制方式/结局形态）做一致性校验。

**框架定义的权威来源**：`references/world-character-scene.md`。本文件不再复述六项定义，避免多处漂移。

不满足条件直接使用超常能力 = P0，禁止推进。

## 七、上下集衔接控制

### 防断裂

- 开场先承接上一集尾钩
- 承接最好在前 10%-20% 内容内完成
- 不要把上一集核心悬念拖很久不接

### 防重合

- 不要大段复述上一集
- 除非信息被新视角改写，否则不要重复同一冲突
- 相似桥段再次出现时，至少升级一项：对手、代价、信息、关系、回报

## 八、写完一集后的 6 项回填

1. 更新 `分集追踪.md` 的角色状态变化
2. 更新悬念新增与兑现
3. 更新关系温度
4. 更新场景/道具/伤势连续性
5. 标记本集是否引入新角色
6. 标记下一集必须承接的尾钩

## 九、总检重点

在 `/总检` 时额外检查：

1. 名字、称谓是否统一
2. 是否有多余功能性角色
3. 台词是否符合当前人设和关系状态
4. 情节推进是否有因果，不靠作者强推
5. 上下集是否既承接又不重复

## 十、格式合规检查

写完单集后，必须额外检查一次格式是否符合模板规范：

1. 每场是否都有 `集数-场次`
2. 每场是否都写明 `时间+内外景`
3. 每场是否都写明具体地点
4. 出场人物首次出现时是否补了角色信息
5. 动作行是否统一用 `△`
6. 心理活动是否统一用 `OS`
7. 画外音是否统一用 `VO`
8. 闪回是否成对使用 `【闪回】` / `【闪回结束】`
9. 若使用了表演重点标记，是否只写 `**【情绪爆点】**`、`**【信息落点】**`、`**【沉默压迫】**` 等叙事级强调符号；**不得**出现 `**【特写】**`、`**【中景】**`、`**【全景】**` 等景别/镜头语言（这些属于分镜阶段）
10. 语气是否统一放在角色名后的括号里
11. **反向检查**：全文是否误写了镜头号、时长求和、`AI画面提示`、`AI动作提示`、机位/运镜等分镜级字段；若有，一律删除并推迟到 `/分镜脚本` 阶段

若以上任一项缺失，不要进入批次质检，先修正文格式。

## 十一、时长外置算术 P0 硬规则（E7-E9 沉淀 · 2026-04-12）

> **背景**：`/剧本质检 E7-E9` clean-context 盲测抓到一个高频踩坑 pattern——生成 agent 写完场次时长区间后口头声称"总计约 75 秒覆盖目标"，但外置 Python 加法显示：E7 三场次实算 56-69s（缺口 6-19s）、E8 单场六拍实算 58-72s（缺口 3-17s）。根因是"约 X 秒" / "不做秒级闭合" / "分镜阶段再算"这类含糊话术让 agent 自己也信了口头 PASS，算术穿底不被发现。本节把这条教训写成硬规则，禁止进入 `/剧本质检`。

### 硬规则

1. **集信息头必须外置算术验算**（P0）：若本集场次头给了时长区间（如 `22-26 秒`），则集信息头必须同时写一行可复算的外置加法，格式：
   ```
   75 秒（N 场次外置算术：{场次1时长} + {场次2时长} + ... + {场次N时长} = {区间下限}-{区间上限} 秒 · 中位 {中位值} 覆盖 75 秒目标 · 分镜阶段闭合到 75±2s）
   ```
   这一行不是装饰，是盲测时主 agent / 质检 agent 第一眼要扫的数据凭证。缺失 = 质检 P0。

2. **禁用含糊话术**（P0）：正文和场次头**严禁**出现以下任一话术：
   - `约 N 秒` / `大约 N 秒` / `总计约 N 秒`
   - `不做秒级闭合`
   - `分镜阶段再算` / `留给分镜阶段闭合`
   - `总计符合 75 秒目标`（无区间数字支撑的空口断言）

   要么不写时长数字（交给分镜阶段全权决定），要么写了就是具体数值或具体区间并当场外置算术闭合。没有中间态。

3. **区间必须覆盖目标 ±2s**（剧本阶段规划容差）：若写区间（如 `22-26s`），N 个场次区间的下限和 + 上限和 = `[下限总计, 上限总计]`，其中位值必须落在 `75 ±2s`（73-77s）内，且 75s 必须落在区间内。**注意**：此处 ±2s 是剧本阶段的规划容差；进入分镜阶段后，实际全集时长由镜头叙事需要决定（见 `storyboard-conversion.md` §五），可以超出此区间。示例：
   - ✅ 22-26 + 26-30 + 22-26 = 70-82s · 中位 76s · 75s ∈ [70, 82] · **通过**
   - ❌ 18-22 + 18-22 + 20-25 = 56-69s · 中位 62.5s · 75s ∉ [56, 69] · **P0 缺口**

4. **B 组自检第 14 项（每集写完前必答）**：
   > **14. 本集场次时长区间外置加法是否闭合到 75±2s？** 把每个场次的时长区间拆开做 Python 风格加法，下限和与上限和各写一遍，中位值计算一遍。中位 ∉ [73, 77] 或目标 75s ∉ [下限总计, 上限总计] = 本集结构失衡，必须调整场次时长分配直到闭合。

5. **质检阶段第一刀**：`/剧本质检` 阶段主 agent 读本集首件事是扫集信息头的外置算术行，用 `python3 -c "print(22+26+...)"` 或心算复核。算术不一致 = 立即 P0 判回，不再进入后续检查项（Voice Print 盲测 / 承接检查等都免谈）。

6. **与 §十格式合规 第 11 条的关系**：§十禁止的是"镜头级时长求和"（那是分镜阶段的活），本条要求的是"场次级时长预算闭合"——两者不冲突：
   - ❌ 剧本阶段写 `镜1 3s + 镜2 4s + ... = 22s`（镜头级，越界）
   - ✅ 剧本阶段写 `1-1 场 22-26s · 三场次累加 70-82s 中位 76s 覆盖 75s`（场次级，合规）

### 违反示例 vs 合规示例

**违反示例（E8 修复前原状）**：
```
## 集信息
- 目标时长：约 75s（拍 1 10-12s / 拍 2 8-10s / 拍 3 10-12s / 拍 4 3-5s / 拍 5 15-18s / 拍 6 12-15s · 总计约 75s · 不做秒级闭合 · 分镜阶段再算）
```
外置加法：10+8+10+3+15+12 = 58s 下限，12+10+12+5+18+15 = 72s 上限，中位 65s。**75s ∉ [58, 72]，P0 缺口**。agent 口头 PASS 但算术穿底。

**合规示例（E8 修复后）**：
```
## 集信息
- 目标时长：75s（单场六拍外置算术：拍 1 12-14s / 拍 2 10-12s / 拍 3 12-14s / 拍 4 4-6s / 拍 5 17-20s / 拍 6 13-15s · 总计 68-81s · 中位 74.5s 覆盖 75s 目标 · 分镜阶段闭合到 75±2s）
```
外置加法：12+10+12+4+17+13 = 68s 下限，14+12+14+6+20+15 = 81s 上限，中位 74.5s。**75s ∈ [68, 81]，且中位 74.5 ∈ [73, 77]，通过**。
