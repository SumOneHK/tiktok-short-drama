# 多剧工作区与活动剧本目录规则

## 目标

当前工作目录默认是**多剧工作区**，一部剧必须有一个独立的完整剧本目录。所有业务产物都写入该剧自己的目录，避免多部剧的状态、设定、大纲、剧本、分镜和导出文件互相污染。

## 目录模型

```text
{工作区}/
├── {剧A目录}/
│   ├── 剧本状态.json
│   ├── 选题分析.md
│   ├── 故事设定.md
│   ├── 故事结构.md
│   ├── 分集大纲.md
│   ├── 分集追踪.md
│   ├── 分集剧本/
│   ├── 分镜脚本/
│   ├── 总检报告.md
│   ├── 合规报告.md
│   └── 导出/
└── {剧B目录}/
    └── ...
```

`references/`、`templates/`、`scripts/` 永远属于 skill 目录；`剧本状态.json`、`故事设定.md`、`分集剧本/` 等业务文件永远属于活动剧本目录。

## 术语

- **工作区目录（WORKSPACE_DIR）**：用户当前要求工作的目录。若当前目录本身含 `剧本状态.json`，其父目录可视为工作区；若当前目录不含 `剧本状态.json`，当前目录就是工作区。
- **活动剧本目录（ACTIVE_DRAMA_DIR）**：本次命令实际读写业务文件的目录。命令文档中所有业务文件相对路径都以它为根。
- **剧本目录名（project.directoryName）**：活动剧本目录的文件夹名，不等同于正式剧名；正式剧名写在 `dramaTitle`。

## 活动剧本目录解析顺序

每条命令读写业务文件前，先按下列顺序解析 `ACTIVE_DRAMA_DIR`：

1. **当前目录就是剧本目录**：若当前目录存在 `剧本状态.json`，直接使用当前目录。
2. **用户显式指定目录**：若用户给出项目目录、剧名目录、`--项目`、`项目：xxx`、`在 xxx 里` 等明确指向，使用工作区下对应目录。非 `/开始` 命令要求该目录已存在且含 `剧本状态.json`。
3. **`/开始` 新建项目**：若当前目录不是剧本目录，且用户是在启动新剧，必须在当前工作区下新建一个子目录作为 `ACTIVE_DRAMA_DIR`，再在其中创建 `剧本状态.json` 与 `选题分析.md`。禁止把这两个文件直接落在工作区根目录。
4. **非 `/开始` 自动恢复**：若当前工作区下恰好只有一个子目录含 `剧本状态.json`，可自动选择该目录。
5. **多项目歧义**：若当前工作区下有多个子目录含 `剧本状态.json`，且用户未明确指定，必须列出候选目录并请用户选择；不得随意选最近修改的项目，也不得在工作区根目录新建散落文件。

## 新建剧本目录命名

`/开始` 阶段新建目录时：

1. 若用户已提供清晰剧名或项目名，用其生成安全目录名。
2. 若还没有剧名，用 `未命名短剧-YYYYMMDD-HHMM` 作为临时目录名。
3. 目录名必须去掉或替换 `/ \ : * ? " < > |` 等不安全字符，空格可替换为 `-`。
4. 若目录已存在，追加短后缀，如 `-02`、`-03`。
5. 正式剧名后来变化时，只更新 `剧本状态.json.dramaTitle`；不要自动重命名目录，除非用户明确要求。

## 状态文件记录

新建 `剧本状态.json` 时，除业务字段外，必须写入：

```json
{
  "project": {
    "directoryName": "{当前剧本目录名}",
    "displayTitle": "",
    "createdAt": "YYYY-MM-DD",
    "workspaceMode": "multi-drama"
  }
}
```

当 `/立项` 确认正式剧名后，同步更新 `project.displayTitle` 与 `dramaTitle`。

## 读写路径规则

- 命令文档中出现的 `故事设定.md`、`分集大纲.md`、`分集剧本/第NNN集.md`、`导出/` 等路径，全部解释为 `{ACTIVE_DRAMA_DIR}/...`。
- 命令文档中出现的 `references/...`、`references/templates/...`、`scripts/...`，全部解释为 skill 目录下路径，不随 `ACTIVE_DRAMA_DIR` 改变。
- shell 命令建议在 `ACTIVE_DRAMA_DIR` 作为 working directory 执行；需要调用 skill 脚本时，用 skill 目录中的脚本路径，并显式传入 `ACTIVE_DRAMA_DIR` 下的 source/output。
- 任何命令不得在工作区根目录直接创建 `剧本状态.json`、`选题分析.md`、`故事设定.md`、`分集剧本/`、`分镜脚本/`、`导出/` 等业务产物，除非当前目录本身就是活动剧本目录。

## 多剧隔离检查

执行任何创建或写入操作前，先确认目标路径在 `ACTIVE_DRAMA_DIR` 内。若发现工作区根目录已有散落业务文件：

1. 不自动移动，不删除。
2. 提醒用户这些文件不符合多剧目录规则。
3. 继续本次任务时，只读写已解析出的 `ACTIVE_DRAMA_DIR`。
