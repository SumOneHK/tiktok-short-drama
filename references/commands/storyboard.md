# /分镜脚本 命令详细执行规范

> 本文件是 `/分镜脚本 {起止集}` 命令的自包含说明书。进入命令后先读本文件，再按本文件的分层加载规则读取必要引用；不需要再回 `SKILL.md` 查加载规则。

## 用途

把已通过剧本质检的单集剧本转成标准、专业、可执行、可直接喂给 AI 视频模型的分镜脚本。

## 分层加载规则

**核心层（每次进入 `/分镜脚本` 必读）：**

工作区定位：

- `references/workspace-projects.md` — 解析 `ACTIVE_DRAMA_DIR`；本命令所有业务文件都从该目录读写

业务数据文件（精读命中批次，不整文件 dump）：

- `故事设定.md` — **只读角色表、场景池、能力边界**
- `AI资产圣经.md` — **只读叙事事实层 + 全局生成规则**，若缺失先基于 `故事设定.md` 创建最小版本
- `分集大纲.md` — **只读本批次所在集段**
- `分集追踪.md` — **只读当前连续性状态**
- 本批次对应的 `分集剧本/第NNN集.md`

规则参照文件：

- `references/storyboard-conversion.md` — 拆镜顺序、镜头粒度、阻塞条件、回写规则
- `references/storyboard-script-templates.md` — 单集分镜脚本权威模板

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 首次进入 `/分镜脚本`（尚未建立 AI 资产圣经基线） | `references/ai-production.md`（**只读 §一 全局资产圣经、§二 制作锚点的最低字段、§三 artStyle 与统一交付管线**）+ `references/tiktok-platform-constraints.md` §一·附 TikTok 竖屏安全区规范 |

## 执行要求

1. **前置质检闸门（强制 P0）**：进入 `/分镜脚本 {起止集}` 前必须核对 `qcStatus.episodes`——
   - 必须存在一条或多条记录，其 `checkType == "script"`、`status == "已通过"`，且 `range` 合并后完整覆盖本次目标集数
   - 目标集数中任一集未被已通过的 `script` 质检 range 覆盖 → 阻塞并提示用户先跑 `/剧本质检 {缺失区间}`
   - 目标集数中任一集落入 `status == "需修改"` 的 `script` 质检 range → 阻塞并提示用户先修复对应批次
   - 若 `deliveryProgress.scriptCompletedRanges` 已覆盖 `1..episodeCount`，且 `checkType=script` 已通过覆盖全剧，则还必须存在覆盖全剧的 `checkType == "full-read"`、`status == "已通过"` 记录；缺失或 `需修改` → 阻塞并提示先执行 `/整剧通读`
   - 不允许以"刚写完手感没问题"为由绕过；前置闸门未通过时**不得**读取任何分镜模板或开始拆镜
2. 分镜脚本是制作稿，不是对白稿重排。必须把"场景目标 -> 动作节拍 -> 镜头/分镜"拆到可执行粒度。
3. 单集权威模板只认 `references/storyboard-script-templates.md`；拆镜顺序、镜头粒度和回写规则只认 `references/storyboard-conversion.md`。
4. 分镜稿默认继承 `AI资产圣经.md` 的全局角色/场景/道具资产，单集附录只补状态变化、增量锁定词和镜头级特例；若拆镜中发现全局资产基线不够稳定，先回写 `AI资产圣经.md`，再继续。角色继承必须按"一个角色卡 + Identity Anchor + Sxx 外观状态"执行：同一角色的现在线、过去线、失忆前、复仇后等差异只能匹配到同一角色卡下的 `Sxx`，不得拆成多个角色设定或在镜头 prompt 中重新写一套脸。
5. **回写剧本字段白名单（强制 P0）**：任何镜头都不得凭空新增剧情结论、人物动机或关键设定；若拆镜过程中发现源剧本缺失必要动作、转场或连续性信息，先回写 `分集剧本/*.md` / `分集追踪.md`，再继续生成分镜稿。**但回写 `分集剧本/*.md` 时只允许补齐以下叙事层字段**——
   - ✅ **允许**：场景状态差量、事实锚点（人物知道/不知道信息 / 伤势 / 道具 / 站位 / 服装）、承接点、遗留点、情绪位置、尾钩描述、动作行（`△` 描述人物动作和表演调度）、对白/OS/VO
   - ❌ **严禁**：镜头号、分镜级时长、景别、机位/角度、运镜、`AI画面提示`、`AI动作提示`、连续性锁定词、负面约束——这些**一律只写进 `分镜脚本/*.md`**，禁止倒灌回剧本稿
   - 违反白名单（把镜头级字段写进剧本稿）= `/剧本质检` 反向检查直接判 **P0**，本批次剧本 `qcStatus.episodes[{range: 起止集, checkType: "script"}].status = 需修改`；回写完成后必须同步更新 `分集剧本/第NNN集.md` 的 YAML frontmatter `version` 字段（+1），触发分镜稿的 `source_episode_version` 重新对齐
   - **回写后强制复检（P0）**：任何回写操作（无论是否在白名单内）完成后，必须对该集**单独**执行 `/剧本质检 {该集}`（不需要重跑整个批次），确认回写内容不破坏叙事连续性。复检不通过 = 回写操作无效，需撤回并重新处理。若该集已通过 `/整剧通读`，还必须把覆盖全剧的 `checkType="full-read"` 记录回退为 `未检查`；若已有旧分镜稿，必须把对应 `qcStatus.storyboards[*]` 回退为 `未检查` 或 `需修改`，并将 `qcStatus.production`、`qcStatus.compliance` 回退为 `未检查`。
6. 每个镜头/分镜至少写清镜头号、预计时长、景别、机位/角度、主体动作、情绪目标、台词/音频、字卡/特效、AI 画面提示、AI 动作提示与制作备注；竖屏安全区与 UI 避让必须在字卡和构图字段中显式标注。
7. 本 skill 的单集目标区间是 60-90 秒竖屏 AI 视频，但 90 秒不是硬性上限；根据源剧本动作事实、对白密度、情绪停留和尾钩落点，分镜可进入 90-108 秒弹性区间（+20% 以内）。超过 90 秒必须写明叙事理由；超过 108 秒默认回修或拆集，除非用户另定平台规格。所有分镜稿都用同一套镜头语言：景别、机位、运镜、动作起止、AI 画面提示、AI 动作提示。具体艺术风格（写实电影感 / 港风复古 / 国潮水墨 / 赛博朋克 / 厚涂插画 / 3D 卡通 等）由 `AI资产圣经.md` 的 `artStyle` 全局生成规则统一定义，不在分镜模板里硬编码；artStyle 切换不影响镜头语言字段，只影响 AI 画面提示的关键词。
8. 拆镜必须服务节奏：前 3-5 秒直接落地钩子；关键反转、爽点、揭露和尾钩必须有独立镜头放大；说明性对白不得原样平铺成连续镜头；所有关键镜头都要具备可单独投喂模型的 prompt 完整度。
9. 拆镜前必须先锁本集与本场事实锚点，包括人物知道/不知道什么、伤势/服装/道具状态、上一集尾钩承接点和本集尾钩切口；镜头只能表达、放大或切分这些事实，不能为了表格好看而改写事实。对每个出场角色必须先确定 `characterId + Sxx`：长期换装、年龄跳跃、持续伤势使用新的 `Sxx`；一次性哭痕、湿发、血迹、灰尘、临时配饰只写入本镜连续性锁定，不新增 `Sxx`。
10. 时长必须逐层闭合：每场 `本场预计时长` 必须等于场内所有镜头/分镜时长求和，全集 `本集目标时长`、各场时长和 `本集分镜统计` 必须闭合；不一致时先重分配时长，再算完成。
11. 镜头时长必须由动作事实、信息落点、情绪停留和全局节奏共同决定，不得为了凑时长无故拉长停顿、堆重复反应镜头或添加功能空镜。
12. 海外模式下，仅对白、旁白、`OS`、`VO`、屏幕字卡允许使用英文；文档内如需便于中文团队审校，可在对应条目补 `中译`，但成片上屏字卡默认只保留目标市场语言，不叠加中文屏幕字。
13. 首次进入 `/分镜脚本 {起止集}` 时：若 `分镜脚本/` 不存在才创建；若 `AI资产圣经.md` 不存在，按核心层业务文件规则先基于 `故事设定.md` 创建最小版本（`AI资产圣经.md` 的首次创建时机统一收敛到本阶段），同时按条件层触发 `references/ai-production.md` 的 §一/§二/§三 精读 + `tiktok-platform-constraints.md §一·附` 竖屏安全区精读；只创建本批次需要的集数文件，不预建后续空集。
14. 每批分镜稿完成后，若发现与源剧本不一致的执行信息，必须按最小修改集同步修正 `分集剧本/*.md`、`分集追踪.md`、`AI资产圣经.md` 和必要的源文档（回写剧本时严格遵守第 5 条白名单）。
    - **分镜复用判定规则**：若源剧本被大幅重写（场次序列变更、情绪峰值时机重排、核心冲突替换），已有分镜稿的 `source_episode_version` 检查必然失败。此时**必须重新执行 `/分镜脚本` 生成**该集分镜，不得手工修补旧分镜。仅当回写只涉及白名单内叙事层字段（事实锚点、台词、动作行）且 version 匹配时，旧分镜可继续使用。
15. 每批分镜稿完成后执行一次"分镜批次内建自检"；关键卡点集、高潮集和尾钩集必须执行 `/分镜质检 {起止集}`。通过后按 `references/state-schema.md` §滚动生产的 currentStep 语义 重新判定 `currentStep`，不得一律硬写 `分镜脚本`。
16. 若全剧尚未完成全部分镜，但用户只要求继续下一批剧本生产，可暂时返回 `/分集剧本`；返回时必须先在 `deliveryProgress` 中写清已完成分镜区间和待补区间，并按 `references/state-schema.md` §滚动生产的 currentStep 语义 更新 `currentStep`，避免现场丢失。
17. **分镜变更后的漂移回滚（强制）**：若本命令修改 `AI资产圣经.md` 的全局生成规则、`artStyle`、角色 `Identity Anchor`、`Sxx` 长期状态、场景/道具资产基线，必须把受影响分镜批次的 `qcStatus.storyboards[*]` 回退为 `未检查` 或 `需修改`；若这些资产字段与 `故事设定.md` 共享，必须同步回写 `故事设定.md` 并把 `qcStatus.bible` 及全部下游状态按管线回退。任何分镜稿内容变更后，`qcStatus.production`、`qcStatus.compliance` 必须回退为 `未检查`。
