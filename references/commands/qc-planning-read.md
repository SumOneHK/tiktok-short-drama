# /策划通读 命令详细执行规范

> 本文件是 `/策划通读` 命令的自包含说明书。进入命令后先读本文件，再按本文件的分层加载规则读取必要引用；不需要再回 `SKILL.md` 查加载规则。

## 用途

在 `/大纲质检` 通过后、首批 `/分集剧本` 开写前，从**市场点击 / 观众追更 / 编剧结构 / 导演与 AI 漫剧可视化 / 本地化可信度 / 反向否决**等视角通读完整策划包，判断“这个项目现在是否值得进入分集剧本成本阶段”。

本命令不是另一个格式质检，不替代 `/大纲质检`，也不替代全剧剧本完成后的 `/整剧通读`：

- `/大纲质检` 检查结构是否可用、逐集是否承接、卡点是否兑现
- `/策划通读` 判断策划作为产品是否有足够吸引力，是否值得开写试点剧本
- `/整剧通读` 在全剧分集剧本完成后，检查成品剧本整体观看体验

## 触发时机

以下任一条件成立时，首批 `/分集剧本 {起止集}` 前必须执行 `/策划通读`：

- `mode=overseas` 或 `languageMode=overseas-en-dialogue`
- `episodeCount >= 30`
- 用户明确要求“精品漫剧 / 复杂世界观 / 出海本地化 / 高概念 / IP感”
- 题材包含科幻、悬疑、奇幻、超自然、异能、复杂阶级制度、调查阴谋、强藏线
- artStyle 含漫剧、动态漫画、AI 视频、强风格化视觉

简单现实题材、国内版、24 集以内项目可跳过，但必须在 `质检检查点.md` 或响应里写明跳过原因。拿不准时默认执行。

## 重点检查

### 1. 市场点击视角

- 3 秒钩子是否能不用解释讲清“谁出事、出了什么事、为什么必须看下去”
- 前 3 集是否连续兑现，而不是只抛设定
- 标题、核心画面、开场危机是否适合 TikTok / Reels / Shorts 竖屏滑动场景
- 首个付费点前是否已经给到足够价值，而不是用硬切强收

### 2. 观众追更视角

- 每 5-8 集是否都有明确回报轮换：爽、疼、惊、急、心动、打脸、关系翻页、代价落地至少一种
- 中段是否有“下一集必须看”的个人代价，而不是只靠证据编号、系统节点或地图推进
- 尾钩是否从符号变成具体欲望：谁会倒霉、谁会被救、谁要付代价、主角会不会越界

### 3. 编剧 / Showrunner 视角

- 主线是否集中，是否每阶段都能回到主角外在目标和内在弧线
- 主角是否主动推动关键结果，不能被男主、系统漏洞或配角替她解决核心问题
- 反派是否有可感知压迫动作，而不是抽象制度或企业标签
- 结局是否回收第一季主承诺，同时留下可接受的续季钩子

### 4. 导演 / AI 漫剧可视化视角

- 前中后段是否各有可切片名场面
- 高价值画面是否阶段升级，不能只重复同一类房间、屏幕、文件、对峙
- 抽象信息是否已经转译成动作、表情、物件、空间变化或可截图字卡
- AI 资产规模是否可控，常驻角色/场景/道具是否足够复用

### 5. 本地化可信度视角

- 出海项目是否基于跨文化可读的现实情绪，如住房、医疗、阶级、算法、气候、身份认证
- 权力关系是否已经从中式宗族、孝道、地方体制、中文社会黑话改成目标市场可理解的制度关系
- 英文对白未来是否能自然短句化，不依赖长篇中文解释

### 6. 反向否决视角

必须明确回答：如果现在不建议开写，根因是什么？

- 题材点击弱
- 主角驱动力弱
- 反派伤害弱
- 中段空转
- 世界观术语压过情绪
- 本地化不成立
- AI 漫剧可视化不稳定
- 付费点不够硬

## 阻塞条件

任一命中 = P0，`qcStatus.planningRead = 需修改`，不得进入首批 `/分集剧本`：

- 3 秒钩子仍需解释世界观才能看懂
- 前 3 集没有交付明确危机、人物目标和至少一次高价值发现/反击/代价
- 主角缺少主动选择，关键推进主要由男主、配角、系统或巧合完成
- 首个付费点前没有足够观看价值，或付费点后 1-3 集没有兑现路径
- 中段核心推进连续依赖术语、证据编号、文件、系统节点，缺少可见代价和人物选择
- 反派没有具体伤害动作，只是抽象制度、公司或算法
- 出海版核心冲突依赖中文社会背景知识，欧美观众无法自然理解
- 结局不回收第一季主承诺，只用更大世界真相转移问题

以下为 P1，必须在开写首批剧本前优化或登记为接受风险：

- 主角人物钉子不够鲜明，动作和台词缺少可复用母题
- CP/同盟线功能成立但情感回报偏弱
- 视觉名场面集中在前段或终局，中段可切片场面不足
- 反派可恨但缺少个人弱点或反噬逻辑
- 术语数量可控但仍需要在剧本阶段强制转译成短句和动作
- 首批试点集风险较高，需限定为 `001-005` 或更小批次开写

P2 可以延后，但必须写明回看时点：

- 字段命名不完全统一但内容可读
- 部分配角高光偏晚
- 个别视觉资产需等 `/分镜脚本` 阶段再锁定

## 执行要求

1. 前置门禁：必须确认 `qcStatus.outline = 已通过`；若未通过，先执行或修复 `/大纲质检`。
2. 默认只读策划包并输出判断；不得创建 `分集剧本/`，不得写任何分集剧本正文。
3. 按六个视角通读，输出清晰结论：`继续` / `需优化后继续` / `暂停重构`。
4. 必须给出综合吸引力评分（10 分制）、最大卖点、最大风险、分集剧本前必改项、是否建议先写试点批次。
5. 若用户要求“先做优化再通读”，允许按 `document-sync.md` 修源文档；只要改动共享字段，就必须触发跨文档漂移复检并按管线回写相关 `qcStatus`。
6. 通过时：写 `lastQcStep = "/策划通读"`，`qcStatus.planningRead = 已通过`，`currentStep` 保持或回到 `分集剧本`。
7. 不通过时：写 `lastQcStep = "/策划通读"`，`qcStatus.planningRead = 需修改`，并按最早受影响文档回退 `currentStep`（通常为 `分集大纲`；若需改设定或结构，则回退到对应阶段）。
8. 本命令只创建或更新 `质检检查点.md` 与 `剧本状态.json`；除非用户明确要求优化，否则不修改 `选题分析.md`、`故事设定.md`、`故事结构.md`、`分集大纲.md`、`分集追踪.md`。
