# /分镜质检 {起止集} 命令详细执行规范

> 本文件是 `/分镜质检 {起止集}` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

对已完成的分镜脚本批次做独立闸门质检，通过后才允许进入 `/总检` 或 `/导出`。

- 本文件 §重点检查 + §阻塞条件（闸门规则已内嵌，**不需要再读 process-qc.md §三**）

- `references/quality-gate.md` §响应固定结构

- `references/storyboard-script-templates.md` §统一硬字段（**14 字段 P0/P1 判级表**为唯一权威源） —— 镜头字段完整度判级

## 重点检查

- 镜头拆解是否达到镜头级，而不是场景描述换行
- **关键镜头 14 字段完整度**：按 `references/storyboard-script-templates.md` §统一硬字段判级表逐镜核对，关键镜头任一 P0 字段缺失 = P0；标准镜头 P0 字段缺失且无合法 ↑继承 = P0
- 每个关键镜头是否具备 `AI画面提示` 与 `AI动作提示`
- 时长、镜头数和节奏是否符合单集约束
- **时长闭合（权威值，均为 P0）**：镜头求和 = 本场预计时长（**±0.5s**） / 场次求和 = 本集目标时长（按叙事需要推导，**±0.5s**） / 单镜头 ≤6s 软上限需备注停留理由（P1） / 单镜头 ≤15s 硬上限（P0） / 对白镜头时长须覆盖台词说完所需时间（P0）—— 速查表见 `references/storyboard-conversion.md` §五 时长守恒规则
- 每场 `本场预计时长` 是否与场内镜头/分镜时长求和一致
- 本集目标时长、场次时长、分镜统计三层数字是否闭合
- 镜头时长是否来自动作事实、信息落点、情绪停留和全局节奏，而不是为了补足秒数
- 本场事实锚点是否被镜头正确承接
- 连续性锁定、负面约束、字卡安全区是否完整
- 是否正确继承 `AI资产圣经.md` 的全局角色/场景/道具资产
- 分镜稿与源剧本是否一致，是否偷偷新增剧情信息
- 海外模式：中文是否只存在于文档说明/中译，没被误写成要上屏的双语字卡
- 尾钩、爽点、揭露镜头是否具备单独生成价值
- 镜头分配是否把表演高光给到真正需要发光的角色
- **情绪峰值镜头一致性**：本集情绪最高点（情绪峰值集数与情绪目标已在 `分集大纲.md` 标注）必须有对应的独立放大镜头，且该镜头 `情绪目标` 字段与大纲描述一致；最强情绪点被分散到普通对话镜头中平铺 = P1
- **时间线归属一致性（含闪回/失忆/双时间线/穿越类项目必查）**：每镜 `时间线归属` 字段必须能在 `分集追踪.md` §五"双时间线/闪回连续性账"中找到对应轨；同一角色在不同时间线的造型/伤势/道具与轨道版本不符 = P0；`时间线归属` 字段缺失（在含闪回项目中）= P0

## 阻塞条件

- 关键镜头缺 prompt
- 镜头粒度过粗，无法直接投喂模型
- 时长失控
- 场次时长与镜头求和不一致
- 本集目标时长、场次时长、分镜统计无法闭合
- 出现仅为凑时长而存在的镜头或长停顿
- 镜头时长明显违背动作事实、信息理解时间或全局节奏
- 连续性锁定缺失
- 分镜稿与源剧本冲突
- 海外成片字卡仍含中文上屏指令
- 关键高潮没有独立放大镜头
- 关键角色高光镜头被平均化处理
- 全局资产继承缺失或单集增量覆盖不清

## 执行要求

1. 只读本批次 `分镜脚本/第NNN集.md`、对应 `分集剧本/第NNN集.md`、`AI资产圣经.md`、`剧本状态.json`；**不得**修改主流程产物。
2. 按本文件 §重点检查 + §阻塞条件逐项扫描。**重点关注**：
   - 时长闭合：镜头求和 = 场次时长（±0.5s）/ 场次求和 = 集时长（按叙事需要推导，±0.5s）/ 对白镜头时长覆盖台词
   - 每关键镜头 `AI画面提示` + `AI动作提示` 是否齐全
   - 连续性锁定 + 负面约束是否完整
   - 全局资产继承是否正确（不得每集重新发明锁定词）
   - 分镜稿与源剧本事实是否一致，是否偷改剧情
   - 海外模式：中文是否只存在于文档说明/中译，没被误写为上屏字卡
3. 若 `质检检查点.md` 不存在则当场创建。
4. 出现任意 P0（时长失控 / 关键镜头缺 prompt / 分镜与剧本冲突 / 字卡越线）—— 写 `qcStatus.storyboards[{起止集}] = 需修改`；**禁止**推进到 `/总检` 或 `/导出`。
5. 全部通过 —— 写 `qcStatus.storyboards[{起止集}] = 已通过`。
6. 长剧模式下按 `process-qc.md` §四 批次质检规则 → 长剧（40集以上）分镜质检分级 标注 A/B/C 级。
7. 响应结构严格按 `quality-gate.md` §响应固定结构。
8. 本命令**不**创建任何主流程产物，只更新 `质检检查点.md` 与 `qcStatus.storyboards`。
