# /结构质检 命令详细执行规范

> 本文件是 `/结构质检` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

对 `故事结构.md` 做独立闸门质检，通过后才允许进入 `/分集大纲`。

## 分层加载规则

**核心层（每次进入 `/结构质检` 必读）：**

- 本文件 §重点检查 + §阻塞条件（闸门规则已内嵌，**不需要再读 process-qc.md §三**）
- `references/state-schema.md` §状态同步规则 —— `/结构质检` 回写 `qcStatus.architecture` 字段的规则
- `references/quality-gate.md` §响应固定结构
- `references/templates/qc-checkpoint.md`
- `references/viewer-payoff-contract.md` §一 价值优先级 + §三 证据型故事的本质约束 + §十一 大纲到剧本的戏剧锁点
- `references/script-quality-engine.md` §二 三密度模型 + §三 单集节奏锚 + §四 钩子设计 + §五 拉扯感 + §九 质检口径

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 情绪曲线或爽点密度存疑 | `references/emotion-design.md` |
| 卡点设计存疑 | `references/paywall-rules.md` |
| 结构与题材匹配存疑 | `references/story-engine.md` §题材与结构类型对照 |
| 发现跨文档冲突 | `references/document-sync.md` |

## 重点检查

- 情绪曲线是否有波形
- 悬念债务是否能回收
- 爽点和卡点是否互相支撑
- 中段是否疲劳
- 结局回收是否有路径
- 高价值场面是否清晰，关键翻页是否具备可视化动作而不只是信息反转
- **观看回报契约是否成立**：结构是否写清主回报、副回报、每阶段回报类型，以及前 3 集/前 8-12 集的明确兑现；不能只有事件推进和设定揭示
- **证据/专业题材价值函数是否正确**：若为证据、法律、调查、商战题材，检查核心证据是否绑定旧伤层、打脸层、后果层；只写编号、文件名、法条或账户链 = P1
- **中段升级是否升级观看回报**：中段升级不能只是证据编号递增或对手层级变大，必须体现公开场合、被打脸者、现实代价、关系翻页或情绪代价升级
- **结局惩罚与情绪回报是否匹配**：若法律惩罚因题材/合规不能很重，结构必须安排公开羞辱、资产冻结、家族瓦解、亲人背离等情绪惩罚补偿
- **核心角色成长弧线**：第 1 集内在状态 → 结局内在状态是否明确，且与核心冲突互相推动而不是平行推进（角色弧线与情节弧线脱节 = P1）
- **三密度主引擎**：结构是否写清情绪密度、信息密度、情节密度分别如何驱动全剧。只写事件列表或设定列表，没有三密度主引擎 = P1；前 3 集承诺段也缺失 = P0
- **前 3 集节奏承诺**：前 3 集是否具备开头冲击、局势变化、强期待和尾钩的连续升级。第 1 集仍以背景说明、世界观介绍或慢描环境为主 = P0
- **钩子系统是否改写局势**：钩子不能只是“有人来了 / 文件出现 / 门被推开”；必须改变角色下一步选择。全剧主钩子多次不改写局势 = P1
- **爽点前拉扯**：大爽点和付费卡点前是否有压低、错误放松、再加压的路径；若总是直接受压后立刻翻盘，中段爽感会轻，判 P1

## 阻塞条件

- 只有事件堆叠，没有情绪波形
- 核心悬念只埋不收
- 卡点前无回报
- 中段无法升级
- 重大翻页只有设定解释，没有可放大的动作或场面
- 核心角色成长弧线缺失或与情节主冲突完全平行无耦合
- 结构主驱动力是编号/证据/程序递增，而非情绪债务和观看回报递增
- 证据型题材没有“旧伤-打脸-后果”总账，后续只能写成找证据闯关
- 第 1 集或前 3 集承诺以背景说明代替强冲突和强情绪
- 关键卡点只有事件名，没有三密度、节奏锚、拉扯路径或可见回报

## 执行要求

1. 只读 `故事结构.md`、`故事设定.md`、`剧本状态.json`；**不得**修改主流程产物。
2. 按本文件 §重点检查 + §阻塞条件逐项扫描。特别关注：情绪波形、悬念债务回收路径、卡点-爽点互补、中段升级可行性。
3. 若 `质检检查点.md` 不存在则当场创建。
4. 出现任意 P0 —— 写 `qcStatus.architecture = 需修改`；**禁止**推进到 `/分集大纲`。
5. 全部通过 —— 写 `qcStatus.architecture = 已通过`，并将 `currentStep = 分集大纲`。
6. 响应结构严格按 `quality-gate.md` §响应固定结构。
7. 本命令**不**创建任何主流程产物，只更新 `质检检查点.md` 与 `qcStatus.architecture` / `currentStep`。
