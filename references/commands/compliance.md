# /合规 命令详细执行规范

> 本文件是 `/合规` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

做内部前置合规筛查，不替代法务或平台终审。

## 分层加载规则

**核心层（每次进入 `/合规` 必读）：**

工作区定位：

- `references/workspace-projects.md` — 解析 `ACTIVE_DRAMA_DIR`；本命令所有业务文件都从该目录读写

业务数据文件：

- `故事设定.md` — **只读角色表、世界规则、本地化替换**
- `故事结构.md` — **只读核心冲突、卡点清单**
- `分集大纲.md` — **只读本批次尾钩与关键集**
- （若存在）`分镜脚本/*.md` 抽样：**含敏感内容的集**
- `总检报告.md` — **只读未闭环项**

规则参照文件：

- `references/compliance-rules.md` — 合规红线与审核口径
- `references/templates/compliance-report.md`（`合规报告.md` 固定模板）

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| `mode=overseas` 且需复查英文对白、屏幕字卡、本地化合规 | `references/language-mode.md` + `references/tiktok-platform-constraints.md`（只读本地化与竖屏字卡段） |
| 合规报告产出时需对齐响应格式 | `references/quality-gate.md` |

## 执行要求

0. **前置质检闸门（P0）**：进入 `/合规` 前必须确认 `qcStatus.production = 已通过`；若为 `未检查` 或 `需修改`，先执行或修复 `/总检`，不得直接做最终合规结论。
1. `合规报告.md` 固定模板以 `references/templates/compliance-report.md` 为唯一权威源。
2. **逐项核对通用红线 R-01 ~ R-06**：按 `references/compliance-rules.md` §通用红线表格逐 ID 扫描全剧场次/对白/字卡，每条红线必须给出"通过 / 命中（场次定位 + 改写出口）"标记；命中任一 = P0，写入 `合规报告.md` 并禁止推进。
3. 出海模式按 TikTok/海外平台高风险类别检查，国内模式按国内微短剧常见红线检查。
4. 首次进入 `/合规` 时，若 `合规报告.md` 不存在才创建；不要提前创建 `导出/`。
5. 若存在 `P0` 合规风险，不得进入 `/导出`。
