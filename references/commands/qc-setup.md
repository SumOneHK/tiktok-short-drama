# /设定质检 命令详细执行规范

> 本文件是 `/设定质检` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

对 `故事设定.md` 做独立闸门质检，通过后才允许进入 `/结构`。

## 分层加载规则

**核心层（每次进入 `/设定质检` 必读）：**

- 本文件 §重点检查 + §阻塞条件（闸门规则已内嵌于下方，**不需要再读 process-qc.md §三**）
- `references/state-schema.md` §状态同步规则 —— `/设定质检` 回写 `qcStatus.bible` 字段的规则
- `references/quality-gate.md` §响应固定结构 —— 输出格式
- `references/templates/qc-checkpoint.md` —— 若 `质检检查点.md` 不存在先按此模板创建

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 含超自然/奇幻/能力系统 | `references/world-character-scene.md` §能力边界设计框架 |
| `languageMode=overseas` | `references/language-mode.md` §`mode=overseas` + `references/world-character-scene.md` §海外本地化 |
| 发现跨文档冲突 | `references/document-sync.md` |
| 高风险题材 | `references/compliance-rules.md` |

## 重点检查

- 世界规则是否自洽
- 命名和称谓体系是否稳定
- 角色功能是否重叠
- 场景池是否足够复用
- 人设与后续情绪引擎是否一致
- 核心角色是否具备一眼记住的标签、首次高光场景和明确的关系站位
- 每个核心角色的"主要戏剧功能"是否能真正在剧情中被使用（不是只写在设定里，要能落到具体的冲突或推进）
- 是否存在功能重叠但尚未合并的角色
- 若为出海模式，人名、称谓、身份标签、场景文化和不可直译表达是否都已完成本地化替换
- **若含超自然/奇幻/能力系统**：每种核心能力的"能力边界六项"是否全部填写完整；任意一项空缺 = P0（六项权威定义见 `references/world-character-scene.md` §能力边界设计框架）
- **主要世界规则前 3 集行动演示**：每条核心世界规则在 `故事设定.md` 中是否已规划"前 3 集如何用具体场景演示"，不允许只用对白说明规则
- **全局视觉基调**：是否已明确主色调、情绪色彩规则、禁止色调；常驻场景视觉锚点是否落在该基调范围内
- **关系升温曲线 8 阶段**：若题材属于 Romance / Forbidden Love / Fake Marriage / Werewolf Mate 等强关系题材，关系网络中是否已为 8 阶段标注对应集数区间（参照 `references/world-character-scene.md` §关系发展8阶段模型）
- **被动状态字段（含失忆/中毒/穿越/系统流/装病/被附身类项目必查）**：角色档案 §1.5 是否填写了"被动状态"+"触发集 / 持续区间 / 解除条件"三件套；缺任意一件 = P1，避免后续随手解除被动状态破坏可信度

## 阻塞条件

- 核心设定自相矛盾
- 主配角戏剧功能重叠严重
- 主角或核心对手没有明确记忆点，角色卖点全靠后续剧情硬补
- 场景过散，无法低成本复用
- 命名体系混乱
- 出海模式下，仍保留中式家庭伦理压迫、宗族长幼秩序、体制单位权威或中文化礼俗作为核心戏剧驱动
- 含能力系统但六项任意一项为空（P0）
- 强关系题材未标注 8 阶段集数区间，或前 3 集核心世界规则只有"对白说明"没有"行动演示"

## 执行要求

1. 只读 `故事设定.md`、`剧本状态.json` 以及必要的上游文档；**不得**修改主流程产物（只追加/更新 `质检检查点.md`）。
2. 按本文件 §重点检查 + §阻塞条件逐项扫描；发现问题以统一 ID 落入 `质检检查点.md`，严重度标为 P0/P1/P2。
3. 若 `质检检查点.md` 尚不存在，当场创建（本命令是允许创建该文件的入口之一）。
4. 出现任意 P0 —— 写 `qcStatus.bible = 需修改`；**禁止**推进到 `/结构`。
5. 仅 P1/P2 —— 写 `qcStatus.bible = 已通过`，并将 `currentStep = 结构`，但在响应里列出"强建议修复项"。
6. 全部通过 —— 写 `qcStatus.bible = 已通过`，并将 `currentStep = 结构`。
7. 响应结构严格按 `quality-gate.md` §响应固定结构：结论 / 已更新文件 / 关键判断 / 专业角色会诊 / 质检结论 / 仍待确认 / 下一步。
8. `/设定质检` 本身**不**创建任何主流程产物（不建 `故事结构.md`、不建 `AI资产圣经.md`），只更新 `质检检查点.md` 与 `剧本状态.json` 的 `qcStatus.bible` / `currentStep` 字段。
