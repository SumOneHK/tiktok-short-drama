# /结构 命令详细执行规范

> 本文件是 `/结构` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

把项目做成可拆集、可卡点、可持续升级的故事引擎。

## 分层加载规则

**核心层（每次进入 `/结构` 必读）：**

业务数据文件：

- `故事设定.md` — 作为结构设计的事实基础
- `选题分析.md` — **只读"前 3 集承诺""主情绪""观众幻想合约"段**

规则参照文件：

- `references/viewer-payoff-contract.md` — 观看回报最高优先级、证据型故事和冷静型主角的价值函数
- `references/script-quality-engine.md`（只读 §二 三密度模型、§三 单集节奏锚、§四 钩子设计、§五 拉扯感）— 把结构设计从"有逻辑"校准到"能留人、能追更"
- `references/story-engine.md` — 全局结构、开篇、钩子、节奏、爽点设计原则
- `references/templates/story-structure.md`（`故事结构.md` 固定模板）

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 设计付费卡点结构 | `references/paywall-rules.md` |
| 设计全剧情绪曲线与单集情绪弧 | `references/emotion-design.md` |
| 初始化悬念债务与资产追踪表 | `references/continuity-ledger.md` |
| 高风险题材（未成年人 / 超自然暴力 / 高度政治敏感） | `references/compliance-rules.md` |
| `mode=overseas` 且结构层仍需复查平台适配 | `references/tiktok-platform-constraints.md`（只读节奏与卡点适配段） |
| 触发 `/结构质检` | `references/commands/qc-structure.md`（独立质检命令，闸门规则已下沉）+ `references/quality-gate.md` |

## 执行要求

0. **前置质检闸门（P0）**：进入 `/结构` 前必须确认 `qcStatus.bible = 已通过`；若为 `未检查` 或 `需修改`，先执行或修复 `/设定质检`，不得凭 `/设定` 内建自检直接推进。
1. `故事结构.md` 固定模板以 `references/templates/story-structure.md` 为唯一权威源，并至少覆盖：一句话故事线、核心冲突、观众幻想合约、**观看回报契约**、宏观结构、前 3 集引擎、情绪曲线、钩子系统、悬念债务与兑现、爽点结构、付费卡点、结局和崩盘风险。
2. 结构设计先服从 `references/viewer-payoff-contract.md` 的价值优先级：观看回报 > 戏剧冲突 > 可视化动作 > 信息推进 > 逻辑专业。不得用“设定完整 / 证据链完整 / 状态差量存在”替代观看回报。
3. 结构层必须先写清三密度主引擎：情绪密度靠什么留人、信息密度靠什么每集给新东西、情节密度靠什么承接升级。若只能写出事件列表，未写三密度主引擎，判 `P1`，前 3 集承诺段缺失则升 `P0`。
4. 前 3 集必须按 `script-quality-engine.md` §三/§八 校准：开头强情绪、15 秒局势变化、45 秒强期待、集尾追更目标；不得把世界观说明当作开篇引擎。
5. 细化原则以 `references/story-engine.md` 为准；卡点规则以 `references/paywall-rules.md` 为准（条件层触发）。
6. 每个卡点前必须已有阶段性回报；若中段只能重复施压、重复吵架、重复误会、重复证据提交或重复程序采纳，判 `P0`。
7. 法律/调查/证据型题材必须在结构阶段先写清“证据总账的情绪功能”：每份核心证据对应的旧伤、打脸对象、可见后果。若只能写出编号和专业名称，必须回退重设结构。
8. 首次进入 `/结构` 时，若 `故事结构.md` 不存在才创建；不要提前创建 `分集大纲.md`、`分集追踪.md` 或 `分集剧本/`。
9. 输出末尾追加"结构内建自检"；通过后只将 `qcStatus.architecture` 保持或回写为 `未检查`，`currentStep` 保持 `结构`，下一步必须执行 `/结构质检`。只有 `/结构质检` 写入 `qcStatus.architecture = 已通过` 后，才允许进入 `/分集大纲`。
