# /衔接质检 {起止集} 命令详细执行规范

> 本文件是 `/衔接质检 {起止集}` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

对相邻集之间的承接、连续性、情绪跳轨做独立闸门质检。不通过时先修相邻集，不得扩写后续。

## 分层加载规则

**核心层（每次进入 `/衔接质检` 必读）：**

- 本文件 §重点检查 + §阻塞条件（闸门规则已内嵌，**不需要再读 process-qc.md §三**）
- `references/state-schema.md` §`qcStatus.episodes` 字段约定
- `references/quality-gate.md` §响应固定结构
- `references/templates/qc-checkpoint.md`
- `references/continuity-ledger.md` §三 悬念债务维护 + §五 资产连续性维护（含事实账/关系账/资产账定义）

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 情绪跳轨嫌疑 | `references/emotion-design.md` |
| 本批次含关系翻页 | `references/world-character-scene.md` §关系发展8阶段模型 |
| 发现跨文档冲突 | `references/document-sync.md` |

## 重点检查

- 上一集尾钩是否在新集开场前 10%-20% 篇幅内被具体承接（不只是"提了一下"）
- 是否出现大段复述上一集已发生事件（复述超过本集前 20% 篇幅 = 节奏停滞）
- 时间线是否连续（相邻集之间的事件先后、跨度、白天黑夜是否能对上）
- 角色已知/未知信息差是否与上集一致（"上集她不知道，下集突然知道"必须有明确触发）
- 角色伤势、造型、道具、服装、站位是否从上集合理推进到本集（参照 `分集追踪.md` 资产账）
- 关系温度是否平滑过渡（参照 `world-character-scene.md` §关系发展8阶段模型，跨阶段必须有触发事件）
- 情绪是否突然跳轨（上集尾压抑、下集开场欢乐 = 必须有过渡或刻意对比的设计意图）
- 相邻集情绪曲线是否避免完全同质（连续两集都是高潮 / 都是低谷会造成观众疲劳）
- 悬念债务在相邻集之间是否被正确推进、升级或兑现（参照 `continuity-ledger.md` §三 悬念债务维护 5/8/12 集老化规则）
- 相邻集是否重复使用同一个钩子机制或同一种反转套路
- 本批次首尾两集与外部相邻集的衔接是否同步检查（不能只看批次内部）

## 阻塞条件

- 上一集尾钩在新集开场被忽略或被一句话敷衍带过
- 角色已知信息差与上集冲突且无明确触发事件
- 关系温度跳级（如从"陌生人"直接到"信任伴侣"无中间触发）
- 时间线矛盾或与 `分集追踪.md` 事实账冲突
- 悬念在本批次内被遗忘 / 被重复埋设 / 被提前泄底
- 相邻集复述超过本集前 20% 篇幅
- 角色伤势/道具状态在相邻集间凭空消失或恢复
- 连续 3 集情绪曲线完全同形（全高潮或全低谷）

## 执行要求

1. 只读本批次及前后各 1 集剧本、`分集追踪.md`、`剧本状态.json`；**不得**修改主流程产物。
2. 按本文件 §重点检查 + §阻塞条件逐项扫描。重点：上一集尾钩是否被接住、状态跳变是否有支撑、大段复述是否导致节奏停滞。
3. 若 `质检检查点.md` 不存在则当场创建。
4. 出现任意 P0 —— 写对应范围 `qcStatus.episodes[{range: 起止集, checkType: "continuity"}].status = 需修改`；**先修相邻集**，不得继续扩写。
5. 全部通过 —— 写 `qcStatus.episodes[{range: 起止集, checkType: "continuity"}].status = 已通过`，并在 `质检检查点.md` 记录"衔接已通过"标记。
6. 响应结构严格按 `quality-gate.md` §响应固定结构。
7. 本命令**不**创建任何主流程产物，只更新 `质检检查点.md` 与 `qcStatus.episodes`。
