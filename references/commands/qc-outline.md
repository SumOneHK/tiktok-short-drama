# /大纲质检 命令详细执行规范

## 重点检查

- 每集是否真的改变状态：检查 `分集大纲.md` 每集的 **本集状态变化** 三段式字段（起点状态 / 终点状态 / 状态差量）是否填齐；任一空白或差量为"无变化" = P0
- 状态差量是否落在身份/关系/信息/筹码/情绪至少一项发生不可逆变化；若仅是情绪起伏但无任何客观状态变化，视为水集，判 P1
- **信息差移动追踪（谜题/秘密/双重身份/失忆/反杀类题材必查）**：每集是否填写"本集信息差移动"五项（观众/主角/反派知识增量 + 信息差状态 + 关联秘密 ID）；连续 3 集"信息差状态 = 不变"且无新秘密引入 = P1，谜题节奏失速；连续 5 集 = P0；信息差状态字段缺失 = P1
- **双线/多线占比追踪（多线项目必查）**：含 A 线主线 + B 线副线/复仇线/谜题线的多线项目，每集是否填写"双线/多线占比"四项（A 线占比 / B 线占比 / 本集主导线 / 双线交织点）；连续 5 集"本集主导线 = A"且 B 线占比为 0 = P1，B 线悄悄失踪；A/B 线高潮撞同一集且无叠加设计 = P1
- 是否存在重复施压或空转
- 上下集是否承接
- 卡点是否有兑现路径
- 是否有明显水集（水集判定：同时满足以下3条以上——①无新增信息差 ②无状态变化 ③无新悬念抛出 ④无旧悬念兑现 ⑤与前一集冲突类型和结果高度相似）
- 是否有悬念挂起超过 8 集且无部分兑现（按 `continuity-ledger.md` 预警规则检查）
- 全剧爽点类型是否单一（同一类型爽点连续出现超过3次，视为 P1，需要引入新类型爽点）
- **直观打脸/回报检查**：每集 `本集爽点` 必须能写成“谁被谁打脸 / 谁失去什么 / 谁被治愈或获得什么”。只写“智性对抗、专业碾压、压抑揭穿、关系温度变化”但没有明确对象和可见后果 = P1；关键集/付费卡点这么写 = P0
- **证据闯关检查（法律/调查/商战题材必查）**：连续 3 集不得都以“发现/提交/采纳/归档 Exhibit/文件/申请”为主要推进。若同一段连续出现“一份证据一个庭审/一次提交一次采纳”的结构，判 P1；超过 5 集或出现在付费区间核心段 = P0，要求改成董事会、媒体、律所、预审动议、内部听证、反派办公室等多场合交替，并给证据获取设置阻力/代价
- **专业术语转译检查**：每个关键证据/法律动作必须在大纲里有人话后果，例如“这意味着 Alex 家族信托是假的 / Richard 个人责任上升 / 资产会被冻结”。只有法条、程序名、编号，没有观众秒懂后果 = P1
- 全剧尾钩类型是否单一：统计 `分集大纲.md` 中每集的 `尾钩类型` 字段，必须落在 10 类枚举内（信息钩/关系钩/结果钩/代价钩/身份钩/误会钩/威胁钩/选择钩/揭露钩/反杀钩，权威定义见 `references/templates/episode-outline.md`）；同一类型钩子连续出现超过3次，视为 P1；出现枚举外的自造类型 = P1
- **尾钩具象化检查**：尾钩不得只写 `Exhibit N`、`Phase N`、`Plan B`、`USB` 等符号。必须写清“里面是什么 / 谁会倒霉 / 为什么危险 / 下一集会兑现什么”。连续 2 次编号式弱钩子 = P1；付费卡点弱钩子 = P0
- 每个 💰 卡点后是否在大纲中能找到对应的"💰兑现"标记集；若卡点后3集内找不到兑现标记，视为 P1
- 主要世界规则是否在前3集对应大纲中有"行动演示场景"而非仅"背景说明"
- **关键集 6 节拍规划**：强卡点集、揭露集、高潮集、结局集是否都已完成节拍级（6 拍）规划而不是"大纲一句话"代替（参照 `references/story-engine.md` §关键集节拍设计 §6节拍模型）；任一关键集仅有一句大纲 = P1
- **角色弧线锚点标注**：核心角色成长弧线的锚点事件是否已落到具体集数区间内（不允许只在 `故事结构.md` 写"角色弧线"但 `分集大纲.md` 找不到锚点集）
- **双时间线碰撞检查**：双线项目每次过去线必须写清“过去伤害/误导/羞辱”与“现在线回击/代价”的铰链。连续 2 次过去线只做解释背景、不触发现在线即时回击 = P1；关键集这么写 = P0
- **情绪代入检查**：女主冷静型人设必须在大纲中安排外化代价（身体、动作、关系、选择、他人反应）。连续 3 集女主只“冷静推进证据/策略”且无情绪裂缝、失败或被他人看见代价 = P1
- **现实可信度检查**：现代现实题材中夸张生理、法律、商业表达必须降级到可信尺度；例如“12 天没睡”这类绝对表达在大纲阶段即判 P1，要求改成“12 天没睡整觉/每晚只睡两小时/12 天没在床上睡过”
- **大纲-剧本锁点准备**：每集大纲必须提供足够锁点供剧本落地：开场钩子、尾钩、被打脸者、可见后果、故事内时间、关键道具/人名。缺少这些字段会导致最终剧本与大纲只保留标题级对应，判 P1
- **观看回报四件套**：每集必须能回答“谁施压/被压迫、谁获得回报或付出代价、可截图动作是什么、下一集具体欲望是什么”。缺 2 项以上 = P1；关键集/付费集 = P0
- **三密度检查**：每集必须能说清情绪密度、信息密度、情节密度分别靠什么成立。任一项空泛（如只写“情绪强”“节奏快”“信息多”）= P1；第 1 集、关键集、付费集缺三密度 = P0
- **节奏锚检查**：第 1 集、关键集、付费卡点集、重大揭露集必须有开头冲击、15 秒变化、45 秒强期待和尾钩；普通集至少有开头钩子、中段变化和尾钩。关键集缺任一项 = P0；普通集缺中段变化 = P1
- **拉扯检查**：大爽点或强打脸前必须有压低、错误放松或再加压中的至少两项；若直接从受压跳到翻盘，爽点前情绪蓄力不足 = P1；付费卡点这么写 = P0
- **开场集专项**：第 1 集前 5 秒若仍在人物背景、世界观、环境慢描或前情说明，且没有施压/被压/危机动作 = P0

## 阻塞条件

- 连续 2-3 集状态不变
- 同类桥段机械重复
- 尾钩经常不被下一集承接
- 连续多集只有证据/程序推进，没有直观打脸、可见代价或关系翻页
- 付费卡点集中但不兑现
- 存在挂起超过 12 集的悬念且未处理
- 全剧前半段（前40%集数）仅有一种爽点类型且无升级
- 关键集（强卡点/揭露/高潮/结局）出现纯一句话大纲，无 6 节拍规划
- 现实题材核心程序逻辑严重失真，例如“一份证据开一次庭且无时间/程序解释”，并被作为连续主结构使用
- 第 1 集开场没有冲突动作，仍以背景说明或世界观介绍为主
- 关键集/付费卡点集没有节奏锚、三密度或有效拉扯，只靠标题级事件推进

## 执行要求

1. 只读 `分集大纲.md`、`分集追踪.md`、`故事结构.md`、`剧本状态.json`；**不得**修改主流程产物。
2. 按本文件 §重点检查 + §阻塞条件逐项扫描。特别关注：每集状态变化、水集判定、尾钩类型多样性、卡点兑现标记、悬念债务 aging、前 3 集世界规则行动演示。
3. 若 `质检检查点.md` 不存在则当场创建。
4. 出现任意 P0（连续 2-3 集状态不变 / 悬念挂 >12 集 / 同类桥段机械重复）—— 写 `qcStatus.outline = 需修改`；**禁止**推进到 `/分集剧本`。
5. 全部通过 —— 写 `qcStatus.outline = 已通过`，并将 `currentStep = 分集剧本`。
6. 长剧模式下按 `process-qc.md` §四「长剧质检分级」标注 A/B/C 级。
7. 响应结构严格按 `quality-gate.md` §响应固定结构。
8. 本命令**不**创建任何主流程产物，只更新 `质检检查点.md` 与 `qcStatus.outline` / `currentStep`。
