# /大纲质检 命令详细执行规范

> 本文件是 `/大纲质检` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

对 `分集大纲.md` 做独立闸门质检，通过后才允许进入 `/分集剧本`。

## 分层加载规则

**核心层（每次进入 `/大纲质检` 必读）：**

- 本文件 §重点检查 + §阻塞条件（闸门规则已内嵌，**不需要再读 process-qc.md §三**）
- `references/state-schema.md` §状态同步规则 —— `/大纲质检` 回写 `qcStatus.outline` 字段的规则
- `references/quality-gate.md` §响应固定结构
- `references/templates/qc-checkpoint.md`
- `references/continuity-ledger.md` §三 悬念债务维护 —— 判断 5/8/12 集预警

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 水集/调剂集判定存疑 | `references/emotion-design.md` §九 情绪调剂集设计 |

## 重点检查

- 每集是否真的改变状态：检查 `分集大纲.md` 每集的 **本集状态变化** 三段式字段（起点状态 / 终点状态 / 状态差量）是否填齐；任一空白或差量为"无变化" = P0
- 状态差量是否落在身份/关系/信息/筹码/情绪至少一项发生不可逆变化；若仅是情绪起伏但无任何客观状态变化，视为水集，判 P1
- **信息差移动追踪（谜题/秘密/双重身份/失忆/反杀类题材必查）**：每集是否填写"本集信息差移动"五项（观众/主角/反派知识增量 + 信息差状态 + 关联秘密 ID）；连续 3 集"信息差状态 = 不变"且无新秘密引入 = P1，谜题节奏失速；连续 5 集 = P0；信息差状态字段缺失 = P1
- **双线/多线占比追踪（多线项目必查）**：含 A 线主线 + B 线副线/复仇线/谜题线的多线项目，每集是否填写"双线/多线占比"四项（A 线占比 / B 线占比 / 本集主导线 / 双线交织点）；连续 5 集"本集主导线 = A"且 B 线占比为 0 = P1，B 线悄悄失踪；A/B 线高潮撞同一集且无叠加设计 = P1
- 是否存在重复施压或空转
- 上下集是否承接
- 卡点是否有兑现路径
- 是否有明显水集（水集判定：同时满足以下3条以上——①无新增信息差 ②无状态变化 ③无新悬念抛出 ④无旧悬念兑现 ⑤与前一集冲突类型和结果高度相似）
- 是否有悬念挂起超过 8 集且无部分兑现（按 `continuity-ledger.md` 预警规则检查）
- 全剧爽点类型是否单一（同一类型爽点连续出现超过3次，视为 P1，需要引入新类型爽点）
- 全剧尾钩类型是否单一：统计 `分集大纲.md` 中每集的 `尾钩类型` 字段，必须落在 10 类枚举内（信息钩/关系钩/结果钩/代价钩/身份钩/误会钩/威胁钩/选择钩/揭露钩/反杀钩，权威定义见 `references/templates/episode-outline.md`）；同一类型钩子连续出现超过3次，视为 P1；出现枚举外的自造类型 = P1
- 每个 💰 卡点后是否在大纲中能找到对应的"💰兑现"标记集；若卡点后3集内找不到兑现标记，视为 P1
- 主要世界规则是否在前3集对应大纲中有"行动演示场景"而非仅"背景说明"
- **关键集 6 节拍规划**：强卡点集、揭露集、高潮集、结局集是否都已完成节拍级（6 拍）规划而不是"大纲一句话"代替（参照 `references/story-engine.md` §关键集节拍设计 §6节拍模型）；任一关键集仅有一句大纲 = P1
- **角色弧线锚点标注**：核心角色成长弧线的锚点事件是否已落到具体集数区间内（不允许只在 `故事结构.md` 写"角色弧线"但 `分集大纲.md` 找不到锚点集）

## 阻塞条件

- 连续 2-3 集状态不变
- 同类桥段机械重复
- 尾钩经常不被下一集承接
- 付费卡点集中但不兑现
- 存在挂起超过 12 集的悬念且未处理
- 全剧前半段（前40%集数）仅有一种爽点类型且无升级
- 关键集（强卡点/揭露/高潮/结局）出现纯一句话大纲，无 6 节拍规划

## 执行要求

1. 只读 `分集大纲.md`、`分集追踪.md`、`故事结构.md`、`剧本状态.json`；**不得**修改主流程产物。
2. 按本文件 §重点检查 + §阻塞条件逐项扫描。特别关注：每集状态变化、水集判定、尾钩类型多样性、卡点兑现标记、悬念债务 aging、前 3 集世界规则行动演示。
3. 若 `质检检查点.md` 不存在则当场创建。
4. 出现任意 P0（连续 2-3 集状态不变 / 悬念挂 >12 集 / 同类桥段机械重复）—— 写 `qcStatus.outline = 需修改`；**禁止**推进到 `/分集剧本`。
5. 全部通过 —— 写 `qcStatus.outline = 已通过`。
6. 长剧模式下按 `process-qc.md` §四「长剧质检分级」标注 A/B/C 级。
7. 响应结构严格按 `quality-gate.md` §响应固定结构。
8. 本命令**不**创建任何主流程产物，只更新 `质检检查点.md` 与 `qcStatus.outline`。
