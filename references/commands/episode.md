# /分集剧本 命令详细执行规范

> 本文件是 `/分集剧本 {起止集}` 命令的自包含说明书。进入命令后先读本文件，再按本文件的分层加载规则读取必要引用；不需要再回 `SKILL.md` 查加载规则。

## 用途

按批次落地单集剧本初稿，并同步修订追踪与质检文档。

## 分层加载规则

**核心层（每次进入 `/分集剧本` 必读，共 7 个）：**

- `references/workspace-projects.md` — 解析 `ACTIVE_DRAMA_DIR`；本命令所有业务文件都从该目录读写
- `references/viewer-payoff-contract.md` — 单集观看回报、专业信息转译、冷静型主角情绪外化、双时间线碰撞
- `references/reveal-design.md` — 藏线写法、发现证据、藏而不糊边界
- `references/script-quality-engine.md` — 三密度、3/15/45 节奏、钩子、拉扯、主角反差/出场、展示式对白和画面感
- `references/episode-writing-protocol.md` — 写前检查、A/B 组 14 问、C 组观众体验 7 项、D 组密度与表现 5 项、名字/台词/逻辑/衔接/回填规则
- `references/episode-script-templates.md` — 统一分集剧本模板(纯叙事中间稿，不含镜头字段)
- `references/continuity-ledger.md` — 当前悬念债务和连续性状态(**只读当前追踪状态，不重读全文**)

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| `languageMode=overseas-en-dialogue` | `references/language-mode.md` |
| 已存在 `AI资产圣经.md` 且本批次涉及角色/场景/道具叙事事实 | `AI资产圣经.md`（**只读叙事事实层，不读 prompt 层**） |
| 本批次含关键集（强卡点/揭露/大爽点/结局） | `references/story-engine.md`（**只读 `关键集节拍设计` 章节**） |
| 本批次含关键集或情绪曲线标注"高波动" | `references/emotion-design.md`（**只读 §单集情绪弧线 + §尾钩情绪设计**），用于预标场景级情绪目标（起/峰/收） |
| 含超自然/奇幻能力场景 | `故事设定.md`（**只读 `能力边界` 表**），权威框架见 `references/world-character-scene.md` §能力边界设计框架 |
| **合规前置三次筛（强制）**—— 本批次含暴力/性/危险行为/仇恨/未成年人等敏感内容 | `references/compliance-rules.md` §通用红线 + §常用安全改写法 |
| 写完批次执行 `/剧本质检` 或 `/衔接质检` | `references/commands/qc-episode.md` 或 `references/commands/qc-link.md`（独立质检命令）+ `references/quality-gate.md` |
| 发现文档冲突需联动修订 | `references/document-sync.md` |

## 执行要求

0. **前置质检闸门（P0）**：进入 `/分集剧本 {起止集}` 前必须确认 `qcStatus.outline = 已通过`；若为 `未检查` 或 `需修改`，先执行或修复 `/大纲质检`，不得凭 `/分集大纲` 内建自检直接写剧本。
1. **分集剧本定位**：分集剧本是基于情节点大纲落地的**剧本初稿 / 纯叙事中间稿**（人、戏、情绪、尾钩），不是终稿制作稿。默认不写场次级时长预算；若用户明确要求剧本阶段给时长，只允许写场次级具体数值/区间和全集外置算术验证。**严禁**在分集剧本里写镜头号、镜头级时长闭合/求和、`AI画面提示`/`AI动作提示`、机位/运镜/景别等镜头语言——这些一律推到 `/分镜脚本 {起止集}` 阶段。若历史文件里已写入上述字段，按最小修改集移除。
2. 单集权威模板只认 `references/episode-script-templates.md`；不要在主 `SKILL.md` 自造简化模板。
3. 写每一集前，按 `references/episode-writing-protocol.md` 建立最小上下文包；若当前集承担旧悬念兑现，还要回看首次埋设集。
4. **角色台词声线卡前置闸门（P0）**：进入本命令前，`故事设定.md` §1.10 必须为本批次涉及的核心角色补齐"句式偏好 / 词汇禁区 / 情绪表达方式 / 信息处理方式"四字段；若缺失，先回到 `/设定` 或在写剧本前按最小修改集补齐 §1.10，并把 `qcStatus.bible` 回退为 `需修改` 等待 `/设定质检` 复核。不得在声线卡缺失时先写对白、再把问题推给 `/剧本质检`。
5. **文件头部硬规范（P0，两项缺一不可）**：每集 `分集剧本/第NNN集.md` 文件头部必须按下列顺序落盘：
   - **① YAML frontmatter**（版本追踪锚点，首次落盘 `version: 1`；任何编辑——包括 `/分镜脚本` 白名单回写——都必须 `+1` 并更新 `updated_at`，否则 `/总检` 断点对齐检查判 P0）：
     ```markdown
     ---
     episode: {NNN}
     version: 1
     updated_at: YYYY-MM-DD
     ---
     ```
  - **② 写前检查 HTML 注释**（`references/episode-writing-protocol.md` §二 是 26 问完整文本的唯一权威源；本文件只维护结构壳，紧跟 frontmatter 之后）：
     ```markdown
     <!-- 写前检查答案（问题全文见 references/episode-writing-protocol.md §二，必须逐项回答 1-26）
     A组-连接8项（1-8）:
     1. ...
     ...
     8. ...
     B组-高质量自检6项（9-14）:
     9. ...
     ...
     14. ...
     C组-短剧观众体验自检（15-21）:
     15. ...
     ...
     21. ...
     D组-密度与表现自检（22-26）:
     22. ...
     ...
     26. ...
     -->
     ```
   frontmatter 缺失、A/B 组 14 问任一问留空或整块缺失 = **P0**，不得落盘本集剧本。C 组整块缺失 = **P0**；C 组任一问留空或虚化 = **P1**，若本集是关键集/付费卡点/法庭证据推进集则升级为 **P0**。D 组整块缺失 = **P1**，若本集是第 1 集、关键集、付费卡点或重大揭露集则升级为 **P0**；D 组任一问虚化 = **P1**。这不是"写作建议"，是落盘前置条件。LLM 跳过写前检查直接写剧本 = self-reward hacking 零分。
6. 若 `AI资产圣经.md` 已存在（说明某批次已跑过 `/分镜脚本`），先读取其中的全局角色/场景/道具**叙事事实**作为人物和场景连续性参考；若不存在（首轮剧本生产的正常状态），**不创建、不降级创建最小版本**，以 `故事设定.md` 作为叙事事实唯一来源继续写。`AI资产圣经.md` 首次创建时机**统一收敛到** `/分镜脚本` 阶段，分集剧本阶段**不再要求**逐集维护 AI prompt 增量附录。
7. 单集末尾的"本集调用资产清单"只登记叙事事实级信息（本集出场的角色/场景/道具 + 本集新增/变化的长期事实），**不写**任何 AI 画面提示、风格关键词、镜头 prompt 或负面约束。
8. 海外模式下，仅对白、旁白、`OS`、`VO`、屏幕字卡允许使用英文；其余说明用中文。国内模式全中文。
9. 海外模式下，凡英文对白、旁白、`OS`、`VO`、屏幕字卡，必须在原文下方或同一条目内紧跟中文翻译，不得只给英文不给中译。
10. 若 `故事设定.md`、`分集大纲.md`、`分集追踪.md` 冲突，先修源文档，再写新剧本。
11. 不得随意新增常驻角色、改名字或改称谓；若必须新增，先更新 `故事设定.md` 与 `分集追踪.md`，并解释原因。
12. 每集开写前必须先按 `references/viewer-payoff-contract.md` 判断：本集观看回报是否成立，戏剧冲突是否成立，可视化动作是否成立。三者不成立时，不得只靠“状态差量/信息增量/证据推进”落盘正文。
13. 每集开写前必须按 `references/script-quality-engine.md` 判断：三密度是否成立、节奏锚是否成立、钩子是否真正改写局势、爽点前是否有压低/错误放松/再加压、关键对白是否可改成动作或反应。答不清时先改大纲或节拍，不得靠正文硬凑。
14. 触及藏线的集数，必须把“暗藏信息”写成可见证据（动作、道具、反应、系统字卡、场景异常），并保证本集显性危机和状态变化清楚。不得用解释台词替代发现证据；不得为了藏线让观众看不懂本集发生了什么。
15. 每场都必须先想清四件事再写：这场承接什么、推进什么、改变什么、遗留什么；答不清就不要保留该场。
16. 每场都必须锁定事实锚点，包括人物已知/未知信息、伤势/道具/站位状态、关系温度和本场在全局情绪曲线中的位置；剧本不能为了占篇幅新增无动作、无新信息、无状态变化的场景。
17. 单集篇幅目标为 60-90 秒（宏观参考）。剧本阶段默认**不写场次级时长预算**，只用"场次数量 + 每场叙事负载"控制节奏，60-90 秒项目推荐 2-4 场。若用户明确要求剧本阶段写场次时长，必须按 `references/episode-script-templates.md` 的"可选但给了就必须闭合"规则做外置算术验证；否则不要写任何"约 N 秒 / 分镜阶段再算"之类含糊话术。即使算术闭合，也要检查正文叙事负载：若一集只有文件交接、电话通知、独处沉默等低动作内容，不能用时长数字冒充 AI 可视化的 75 秒，必须加冲突/反应/动作转折或并入相邻集。
18. 每场必须承担情绪推进；开场前 10%-20% 篇幅内必须接住上一集尾钩，不能大段复述。
19. 证据/法庭/调查题材写作时，不能把“提交证据”当成默认高潮：每次 Exhibit 推进都要写明“证据内容的人话版本 + 被打脸者 + 见证反应 + 可见后果”。连续 3 集以同类证据提交作为主要爽点，必须回退大纲重排。
20. 双时间线项目写剧本时，过去线必须与现在线产生动作碰撞。过去线如果只解释背景，现在线必须补一个同源回击动作；否则该过去线并入设定或追踪，不保留为本集正文。
21. 角色实体锁必须执行：涉及亲属、墓碑、案名、旧文件、签字页、照片背面、信件称谓时，必须回查 `故事设定.md` 标准名/别名边界。若发现名字漂移，先修源文档并回退相关质检，不得继续写。
22. 每批写完后，回填 `分集大纲.md`、`分集追踪.md`；若本批次已触发 `/剧本质检` 或 `/衔接质检`，或 `质检检查点.md` 已存在，则同步更新该文件（首次创建时机以过程质检命令为准，`/分集剧本` 本身不预建该文件）。发现命名、关系、道具、伤势或逻辑问题时，按 `references/document-sync.md` 联动修订。
23. 每 3-5 集至少执行一次 `/剧本质检`；关键卡点集、重大揭露集和关系翻页集前后必须执行 `/衔接质检`。
24. 首次进入 `/分集剧本 {起止集}` 时，若 `分集剧本/` 不存在才创建；只创建本批次需要的集数文件，不预建后续空集。
25. 进入 AI 视频执行阶段的工作统一在 `/分镜脚本 {起止集}` 完成——不要在本命令里反复补镜头提示、prompt 或时长闭合。
26. 默认采用滚动生产：全剧级设定、大纲完成后，按批次写剧本并按批次转分镜；不要默认等全剧所有单集剧本全部完成后才开始拆分镜。
27. **剧本变更后的漂移回滚（强制）**：若本命令新写的是未质检集数，只按 `state-schema.md` 追加 `deliveryProgress.scriptCompletedRanges`，不代表通过；若修改已通过 `/剧本质检` 的集数，必须把对应 `qcStatus.episodes[{checkType:"script"}]` 回退为 `未检查` 或 `需修改`，并 bump YAML `version`。若这些集已通过 `/整剧通读`，必须把 `checkType="full-read"` 记录回退为 `未检查`；若已生成分镜，必须回退对应 `qcStatus.storyboards[*]`，并将 `qcStatus.production`、`qcStatus.compliance` 回退为 `未检查`。
28. **剧本反向同步上游（强制）**：若写剧本时改动了开场钩子、结尾钩子、付费卡点、关键锚点台词、状态差量最终结果、秘密兑现集、藏线发现证据/兑现集、角色弧线锚点或长期事实，必须同步更新 `分集大纲.md` / `分集追踪.md`；若这些字段属于 `故事结构.md` 或 `故事设定.md` 的共享字段，还必须同步回源并把 `qcStatus.outline`、`qcStatus.architecture` 或 `qcStatus.bible` 回退为 `需修改`，按管线顺序复检。
