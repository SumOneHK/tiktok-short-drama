# /分集剧本 命令详细执行规范

> 本文件是 `/分集剧本 {起止集}` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

按批次落地单集剧本，并同步修订追踪与质检文档。

## 分层加载规则

**核心层（每次进入 `/分集剧本` 必读，共 4 个）：**

- `references/viewer-payoff-contract.md` — 单集观看回报、专业信息转译、冷静型主角情绪外化、双时间线碰撞
- `references/episode-writing-protocol.md` — 写前检查、A/B 组 14 问、C 组观众体验 7 项、名字/台词/逻辑/衔接/回填规则
- `references/episode-script-templates.md` — 统一分集剧本模板(纯叙事中间稿，不含镜头字段)
- `references/continuity-ledger.md` — 当前悬念债务和连续性状态(**只读当前追踪状态，不重读全文**)

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| `languageMode=overseas-en-dialogue` | `references/language-mode.md` |
| 已存在 `AI资产圣经.md` 且本批次涉及角色/场景/道具叙事事实 | `AI资产圣经.md`（**只读叙事事实层，不读 prompt 层**） |
| 本批次含关键集（强卡点/揭露/大爽点/结局） | `references/story-engine.md`（**只读 `关键集节拍设计` 章节**） |
| 本批次含关键集或情绪曲线标注"高波动" | `references/emotion-design.md`（**只读 §单集情绪弧线 + §尾钩情绪设计**），用于预标场景级情绪目标（起/峰/收） |
| 含超自然/奇幻能力场景 | `故事设定.md`（**只读 `能力边界` 表**），权威框架见 `references/world-character-scene.md` §能力边界设计框架 |
| **合规前置三次筛（强制）**—— 本批次含暴力/性/危险行为/仇恨/未成年人等敏感内容 | `references/compliance-rules.md` §通用红线 + §常用安全改写法 |
| 写完批次执行 `/剧本质检` 或 `/衔接质检` | `references/commands/qc-episode.md` 或 `references/commands/qc-link.md`（独立质检命令）+ `references/quality-gate.md` |
| 发现文档冲突需联动修订 | `references/document-sync.md` |

## 执行要求

0. **前置质检闸门（P0）**：进入 `/分集剧本 {起止集}` 前必须确认 `qcStatus.outline = 已通过`；若为 `未检查` 或 `需修改`，先执行或修复 `/大纲质检`，不得凭 `/分集大纲` 内建自检直接写剧本。
1. **分集剧本定位**：分集剧本是**纯叙事中间稿**（人、戏、情绪、尾钩），不是制作稿。**严禁**在分集剧本里写镜头号、时长闭合/求和、场景时长预算、`AI画面提示`/`AI动作提示`、机位/运镜/景别等镜头语言——这些一律推到 `/分镜脚本 {起止集}` 阶段。若历史文件里已写入上述字段，按最小修改集移除。
2. 单集权威模板只认 `references/episode-script-templates.md`；不要在主 `SKILL.md` 自造简化模板。
3. 写每一集前，按 `references/episode-writing-protocol.md` 建立最小上下文包；若当前集承担旧悬念兑现，还要回看首次埋设集。
4. **角色台词声线卡前置闸门（P0）**：进入本命令前，`故事设定.md` §1.10 必须为本批次涉及的核心角色补齐"句式偏好 / 词汇禁区 / 情绪表达方式 / 信息处理方式"四字段；若缺失，先回到 `/设定` 或在写剧本前按最小修改集补齐 §1.10，并把 `qcStatus.bible` 回退为 `需修改` 等待 `/设定质检` 复核。不得在声线卡缺失时先写对白、再把问题推给 `/剧本质检`。
5. **文件头部硬规范（P0，两项缺一不可）**：每集 `分集剧本/第NNN集.md` 文件头部必须按下列顺序落盘：
   - **① YAML frontmatter**（版本追踪锚点，首次落盘 `version: 1`；任何编辑——包括 `/分镜脚本` 白名单回写——都必须 `+1` 并更新 `updated_at`，否则 `/总检` 断点对齐检查判 P0）：
     ```markdown
     ---
     episode: {NNN}
     version: 1
     updated_at: YYYY-MM-DD
     ---
     ```
   - **② 写前检查 HTML 注释**（`references/episode-writing-protocol.md` §二 的 A/B 组 14 问 + C 组 7 项完整答案，紧跟 frontmatter 之后）：
     ```markdown
     <!-- 写前检查答案 (episode-writing-protocol.md §二：A/B组14问 + C组7项)
     A组-连接8项:
     1. 上一集尾钩是什么 / 本集如何承接: ...
     2. 本集必须改变的最小状态是什么: ...
     3. 本集核心冲突点是什么: ...
     4. 本集哪些角色必须在场 / 为什么: ...
     5. 本集推动哪条悬念 / 挂起或兑现: ...
     6. 本集新增事实/道具/伤势锚点: ...
     7. 本集结束时角色关系温度: ...
     8. 本集尾钩是什么 / 下集如何承接: ...
     B组-高质量自检6项:
     9. 本集最后一句台词/最后一个动作是否必要，是否形成真正尾钩: ...
     10. 任意一场整段删掉后剧情是否仍能推进；若能，如何改成有差量事件: ...
     11. 关键事件是否有见证人/反应方: ...
     12. 观众最可能记住的一个画面/一句台词/一个动作是什么: ...
     13. 本集信息差移动：观众/主角/反派三方知识增量与收场信息距离: ...
     14. 时长外置算术闭合：若给了场次时长，写下限/上限/中位外置加法；若未给，明确标注"本集不给场次级时长预算，由分镜阶段全权分配": ...
     C组-短剧观众体验自检:
     15. 本集肉眼可见的冲突场是什么，谁主动施压、谁被迫反应: ...
     16. 本集可视化打脸/回报是什么，谁失去什么或被谁当场压住: ...
     17. 女主情绪如何外化，冷静是否有身体代价/动作裂缝/他人反应: ...
     18. 尾钩具体内容是什么，是否避免只用编号/代号做钩子: ...
     19. 若为双时间线，本集过去伤害如何触发现在线回击；若无双线写"不适用": ...
     20. 本集是否足够撑起目标时长，是否存在只有文件/电话/独处的短场: ...
     21. 是否存在无现实可信度的夸张表达或过量专业术语，如何转译/降级: ...
     -->
     ```
   frontmatter 缺失、A/B 组 14 问任一问留空或整块缺失 = **P0**，不得落盘本集剧本。C 组整块缺失 = **P0**；C 组任一问留空或虚化 = **P1**，若本集是关键集/付费卡点/法庭证据推进集则升级为 **P0**。这不是"写作建议"，是落盘前置条件。LLM 跳过写前检查直接写剧本 = self-reward hacking 零分。
6. 若 `AI资产圣经.md` 已存在（说明某批次已跑过 `/分镜脚本`），先读取其中的全局角色/场景/道具**叙事事实**作为人物和场景连续性参考；若不存在（首轮剧本生产的正常状态），**不创建、不降级创建最小版本**，以 `故事设定.md` 作为叙事事实唯一来源继续写。`AI资产圣经.md` 首次创建时机**统一收敛到** `/分镜脚本` 阶段，分集剧本阶段**不再要求**逐集维护 AI prompt 增量附录。
7. 单集末尾的"本集调用资产清单"只登记叙事事实级信息（本集出场的角色/场景/道具 + 本集新增/变化的长期事实），**不写**任何 AI 画面提示、风格关键词、镜头 prompt 或负面约束。
8. 海外模式下，仅对白、旁白、`OS`、`VO`、屏幕字卡允许使用英文；其余说明用中文。国内模式全中文。
9. 海外模式下，凡英文对白、旁白、`OS`、`VO`、屏幕字卡，必须在原文下方或同一条目内紧跟中文翻译，不得只给英文不给中译。
10. 若 `故事设定.md`、`分集大纲.md`、`分集追踪.md` 冲突，先修源文档，再写新剧本。
11. 不得随意新增常驻角色、改名字或改称谓；若必须新增，先更新 `故事设定.md` 与 `分集追踪.md`，并解释原因。
12. 每集开写前必须先按 `references/viewer-payoff-contract.md` 判断：本集观看回报是否成立，戏剧冲突是否成立，可视化动作是否成立。三者不成立时，不得只靠“状态差量/信息增量/证据推进”落盘正文。
13. 每场都必须先想清四件事再写：这场承接什么、推进什么、改变什么、遗留什么；答不清就不要保留该场。
14. 每场都必须锁定事实锚点，包括人物已知/未知信息、伤势/道具/站位状态、关系温度和本场在全局情绪曲线中的位置；剧本不能为了占篇幅新增无动作、无新信息、无状态变化的场景。
15. 单集篇幅目标为 60-90 秒（宏观参考）。剧本阶段默认**不写场次级时长预算**，只用"场次数量 + 每场叙事负载"控制节奏，60-90 秒项目推荐 2-4 场。若用户明确要求剧本阶段写场次时长，必须按 `references/episode-script-templates.md` 的"可选但给了就必须闭合"规则做外置算术验证；否则不要写任何"约 N 秒 / 分镜阶段再算"之类含糊话术。即使算术闭合，也要检查正文叙事负载：若一集只有文件交接、电话通知、独处沉默等低动作内容，不能用时长数字冒充可拍 75 秒，必须加冲突/反应/动作转折或并入相邻集。
16. 每场必须承担情绪推进；开场前 10%-20% 篇幅内必须接住上一集尾钩，不能大段复述。
17. 证据/法庭/调查题材写作时，不能把“提交证据”当成默认高潮：每次 Exhibit 推进都要写明“证据内容的人话版本 + 被打脸者 + 见证反应 + 可见后果”。连续 3 集以同类证据提交作为主要爽点，必须回退大纲重排。
18. 双时间线项目写剧本时，过去线必须与现在线产生动作碰撞。过去线如果只解释背景，现在线必须补一个同源回击动作；否则该过去线并入设定或追踪，不保留为本集正文。
19. 角色实体锁必须执行：涉及亲属、墓碑、案名、旧文件、签字页、照片背面、信件称谓时，必须回查 `故事设定.md` 标准名/别名边界。若发现名字漂移，先修源文档并回退相关质检，不得继续写。
20. 每批写完后，回填 `分集大纲.md`、`分集追踪.md`；若本批次已触发 `/剧本质检` 或 `/衔接质检`，或 `质检检查点.md` 已存在，则同步更新该文件（首次创建时机以过程质检命令为准，`/分集剧本` 本身不预建该文件）。发现命名、关系、道具、伤势或逻辑问题时，按 `references/document-sync.md` 联动修订。
21. 每 3-5 集至少执行一次 `/剧本质检`；关键卡点集、重大揭露集和关系翻页集前后必须执行 `/衔接质检`。
22. 首次进入 `/分集剧本 {起止集}` 时，若 `分集剧本/` 不存在才创建；只创建本批次需要的集数文件，不预建后续空集。
23. 进入 AI 视频执行阶段的工作统一在 `/分镜脚本 {起止集}` 完成——不要在本命令里反复补镜头提示、prompt 或时长闭合。
24. 默认采用滚动生产：全剧级设定、大纲完成后，按批次写剧本并按批次转分镜；不要默认等全剧所有单集剧本全部完成后才开始拆分镜。
