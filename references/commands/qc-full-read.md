# /整剧通读 命令详细执行规范

> 本文件是 `/整剧通读` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

在全剧分集剧本完成并通过批次 `/剧本质检` 后，从**编剧 / 导演 / 观众 / 平台商业 / 本地化可信度**等宏观视角通读全剧，发现逐批质检不容易暴露的整体体验问题。

本命令是**分镜前置门禁**，不替代 `/剧本质检 {起止集}`：

- `/剧本质检` 抓单批格式、承接、台词声线、场景字段、实体锁
- `/整剧通读` 抓整剧弧线、后半程疲软、反派重复、情绪不值、场面可拍性、观众追更欲、结局释放和商业卡点
- `/总检` 抓制作包收口、跨文档一致性、分镜执行性、AI 制作就绪和导出前阻塞项

## 触发时机

**强制触发：**

1. `deliveryProgress.scriptCompletedRanges` 已覆盖 `1..episodeCount`
2. `qcStatus.episodes` 中 `checkType=script` 且 `status=已通过` 的 range 已覆盖 `1..episodeCount`
3. 准备在全剧剧本完成后首次进入或继续 `/分镜脚本`

以上三项同时成立，但不存在覆盖全剧的 `checkType=full-read` 且 `status=已通过` 记录时，必须先执行 `/整剧通读`。

**滚动生产例外：**

- 全剧分集剧本尚未写完时，可继续按批次 `/分镜脚本 {起止集}`，不要求提前整剧通读
- 如果已经滚动产出部分分镜，等全剧剧本完成后仍必须执行 `/整剧通读`；若通读修源改动了已分镜集的源剧本，必须按 `source_episode_version` 规则重做对应分镜

## 检查视角

### 1. 编剧 / Showrunner 视角

- 主线是否从 EP001 到结局持续升级，而不是中后段只换文件名推进
- 主角弧线是否真正翻页：她的选择、行动方式、关系站位是否发生不可逆变化
- 反派压迫方式是否重复：是否一直靠威胁、邮件、文件、电话，而没有新的权力展示
- 悬念兑现是否有情绪价值：是否只给答案，没有让观众“爽 / 心疼 / 震惊 / 松一口气”
- 结局是否兑现开篇承诺：前 3 集抛出的核心看点是否在结尾有可见回收

### 2. 导演 / 画面视角

- 关键集是否有可拍的大场面、动作、站位、道具和群体反应
- 是否连续多集依赖电话、文件、电脑屏幕、独处解释
- 爽点是否能被竖屏镜头放大：打脸者、见证者、受益者、代价对象是否在场
- 场景池是否过度重复，是否需要在源剧本层补动作或公共空间反应
- 分镜阶段是否会缺画面抓手；若缺，必须回修源剧本动作行或场景事实锚点

### 3. 观众 / 用户视角

- 前 3 秒是否能让非项目内人员知道“谁被压、谁要反击、危险是什么”
- 每 3-5 集是否都有一个能记住的场面，而不是只有信息推进
- 低谷是否压得住但不劝退：代价是否具体，是否有继续看的希望
- 尾钩是否具象：下一集危险、倒霉对象、反转内容是否明确
- 结局是否让观众获得足够释放：主角赢在哪里、反派失去什么、被抹掉的人如何重新被看见

### 4. 平台 / 商业视角

- 付费点前后是否有强期待与强兑现，不靠“下一份文件”虚晃
- 9 集后是否持续存在追更理由，是否中段有掉速区
- 是否有适合投放剪辑的高价值片段：公开羞辱、反杀、身份揭露、关系翻页、最终清算
- 结尾 3 集是否有传播点和标题点，是否能剪出“必须看完”的短视频切片
- 是否存在可被前台运营误读的集数标题或卖点空洞

### 5. 本地化 / 可信度视角

- 海外版是否仍依赖中文社会语境、孝道/宗族/体制惯性或本土梗
- 校园、Greek life、donor board、conduct procedure、settlement 等是否像“戏剧化压缩”，而不是明显胡编
- 英文对白是否只是中文直译，关键威胁和爽点是否符合目标市场表达习惯
- 法律/行政动作是否被翻译成观众后果，避免术语堆砌

## 阻塞条件

任一命中均不得进入全剧完成后的 `/分镜脚本`：

- P0：结局没有兑现前 3 集核心承诺，或主角终局胜利不清楚
- P0：中后段连续 5 集以上主要靠文件/电话/程序递进，没有可视化爽点或人物关系翻页
- P0：主角、核心反派、核心同盟任一人的弧线终局与 `故事设定.md` 明显不一致
- P0：关键悬念没有完全兑现，或兑现集与 `分集追踪.md` 冲突
- P0：海外版核心冲突仍依赖目标市场难以理解的中文社会背景
- P0：通读发现需改动已通过剧本的核心剧情、尾钩、关键台词或长期事实，但未回退受影响批次 `/剧本质检`
- P1：存在 2-4 集体验低谷，逻辑完整但观众回报弱，需要补强场面或情绪释放
- P1：导演视角发现关键集画面抓手不足，分镜阶段会被迫凭空添加动作
- P1：付费卡点前后兑现弱，商业剪辑点不足
- P1：角色声线整体仍可读，但局部高价值场景里两名核心角色口吻接近

## 执行步骤

1. **前置覆盖核对**
   - 用 `剧本状态.json` 确认 `scriptCompletedRanges` 覆盖 `1..episodeCount`
   - 用 `qcStatus.episodes` 确认 `checkType=script` 且 `status=已通过` 覆盖 `1..episodeCount`
   - 若覆盖不完整，阻塞并提示先补 `/分集剧本` 或 `/剧本质检`

2. **全剧通读切片**
   - 按结构分段读：开局承诺段、付费启动段、中段升级段、低谷反噬段、终局清算段
   - 每段至少记录：核心冲突、最大观看回报、最弱集、最强尾钩、下一段承接
   - 不允许只读大纲或追踪表；必须抽读/精读对应 `分集剧本/*.md`

3. **五视角评估**
   - 分别从编剧、导演、观众、平台商业、本地化可信度写出问题
   - 每个问题必须落到具体集数、场次或台词，不接受“中段略弱”这种无锚点结论
   - 每个问题标 P0/P1/P2，并写清“必须修源 / 可分镜阶段放大 / 可接受风险”

4. **修源决策**
   - 影响剧情、尾钩、关键台词、角色弧线、长期事实的问题：必须修 `分集剧本/*.md`，并同步 `分集大纲.md` / `分集追踪.md`
   - 只影响镜头表现、节奏停留、视觉强调的问题：可标记为“分镜阶段放大”，但必须写入检查点，供 `/分镜脚本` 使用
   - 任何修源都必须 bump 对应剧本 `version`，并回退受影响 `qcStatus.episodes` 的 `checkType=script` 记录为 `未检查` 或 `需修改`

5. **复检闭环**
   - 修源后必须重跑受影响批次 `/剧本质检 {起止集}`；若只改单集，也至少执行该集的 `/剧本质检 {该集}`
   - 所有受影响批次重新通过后，再执行或更新 `/整剧通读`
   - `/整剧通读` 通过后，在 `qcStatus.episodes` 写入 `checkType=full-read`、`range=001-XXX`、`status=已通过`

状态回写：

- 通过：`lastQcStep = "/整剧通读"`，`qcStatus.episodes` 新增或更新 `{range:"001-XXX", checkType:"full-read", status:"已通过", blockingIssues:[], notes:"..."}`
- 不通过：`lastQcStep = "/整剧通读"`，`qcStatus.episodes` 新增或更新 `{range:"001-XXX", checkType:"full-read", status:"需修改", blockingIssues:[...], notes:"..."}`
- 修源后：受影响 `checkType=script` 批次必须重新通过，`full-read` 才能写回 `已通过`

## 推进规则

- `/整剧通读` 通过后，若全剧分集剧本已通过且存在剧本/分镜缺口，`currentStep` 按 `state-schema.md` §滚动生产语义通常进入 `分镜脚本`
- `/整剧通读` 未通过时，`currentStep` 保持或回退为 `分集剧本`
- 通过后不创建 `AI资产圣经.md`、`分镜脚本/`、`总检报告.md` 或 `合规报告.md`
