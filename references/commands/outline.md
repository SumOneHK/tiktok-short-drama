# /分集大纲 命令详细执行规范

> 本文件是 `/分集大纲` 命令的自包含说明书。进入命令后先读本文件，再按本文件的分层加载规则读取必要引用；不需要再回 `SKILL.md` 查加载规则。

## 用途

把结构引擎拆成全剧分集推进图，并同步维护 `分集追踪.md`。

## 分层加载规则

**核心层（每次进入 `/分集大纲` 必读）：**

工作区定位：

- `references/workspace-projects.md` — 解析 `ACTIVE_DRAMA_DIR`；本命令所有业务文件都从该目录读写

业务数据文件：

- `故事设定.md` — **只读角色表、场景池、能力边界**
- `故事结构.md` — 作为大纲拆解的事实基础

规则参照文件：

- `references/viewer-payoff-contract.md` — 每集观看回报四件套、尾钩从符号变欲望、戏剧锁点
- `references/reveal-design.md` — 逐集显点/藏点/发现证据/误读/兑现责任
- `references/script-quality-engine.md`（只读 §二 三密度模型、§三 单集节奏锚、§四 钩子设计、§五 拉扯感、§九 质检口径）— 拆集时校准密度、节奏、钩子和追更欲望
- `references/story-engine.md`（**只读 §按集数映射的结构骨架 与 §节奏设计 段**，对应 story-engine.md 顶部按需分章表的"拆集节拍与前 3 集引擎"用途）
- `references/templates/episode-outline.md`（`分集大纲.md` 固定模板）
- `references/templates/episode-tracker.md`（`分集追踪.md` 固定模板）

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 单集情绪弧与尾钩设计 | `references/emotion-design.md` |
| 卡点集标注与付费兑现设计 | `references/paywall-rules.md` |
| 初始化或维护 `分集追踪.md` 的悬念债务 | `references/continuity-ledger.md` |
| `mode=overseas` 且大纲需复查竖屏节奏 | `references/tiktok-platform-constraints.md`（只读节奏段） |
| 触发 `/大纲质检` | `references/commands/qc-outline.md`（独立质检命令，闸门规则已下沉）+ `references/quality-gate.md` |

## 执行要求

0. **前置质检闸门（P0）**：进入 `/分集大纲` 前必须确认 `qcStatus.architecture = 已通过`；若为 `未检查` 或 `需修改`，先执行或修复 `/结构质检`，不得凭 `/结构` 内建自检直接推进。
1. `分集大纲.md` 和 `分集追踪.md` 的固定模板分别以 `references/templates/episode-outline.md` 和 `references/templates/episode-tracker.md` 为唯一权威源。
2. 拆集时先按 `references/viewer-payoff-contract.md` 写每集的观看回报四件套：谁施压/被压迫、谁获得回报或付出代价、可截图动作、下一集具体欲望。四件套缺失，不得只用状态差量补位。
3. `分集大纲.md` 的本质是**逐集情节点大纲**：每集必须写清"本集情节点"（这一集发生的关键事件）和"情节点后果"（它如何改变状态、关系、信息差、筹码或风险）。如果一集只是一句剧情简介，没有可追踪的情节点和后果，视为大纲不成立。
4. 每集必须能说清三密度：情绪密度靠哪个刺激/释放，信息密度给哪个新有效信息，情节密度完成哪个承接-升级-不可逆变化-下集勾连。
5. 每集必须区分“显性推进”和“暗藏信息”：显性推进保证观众当下看懂本集危机与状态变化；暗藏信息只负责发现感。凡触及结构阶段藏线 ID 的集数，必须填写发现证据、当时误读、对应藏线 ID 和最迟小回报/兑现责任。
6. 关键集、付费卡点集、重大揭露集必须预标节奏锚：开头冲击、15 秒变化、45 秒强期待、结尾钩子；普通集至少写清开头钩子、中段变化和尾钩。
7. 覆盖全部集数；前 3 集完成题材承诺与主关系建立，前 8-12 集出现第一轮高价值兑现。
8. 每集至少有一个明确状态变化，且至少有一个可感知回报（打脸、心疼、震惊、危险、心动、反杀、关系翻页、代价落地）。`💰` 标记要有商业逻辑，不能把所有关键集都标成卡点。
9. 法律/调查/证据型题材拆集时，连续 3 集不得都以“证据发现/提交/采纳/归档”作为主要推进；必须交替安排公开打脸、关系翻页、反派反扑、证据代价、盟友分歧或现实惩罚。
10. 每完成一轮大纲更新，同步回填 `分集追踪.md` 中的新增悬念、兑现节点、藏线进度、关系温度、资产变化和**观看回报债务**（若本集只铺垫未回报，必须标明最迟兑现集）。
11. 若连续 2-3 集没有状态变化、尾钩经常不被承接、只有功能推进没有观看回报，或前 3 集未完成题材承诺，判 `P0`。
12. 首次进入 `/分集大纲` 时，若 `分集大纲.md` 或 `分集追踪.md` 不存在才创建；不要提前创建 `分集剧本/` 或导出目录。
13. 输出末尾追加"大纲内建自检"；通过后只将 `qcStatus.outline` 保持或回写为 `未检查`，`currentStep` 保持 `分集大纲`，下一步必须执行 `/大纲质检`。只有 `/大纲质检` 写入 `qcStatus.outline = 已通过` 后，才允许进入 `/分集剧本`。
14. **大纲变更后的漂移回滚（强制）**：若本命令修改了已通过质检的 `分集大纲.md` / `分集追踪.md` 中的共享字段（开场钩子、结尾钩子、状态差量最终结果、秘密兑现集、藏线兑现集、付费卡点位置/类型、角色弧线锚点、关键道具/伤势/身份事实），必须将受影响集数的 `qcStatus.episodes[*]` 回退为 `未检查` 或 `需修改`；若这些集已生成分镜，还必须回退对应 `qcStatus.storyboards[*]`，并将 `qcStatus.production`、`qcStatus.compliance` 回退为 `未检查`。若大纲修订反向要求改 `故事结构.md` 或 `故事设定.md` 的稳定事实，必须同步回源并分别把 `qcStatus.architecture` / `qcStatus.bible` 回退为 `需修改`，按管线顺序复检。
