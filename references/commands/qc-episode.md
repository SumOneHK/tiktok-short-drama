# /剧本质检 {起止集} 命令详细执行规范

> 本文件是 `/剧本质检 {起止集}` 命令的自包含说明书。进入命令后只读本文件即可。

## 用途

对已完成的分集剧本批次做独立闸门质检，通过后才允许写后续批次或推进 `/分镜脚本`。

## 分层加载规则

**核心层（每次进入 `/剧本质检` 必读）：**

- 本文件 §重点检查 + §阻塞条件（闸门规则已内嵌，**不需要再读 process-qc.md §三**）
- `references/state-schema.md` §`qcStatus.episodes` 字段约定
- `references/quality-gate.md` §响应固定结构
- `references/templates/qc-checkpoint.md`
- `references/continuity-ledger.md` §三 悬念债务维护 + §五 资产连续性维护
- `references/episode-script-templates.md` §单集格式总规范 + §5. 格式控制原则 —— 反向检查镜头字段

**条件层（仅在满足触发条件时读取）：**

| 触发条件 | 读取文件 |
|---------|---------|
| 本批次含关键集（强卡点/揭露/大爽点/结局） | `references/story-engine.md` §关键集节拍设计 §6节拍模型 |
| 奇幻/超自然题材 | `references/world-character-scene.md` §能力边界设计框架 |

## 重点检查

> 本阶段只检查"叙事中间稿"层面的质量，不检查镜头级字段和秒级时长闭合。镜头级问题一律推迟到 `/分镜质检`。

- 名字、称谓是否统一
- 台词是否符合当前人设
- 是否乱加功能性角色
- 相邻集是否承接
- 因果和代价是否成立
- 单集场次数量是否匹配 60-90 秒项目的宏观篇幅参考（推荐 2-5 场）
- 每场是否都能说清承接点、冲突目标、事实锚点、**场景状态差量**和情绪位置（场景状态差量字段权威定义见 `references/episode-script-templates.md` §模板字段说明）
- **场景状态差量逐场扫描**：每场必须有"开场状态 → 收场状态"的具体变化；写"无变化"或留空 = 灌水场，判 P1，要求合并或删除
- **写前 13 问留痕（强制 P0）**：每集剧本文件头部**必须**包含 `references/episode-writing-protocol.md` §二的 13 问完整答案 HTML 注释（格式见 `references/commands/episode.md` §执行要求 第 4 条）。注释缺失、残缺或任一问留空 = **P0**，直接阻塞。注释完整后抽查 B 组（问题 9-13）是否真的对应到剧本结构；若 B 组答案虚化且本批次出现"灌水场 / 无尾钩拉力 / 无记忆点 / 信息差停滞" = P1
- 是否存在只为占篇幅而保留的解释场、空转场、重复反应场
- **反向检查**：是否误写了分镜级字段（镜头号、时长求和、`AI画面提示`、`AI动作提示`、机位/运镜/景别等）
- **本集调用资产清单**是否已登记本集出场的角色/场景/道具，以及本集新增/变化的长期事实
- 本批次中每个出场的核心角色，其"主要戏剧功能"是否被实际调用
- 检查 `分集追踪.md` 悬念债务表：当前挂起超过 8 集且无部分兑现的悬念 = P1，需在后续 3 集内安排部分兑现；超过 12 集未处理升为 P0
- 本批次中角色重大决定是否与 `故事设定.md` 角色档案的隐性伤口/情绪触发器/失控点一致
- 奇幻/超自然题材：涉及超常能力的场景是否均符合 `故事设定.md` 能力边界表的"正常上限/触发条件/触发代价"
- 核心角色成长弧线锚点事件是否按时发生或正在推进
- **台词个性化（clean-context 盲测，强制）**：通过 Task tool 派发 `general-purpose` 子 agent 做独立盲测判别，详见本文件 §盲测执行规范。**严禁主 agent 上下文内自测**——写剧本的 LLM 自测必然宽松，零信号价值。
- 本批次是否至少给核心角色留下可被观众记住的动作、表情、场面或关系翻页点
- 本集调用资产清单中出现的角色/场景/道具是否都能在 `故事设定.md` 找到对应记录（`AI资产圣经.md` 首次创建延后到 `/分镜脚本`）

## 阻塞条件

- 角色口吻失真
- 相邻集断裂
- 情节强推
- 剧本中误写镜头号、时长求和、AI 画面/动作提示、机位/运镜/景别等分镜级字段（P0）
- **13 问留痕缺失**：剧本文件头未包含完整 13 问答案 HTML 注释，或任一问留空 / 整块缺失（P0）
- **YAML frontmatter 缺失**：剧本文件头未包含 `episode / version / updated_at` 三字段 frontmatter，或 `version` 为 0 / 非整数（P0）
- 场景只是为了占篇幅存在
- 关键事实、冲突或尾钩被埋在低效场景里，本集结构失焦
- 新角色冗余且未登记
- 核心角色连续 5 集以上未调用其主要戏剧功能
- 悬念挂起超过 12 集且无任何部分兑现
- 角色重大决定存在明显动机断层且无前期铺垫
- 奇幻题材出现无代价突破能力上限的开挂
- 连续一个批次只有功能推进，没有任何角色高光或关系记忆点
- 本集调用资产清单缺失，或与 `故事设定.md` 长期不一致
- **台词个性化 clean-context 盲测准确率 <80%**（P0；主 agent 自测结果一律不计数，必须走 Task tool 子 agent）

## 盲测执行规范（clean-context 子 agent）

> 本命令的核心独立信号是 clean-context 盲测——由一次全新上下文的子 agent 做"遮名可辨"判别。**原 `episode-writing-protocol.md` §五的"每场写完自检"仍保留作为写作阶段参考**，但进入 `/剧本质检` 的**权威判定**以本规范为准，不以主 agent 自测为准。写剧本的 LLM 自测必然偏袒自己写的台词，信号不可用。

### 为什么必须 clean-context

- 主 agent 当前上下文已携带：本集大纲、状态差量、写前 13 问、创作意图——这些信息会让它"脑补"出每句台词属于谁，而非真的凭声音判别
- clean-context 子 agent 没有任何创作意图包袱，只能凭角色声音档案和台词本身匹配——这才是观众实际的判别条件
- 独立信号是打破 self-reward hacking 的唯一手段

### 执行步骤

1. **抽样**（由主 agent 完成）
   - 从本批次每一集抽取 2 场主要对话：含 ≥2 个核心角色、≥6 轮对白
   - 若单集主要对话不足 2 场，全部取
   - **优先抽**：冲突场、情绪峰值场、关系翻页场；**禁止**避开高密度对话场
   - 单次盲测目标：覆盖本批次全部核心角色至少一次

2. **构造盲测输入包**（由主 agent 完成，严格遵守"最小信息"原则）
   - **声音档案**：从 `故事设定.md` 抽出本批次涉及的核心角色"说话声音档案"四字段（句式偏好 / 词汇禁区 / 情绪表达方式 / 信息处理方式），每个角色不超过 150 字
   - **匿名对白**：把抽样场次的所有角色名与称谓替换为 `角色A / 角色B / 角色C...`，保留动作行（`△`）但去掉任何身份泄露词（如"总裁""医生"等标签称谓也要匿名化）
   - **禁止携带**：本集大纲、状态差量、写前 13 问、创作意图、情绪曲线、前情提要、尾钩——子 agent **只能**看到声音档案 + 匿名对白
   - **映射表**：主 agent 保留一份 `角色A → 真实角色名` 的映射表本地留底，供评分用，**不**发给子 agent

3. **派发子 agent**（使用 Task tool，`subagent_type: general-purpose`）

   Prompt 模板：
   ```
   你是独立戏剧评审员。任务：凭角色声音档案判别匿名对白的说话人。

   【规则】
   - 只凭"声音档案四字段"和"台词本身"判断，不要推理剧情
   - 每一句对白给出你认为最匹配的匿名标签（角色A/B/C...）
   - 每条判断必须说明依据：声音档案的哪条字段支撑
   - 不要假设任何未提供的信息，不要脑补上下文

   【声音档案】
   {四字段表}

   【待判别对白】
   {匿名化对白，每句前有编号}

   【输出格式】
   JSON 数组：[{line_id, predicted_speaker, evidence_field, confidence(0-1)}, ...]
   ```

4. **评分**（主 agent 用本地映射表对比）
   - 分母 = 子 agent 实际判别的对白总句数
   - 分子 = `predicted_speaker` 与真实角色匹配的句数
   - **准确率 < 80%** → P0 阻塞，判定本批次存在"同质化台词"，写 `qcStatus.episodes[{起止集}] = 需修改`
   - **80% ≤ 准确率 < 90%** → P1，列出子 agent 判错的具体场次和句子，要求在本批次内修正后重跑盲测
   - **准确率 ≥ 90%** → 台词个性化检查通过

5. **留痕**（写入 `质检检查点.md`）
   - 抽样场次清单
   - 声音档案片段（原样复制）
   - 子 agent 完整回复（JSON）
   - 映射表与命中率
   - 被判错的对白行号与修复建议
   - **禁止**只写"已做盲测"或"盲测通过"——无证据等同未做

### 禁止清单（违反即判 P0）

- 主 agent 自己在当前上下文内"盖住角色名"声称能分辨——零信号价值
- 把完整 `故事设定.md` 或角色档案全文塞给子 agent（超出四字段就是信息泄露）
- 把本集大纲/状态差量/尾钩/写前 13 问塞给子 agent（剧情脑补污染）
- 抽样时避开冲突场、情绪峰值场，只抽平淡对话凑数
- 用一次抽样覆盖不到本批次全部核心角色（核心角色缺席 = 盲测无效）

### 与 `episode-writing-protocol.md` §五的关系

- `episode-writing-protocol.md` §五定义了**写作阶段**的台词个性化要求（声音档案四字段 + 每场写完自检 3 问）
- 本规范定义了**质检阶段**的独立评估机制（clean-context 子 agent 盲测）
- 两者不冲突：前者是"写前看"，后者是"写后验"。质检阶段的权威判定以本规范盲测结果为准。

## 执行要求

1. 只读本批次 `分集剧本/第NNN集.md`、`分集追踪.md`、`故事设定.md` 中必要段、`剧本状态.json`；**不得**修改主流程产物。
2. 按本文件 §重点检查 + §阻塞条件逐项扫描。**重点关注**：
   - **反向检查**（P0）：是否误写了镜头号、时长求和、AI 画面/动作提示、机位/运镜/景别
   - 本集调用资产清单完整性
   - 角色决定动机 vs 角色档案隐性伤口/情绪触发器一致性
   - 能力边界六项是否被违反
3. **必跑动作：台词个性化 clean-context 盲测**（独立步骤，不得省略）
   - 严格按本文件 §盲测执行规范 5 步执行
   - 使用 Task tool 派发 `general-purpose` 子 agent，**禁止**主 agent 自测
   - 子 agent 准确率 < 80% = P0 阻塞；80%-90% = P1 本批次修正；≥90% 通过
   - 盲测完整证据（输入包 + 子 agent 回复 + 命中率）必须写入 `质检检查点.md`，不留证据视同未做 = P0
4. 若 `质检检查点.md` 不存在则当场创建。
5. 出现任意 P0 —— 写 `qcStatus.episodes[{起止集}] = 需修改`；**禁止**写后续批次或推进 `/分镜脚本`。
6. 全部通过 —— 写 `qcStatus.episodes[{起止集}] = 已通过`。
7. 长剧模式下按 `process-qc.md` §四 批次质检规则 → 长剧（40集以上）质检分级 标注 A/B/C 级。
8. 响应结构严格按 `quality-gate.md` §响应固定结构。
9. 本命令**不**创建任何主流程产物（不建 `AI资产圣经.md`、不建分镜），只更新 `质检检查点.md` 与 `qcStatus.episodes`。
