## 使用规则

本 skill 的最终交付一律是 60-90 秒竖屏 AI 视频，由 AI 视频模型（可灵 / Sora / Veo / Runway / 海螺 等）按**分镜脚本**生成成片。**分集剧本只是叙事中间稿，不是制作稿**——以下四件事全部交给 `/分镜脚本` 阶段处理，**不得**在分集剧本里出现：

- ❌ 镜头号、镜头求和、镜头级时长闭合、未经用户要求的场次级时长预算
- ❌ `AI画面提示`、`AI动作提示`、镜头级 prompt 关键词
- ❌ 机位/角度/运镜等镜头语言字段
- ❌ 连续性锁定词、负面约束等 AI 生成约束

分集剧本只回答一件事：**"这一集作为一段可读的戏，人怎么走、戏怎么推、情绪怎么翻、尾钩怎么切。"** 其他一切都推到分镜阶段。除非用户明确要求剧本阶段给场次时长，否则本模板默认不写任何场次级时长预算。

单集配置规则：

- 艺术风格（`artStyle`）在 `/设定` 阶段由 `故事设定.md` 预锁，并在首次 `/分镜脚本` 创建 `AI资产圣经.md` 时继承为全局生成规则；剧本模板里**不出现**任何风格关键词或 prompt 片段
- 海外模式下，模板内只有对白、旁白、`OS`、`VO`、字卡允许使用英文，其余说明仍用中文
- 海外模式下，凡英文对白、旁白、`OS`、`VO`、字卡，必须紧跟中文翻译；推荐下一行使用 `中译：...`

## 单集格式总规范

### 1. 场次头信息

每场都必须先写清：

- 场景序号：`集数-场次`
- 时间 + 内外景：`日外` / `日内` / `夜外` / `夜内`
- 地点：具体场景名
- 出场人物：首次出现时补角色信息

推荐写法：

```markdown
## 1-1
- 时间+内外景：日内
- 地点：主任医师办公室
- 出场人物：裴准章（男主，心外科主任医师）、林晚（女主，住院总医师）
```

### 2. 正文标记

- `△`：人物动作、神态、表演调度（只写"人怎么动/什么表情"，不写机位和镜头运动）
- `OS`：人物心理活动
- `VO`：画外音
- `【闪回】` / `【闪回结束】`：回忆片段边界

### 3. 表演重点标记（可选）

必要时可以用**粗体括号**标记本段落要强化的**表演重点**，但这**不是景别/镜头指示**，而是留给导演和分镜师的强调符号。例如：

- `**【情绪爆点】**` — 本段要在此处完成一次情绪翻面
- `**【信息落点】**` — 本段要让观众看清某个关键信息
- `**【沉默压迫】**` — 本段需要留呼吸

**不要写** `**【特写】**` `**【全景】**` `**【中景】**` 这类景别符号。景别是镜头语言，属于分镜阶段。

### 4. 对白与旁白

- 语气描写统一用括号：`（轻声）`、`（愤怒）`、`（迟疑）`
- 旁白、`OS`、`VO` 中若出现人物名，人物名需加粗

推荐写法：

```markdown
△ 裴准章盯着病历，指节发白。

裴准章（压低声音）："Get out."
中译：滚出去。

OS：**裴准章** He can't know the truth. Not yet.
中译：他还不能知道真相。还不到时候。

VO：**林晚** I thought saving him would save us.
中译：我以为救了他，就能救回我们。
```

### 5. 格式控制原则

- 场次头信息是硬字段，不可省略
- 动作、对白、心理活动必须在场景正文内明确区分
- 首次登场人物必须补角色身份说明
- 旁白、心理活动、画外音不要和动作行混写
- `【闪回】` 必须成对出现
- 表演重点标记仅用于真正需要强化的戏点，不要逐段装饰

## 单集硬约束

- 冷开场约束：每集开场前 3-5 秒内必须有人物动作、情绪反应或上一集尾钩的直接叙事落地，禁止先写无人物的环境铺陈
- 开场前 10%-20% 篇幅内必须接住上一集尾钩
- 单集开头 15-30 秒内必须出现新动作、新信息或新压迫，不要先写铺垫闲聊
- 单场对白连续超过 6 句时，默认视为节奏风险；应补动作、打断、反应或信息变化
- 单场如果没有情绪变化或状态变化，优先考虑删场或并场
- 每场至少要承担以下之一：新动作、新信息、新压迫、新代价、关系变化；若都没有，这场默认无效
- 每集必须完成三密度校准：情绪密度、信息密度、情节密度各有一个具体抓手；任一抓手说不清，先回修节拍
- 关键集、付费卡点集、重大揭露集必须标出开头 3 秒情绪冲击、15 秒局势变化、45 秒强期待和结尾钩子；普通集至少要有开头钩子、中段变化和尾钩
- 禁止为了占篇幅增加空转对话、重复解释、无新信息的反应段或无功能过场
- 每场都要标明本场事实锚点，确保剧本先服从故事事实与全局节奏
- 一集默认控制在 2-5 场（60-90 秒项目）；若超出 5 场，必须有明确叙事理由
- 结尾钩子不能只是"有人推门而入"；必须带结果、关系、代价或信息中的至少一种明确翻转
- 结尾钩子不能只是编号、代号或阶段名；若使用 `Exhibit N`、`Phase N`、`Plan B` 等符号，必须同时写出该符号对应的具体丑闻、关系危险、可见代价或下一集会打脸的人
- 每集必须有“观众能看见”的冲突或反应方：打脸场要有被打脸者、见证人或受益人；只写“文件被采纳 / 申请通过 / 证据归档”不算有效爽点
- 主角冷静时必须外化情绪：用动作迟疑、身体代价、选择代价、他人指出异常、短暂失控边缘来承载；不能连续多集只写“低声、冷静、没有表情”
- 现实题材禁止无支撑的夸张生理/程序表达；如“12 天没睡”需改为“连续 12 天没睡整觉 / 每晚只睡两小时 / 12 天没在床上睡过”等可信表述

> **关于时长（E7-E9 沉淀 · 2026-04-12 硬规则升级）**：
>
> - **默认不写场次时长**：分集剧本是纯叙事初稿。默认在 `## 集信息` 写 `本集不给场次级时长预算，由分镜阶段全权分配`，不要为了看起来完整强行估秒数。
> - **可选但给了就必须可复算**：只有用户明确要求剧本阶段做时长预算时，才允许在场次头写**场次级规划预算**。一旦写了，就必须是具体数值或具体区间（如 `22-26 秒`），全集所有场次预算必须能外置算术求出下限、上限和中位值；若声明了 `episodeDurationSec` 规划目标，该目标必须落在 `[下限总计, 上限总计]` 内。这里校验的是剧本阶段的内部规划自洽，仍然禁止镜头级时长、镜头求和和分镜级闭合。
> - **禁止含糊话术**（违反 = P0）：严禁写 `约 75 秒` / `不做秒级闭合` / `总计约 N 秒` / `分镜阶段再算` 等任何带"约"字或把已声明数字的闭合责任甩给分镜阶段的话术。要么不写时长数字，要么给具体区间并自己验算。
> - **集信息头必须外置验算**：若选择写场次时长，则集信息头（见模板 `## 集信息` 字段）必须同时给出一行**外置算术验证**，格式示例：`规划目标 75 秒（三场次规划预算：1-1 23-24s + 1-2 25-26s + 1-3 25-27s = 73-77s · 75s ∈ [73,77]）`。没有这一行 = 质检 P0。
> - **根因说明**：E7/E8 两集 agent 曾写"约 75s · 不做秒级闭合"但外置加法实际 56-69s / 58-72s 缺口 6-19s，口头 PASS 但算术穿底。根本解法是**要么不写数字，要么写了就必须闭合**——不给含糊中间态。
> - **分镜阶段的关系**：剧本阶段的场次预算只是规划参考，不锁死最终成片时长。进入 `/分镜脚本` 后，时长唯一权威源是 `references/storyboard-conversion.md` §五：按镜头功能、对白量、动作复杂度和情绪停留自下而上推导，场次/全集只做算术闭合（±0.5s），并按 60-90s 目标区间与 90-108s 弹性区间判定。

## 单集专业设计规则

`/分集剧本` 阶段要站在专业编剧视角做叙事设计，不做镜头设计。

### 1. 先做单集叙事预算

每集先确定：

1. 本集承接点（上一集尾钩的落点是什么）
2. 本集要推进的核心目标
3. 本集场次数量（60-90 秒推荐 2-4 场）
4. 每场承担的叙事功能
5. 尾钩要把哪一个结果切给下一集

### 2. 每场都要有"存在理由"

每场开写前，至少明确：

- **承接点**：接住上一场或上一集什么
- **场景目标**：这场推进什么
- **事实锚点**：这一场有哪些不能改的故事事实（人物已知/未知、伤势、道具、站位、关系温度）
- **情绪位置**：这场在本集情绪曲线中的位置（开钩 / 升压 / 翻转 / 尾钩）
- **遗留**：这场写完后留下了什么待承接

若回答不清其中任何一项，这场视为无效，不要写。

### 3. 在剧本阶段消灭无效场景

以下情况在 `/分集剧本` 阶段就应删改，不要留给 `/分镜脚本` 挽救：

- 场景只重复上一集信息
- 场景只有解释没有动作
- 场景只有情绪没有推进
- 场景只是为了占篇幅存在
- 场景写完后，人物状态与关系温度没有任何变化

### 4. 分集剧本不是分镜稿

**绝对禁止**在分集剧本里：

- 写镜头号（`镜1` `镜2`）
- 写镜头求和或镜头级时长闭合
- 写 `AI画面提示：` `AI动作提示：` 这类字段
- 写机位/角度/运镜关键词（特写/近景/中景/全景/航拍/摇/推/拉/甩 等）
- 写连续性锁定词或负面约束

如果写了以上任何一项，就是**把分镜工作偷偷挪到了分集剧本阶段**，会让 `/剧本质检` 和 `/分镜质检` 边界混乱。发现后按最小修改集移除，相关内容在 `/分镜脚本 {起止集}` 阶段重新生产。

## 模板检查

在写 `第NNN集.md` 前先确认：
