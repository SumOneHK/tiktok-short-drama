## 使用规则

本 skill 的最终交付一律是 60-90 秒竖屏 AI 视频，由 AI 视频模型（可灵 / Sora / Veo / Runway / 海螺 等）按**分镜脚本**生成成片。**分集剧本只是叙事中间稿，不是制作稿**——以下四件事全部交给 `/分镜脚本` 阶段处理，**不得**在分集剧本里出现：

- ❌ 镜头号、镜头求和、时长闭合、场景时长预算
- ❌ `AI画面提示`、`AI动作提示`、镜头级 prompt 关键词
- ❌ 机位/角度/运镜等镜头语言字段
- ❌ 连续性锁定词、负面约束等 AI 生成约束

分集剧本只回答一件事：**"这一集作为一段可读的戏，人怎么走、戏怎么推、情绪怎么翻、尾钩怎么切。"** 其他一切都推到分镜阶段。

单集配置规则：

- 艺术风格（`artStyle`）由 `AI资产圣经.md` 全局生成规则统一定义，剧本模板里**不出现**任何风格关键词或 prompt 片段
- 海外模式下，模板内只有对白、旁白、`OS`、`VO`、字卡允许使用英文，其余说明仍用中文
- 海外模式下，凡英文对白、旁白、`OS`、`VO`、字卡，必须紧跟中文翻译；推荐下一行使用 `中译：...`

## 单集格式总规范

### 1. 场次头信息

每场都必须先写清：

- 场景序号：`集数-场次`
- 时间 + 内外景：`日外` / `日内` / `夜外` / `夜内`
- 地点：具体场景名
- 出场人物：首次出现时补角色信息

推荐写法：

```markdown
## 1-1
- 时间+内外景：日内
- 地点：主任医师办公室
- 出场人物：裴准章（男主，心外科主任医师）、林晚（女主，住院总医师）
```

### 2. 正文标记

- `△`：人物动作、神态、表演调度（只写"人怎么动/什么表情"，不写机位和镜头运动）
- `OS`：人物心理活动
- `VO`：画外音
- `【闪回】` / `【闪回结束】`：回忆片段边界

### 3. 表演重点标记（可选）

必要时可以用**粗体括号**标记本段落要强化的**表演重点**，但这**不是景别/镜头指示**，而是留给导演和分镜师的强调符号。例如：

- `**【情绪爆点】**` — 本段要在此处完成一次情绪翻面
- `**【信息落点】**` — 本段要让观众看清某个关键信息
- `**【沉默压迫】**` — 本段需要留呼吸

**不要写** `**【特写】**` `**【全景】**` `**【中景】**` 这类景别符号。景别是镜头语言，属于分镜阶段。

### 4. 对白与旁白

- 语气描写统一用括号：`（轻声）`、`（愤怒）`、`（迟疑）`
- 旁白、`OS`、`VO` 中若出现人物名，人物名需加粗

推荐写法：

```markdown
△ 裴准章盯着病历，指节发白。

裴准章（压低声音）："Get out."
中译：滚出去。

OS：**裴准章** He can't know the truth. Not yet.
中译：他还不能知道真相。还不到时候。

VO：**林晚** I thought saving him would save us.
中译：我以为救了他，就能救回我们。
```

### 5. 格式控制原则

- 场次头信息是硬字段，不可省略
- 动作、对白、心理活动必须在场景正文内明确区分
- 首次登场人物必须补角色身份说明
- 旁白、心理活动、画外音不要和动作行混写
- `【闪回】` 必须成对出现
- 表演重点标记仅用于真正需要强化的戏点，不要逐段装饰

## 单集硬约束

- 冷开场约束：每集开场前 3-5 秒内必须有人物动作、情绪反应或上一集尾钩的直接叙事落地，禁止先写无人物的环境铺陈
- 开场前 10%-20% 篇幅内必须接住上一集尾钩
- 单集开头 15-30 秒内必须出现新动作、新信息或新压迫，不要先写铺垫闲聊
- 单场对白连续超过 6 句时，默认视为节奏风险；应补动作、打断、反应或信息变化
- 单场如果没有情绪变化或状态变化，优先考虑删场或并场
- 每场至少要承担以下之一：新动作、新信息、新压迫、新代价、关系变化；若都没有，这场默认无效
- 禁止为了占篇幅增加空转对话、重复解释、无新信息的反应段或无功能过场
- 每场都要标明本场事实锚点，确保剧本先服从故事事实与全局节奏
- 一集默认控制在 2-5 场（60-90 秒项目）；若超出 5 场，必须有明确叙事理由
- 结尾钩子不能只是"有人推门而入"；必须带结果、关系、代价或信息中的至少一种明确翻转

> **关于时长（E7-E9 沉淀 · 2026-04-12 硬规则升级）**：
>
> - **可选但给了就必须闭合**：分集剧本场次头的"时长"字段是**可选**的——如果不写，由分镜阶段自由分配；**但一旦写了**就必须是具体数值或具体区间（如 `22-26 秒`），且全集所有场次时长累加必须外置算术闭合到 `target_duration_sec ± 2s`（75 秒项目即 73-77s）。
> - **禁止含糊话术**（违反 = P0）：严禁写 `约 75 秒` / `不做秒级闭合` / `分镜阶段再算` / `总计约 N 秒` 等任何带"约"字或把闭合责任甩给分镜阶段的话术。要么不写，要么给具体区间并自己闭合。
> - **集信息头必须外置验算**：若选择写场次时长，则集信息头（见模板 `## 集信息` 字段）必须同时给出一行**外置算术验证**，格式示例：`75 秒（三场次外置算术：1-1 22-26s + 1-2 26-30s + 1-3 22-26s = 70-82s · 中位 76s 覆盖 75s 目标 · 分镜阶段闭合到 75±2s）`。没有这一行 = 质检 P0。
> - **根因说明**：E7/E8 两集 agent 曾写"约 75s · 不做秒级闭合"但外置加法实际 56-69s / 58-72s 缺口 6-19s，口头 PASS 但算术穿底。根本解法是**要么不写数字，要么写了就必须闭合**——不给含糊中间态。
> - **分镜阶段的关系**：剧本阶段给的是场次区间预算，分镜阶段做的是镜头级收敛到单点值——两者都必须独立闭合到 75±2s，上下阶段分别负责各自层级的算术。

## 单集专业设计规则

`/分集剧本` 阶段要站在专业编剧视角做叙事设计，不做镜头设计。

### 1. 先做单集叙事预算

每集先确定：

1. 本集承接点（上一集尾钩的落点是什么）
2. 本集要推进的核心目标
3. 本集场次数量（60-90 秒推荐 2-4 场）
4. 每场承担的叙事功能
5. 尾钩要把哪一个结果切给下一集

### 2. 每场都要有"存在理由"

每场开写前，至少明确：

- **承接点**：接住上一场或上一集什么
- **场景目标**：这场推进什么
- **事实锚点**：这一场有哪些不能改的故事事实（人物已知/未知、伤势、道具、站位、关系温度）
- **情绪位置**：这场在本集情绪曲线中的位置（开钩 / 升压 / 翻转 / 尾钩）
- **遗留**：这场写完后留下了什么待承接

若回答不清其中任何一项，这场视为无效，不要写。

### 3. 在剧本阶段消灭无效场景

以下情况在 `/分集剧本` 阶段就应删改，不要留给 `/分镜脚本` 挽救：

- 场景只重复上一集信息
- 场景只有解释没有动作
- 场景只有情绪没有推进
- 场景只是为了占篇幅存在
- 场景写完后，人物状态与关系温度没有任何变化

### 4. 分集剧本不是分镜稿

**绝对禁止**在分集剧本里：

- 写镜头号（`镜1` `镜2`）
- 写镜头求和或时长闭合
- 写 `AI画面提示：` `AI动作提示：` 这类字段
- 写机位/角度/运镜关键词（特写/近景/中景/全景/航拍/摇/推/拉/甩 等）
- 写连续性锁定词或负面约束

如果写了以上任何一项，就是**把分镜工作偷偷挪到了分集剧本阶段**，会让 `/剧本质检` 和 `/分镜质检` 边界混乱。发现后按最小修改集移除，相关内容在 `/分镜脚本 {起止集}` 阶段重新生产。

## 模板字段说明

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 集信息 | 必填 | 本集篇幅目标、情绪曲线、爽点、尾钩类型、承接上一集尾钩的落点 |
| 场次头 | 必填 | 时间+内外景、地点、出场人物、承接点、场景目标、情绪目标、事实锚点、场景状态差量、情绪位置、遗留 |
| 场景状态差量 | 必填 | "开场 → 收场"一句话；无差量 = 灌水场，必须删或并 |
| 情绪位置 | 必填 | 标明该场处于开钩 / 升压 / 翻转 / 尾钩中的哪一段（情绪位置不是镜头位置） |
| 场景正文 | 必填 | `△` 写人物动作/表演/神态；对白配语气括号；OS/VO 单独成行；不写镜头语言 |
| 状态变化 | 必填 | 单集结尾的人物 / 关系 / 信息 / 资产 / 风险变化 |
| 结尾钩子 | 必填 | 单句即可，必须带结果 / 关系 / 代价 / 信息中至少一种翻转 |
| 本集调用资产清单 | 必填 | 只写叙事事实级登记，不写 prompt 增量 |

## 模板检查

在写 `第NNN集.md` 前先确认：

1. 集信息和每场场次头是否填齐
2. 每场是否能回答"承担了哪个新动作 / 新信息 / 新压迫 / 新代价 / 关系变化"
3. 每场是否已锁定事实锚点和情绪位置
4. 是否在本集调用资产清单里登记了本集出场的全局角色/场景/道具
5. 海外模式下，所有英文对白 / 旁白 / OS / VO / 字卡是否都补了 `中译：...`
6. 结尾钩子是否带明确翻转，而不是模糊的"门被推开"
7. **反向检查**：本文件是否误写了镜头号、时长求和、AI 画面提示、机位/运镜等分镜级字段；若有，全部移除并推迟到 `/分镜脚本` 阶段
