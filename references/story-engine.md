## 目标

这一阶段要把项目做成“可拆集、可升级、可卡点”的引擎，而不是写一篇长篇故事梗概。

## 结构原则

1. 前 3 集完成承诺，不要把真正的戏留到后面。
2. 每一大段都要有新的压迫类型，不要重复同一种羞辱或误会。
3. 中段必须升级筹码，不只是放大情绪音量。
4. 爽点要有预埋和成本，不靠凭空开挂。
5. 情绪线和剧情线要互相推动。

## 常用宏观结构

### 五段波形

适合大多数 TikTok 情绪型和逆袭型短剧：

1. 承诺建立
2. 首轮压迫与小反击
3. 身份/关系/力量翻页
4. 失控与代价
5. 总清算与回收

### 双线拉扯

适合 romance、forbidden love、secret baby：

- 外在线：身份、婚约、家族、利益
- 内在线：信任、欲望、羞耻、误解

### 升级链

适合 revenge、billionaire、mafia、supernatural：

- 小敌人
- 近身阻力
- 中层权力者
- 核心对手

## 题材与结构类型对照

不同题材天然适配不同的结构类型，选结构骨架前先查此表：

| 题材类型 | 推荐主结构 | 辅助结构 | 理由 |
|---------|---------|---------|------|
| 霸总/逆袭/权力升级 | 升级链 | 双线拉扯 | 核心爽点是权力层级逐步突破，需要清晰的对手升级路径 |
| Romance/Fake Marriage/Contract Love | 双线拉扯 | 五段波形 | 外在障碍+内在情感同步推进，两条线互补 |
| 复仇/背叛/Wedding Crash | 五段波形 | 升级链 | 情绪弧完整，清算段是最高潮，需要完整情绪路径 |
| Werewolf/Vampire/Supernatural | 双线拉扯+升级链嵌套 | — | 超自然冲突（升级链）+人性冲突（双线）必须同时维护 |
| Mafia/Forbidden Love | 双线拉扯 | 五段波形 | 身份/立场外在线和感情内在线形成持续张力 |
| Secret Baby/Hidden Identity | 五段波形 | 双线拉扯 | 信息差驱动节奏，揭露时机规划是核心 |
| Chosen One/Magic School/Prophecy | 升级链 | 双线拉扯 | 能力和身份同步升级，需要明确的层级对手体系 |
| 悬疑/追查/双线调查 | 信息释放型（自定义） | 五段波形 | 信息差是核心驱动，悬念释放节奏需专项规划 |

使用规则：
- 主结构决定全剧的主要推进逻辑
- 辅助结构补充主结构缺少的维度
- 题材融合时，以情绪承诺更强的题材选主结构

## 题材融合设计原则

TikTok 高转化剧常见双题材融合（如 mafia + romance，billionaire + supernatural），融合时遵守以下规则：

### 主题材与副题材分工

| 分工维度 | 主题材承担 | 副题材承担 |
|---------|---------|---------|
| 情绪高潮 | 最大爽点和最强情绪峰值 | 调剂和次级爽点 |
| 钩子类型 | 主干钩子（如 romance 的关系钩子） | 差异化钩子（如 supernatural 的代价钩子） |
| 卡点类型 | 选择主题材最适配的卡点类型 | 激化主卡点的压迫感，不做主卡点 |
| 前3集承诺 | 核心情绪必须在前3集交付 | 可在第5-8集后才充分展开 |

### 融合常见失误
- 两条题材线平均用力，观众不知道"这部剧主要卖什么"
- 副题材钩子比主题材更吸引人，导致主线被架空
- 题材融合后主情绪模糊（找不到统一的情绪承诺）
- 两条线的高潮撞车，互相稀释而不是叠加

## 按集数映射的结构骨架

以下不是唯一方案，但默认先按这个骨架检查是否具备升级路径：

### 30-40 集项目

- 1-3 集：承诺建立，主冲突快速落地
- 4-8 集：首轮压迫与第一次明确回报
- 9-14 集：关系或身份翻页，首个强卡点
- 15-24 集：中段升级，旧悬念部分兑现，新对手抬高代价
- 25-32 集：失控与代价，主角暂时失利
- 33-40 集：总清算与核心情绪回收

### 45-60 集项目

- 1-3 集：承诺建立
- 4-10 集：压迫升级与连续小兑现
- 11-16 集：首个强卡点与身份/关系翻页
- 17-30 集：中段升级链，避免同类桥段重复
- 31-42 集：失控、代价与大反噬
- 43-52 集：终局前的二次翻页
- 53-60 集：总清算与回收

### 60 集以上项目

- 必须额外写清中段疲劳补偿机制
- 每 8-12 集至少出现一次高价值兑现
- 每 12-18 集至少发生一次权力、身份、关系三者之一的翻页
- 若只能继续重复羞辱或误会，应主动缩短总集数

## 钩子设计

钩子至少分为四类：

- 信息钩子：观众知道或怀疑了某件事
- 关系钩子：两人关系即将翻转
- 结果钩子：某行动的结果即将揭晓
- 代价钩子：主角将为某个决定付出代价

不要只靠“有人推门而入”式空悬念。

### 尾钩具象化要求

尾钩不能只交代一个抽象编号、代号或阶段名。凡使用 `Exhibit 7`、`Phase 3`、`Plan B`、`Case 14`、`档案编号` 等符号做结尾，必须同时给观众一个可理解的具体兴趣点：

- 这份东西会打谁的脸：例如“Exhibit 9 = Alex 家族离岸信托真实受益人清单”
- 它对应哪段过去伤害：例如“这张照片证明 Vivian 认识 Erin 父亲”
- 它会造成什么可见后果：例如“附件被搁置，反派终于赢下一步”
- 它揭开什么禁忌关系或危险：例如“USB 里有 Vivian 和父亲的旧文件”

只写 `EXHIBIT N ->`、`PHASE N ->`、`NEXT: FILE N`，但不告诉观众“里面是什么 / 谁会倒霉 / 为什么危险”，视为弱钩子。连续 2 次弱钩子 = P1；付费卡点使用弱钩子 = P0。

## 关键集节拍设计（Key Episode Beat Sheet）

关键集（强卡点集、身份揭露集、大爽点集、关系翻页集）不能只靠场景分组，必须在写剧本前完成节拍级规划。

### 必须完成节拍设计的集类型
- 首个强卡点集
- 身份/秘密重大揭露集
- 终极反杀/公开清算集
- 感情线确认或决裂集
- 结局集

### 6节拍模型

| 节拍 | 功能 | 占全集比例 | 写什么 |
|------|------|---------|------|
| 节拍1：承接 | 接住上一集尾钩，立即推进而非复述 | 10-15% | 直接落地上一集悬念的第一个动作 |
| 节拍2：升压 | 形势变得更糟或更危险 | 20-25% | 新障碍、新信息或新威胁出现 |
| 节拍3：临界点 | 最高压迫，主角/关系到达极限 | 15-20% | 主角必须做选择或被迫暴露 |
| 节拍4：转折动作 | 爽点的核心执行动作 | 20-25% | 具体的反杀、揭露、告白或代价 |
| 节拍5：余震 | 爽点落地后的反应和后果 | 15-20% | 见证人反应、关系变化、情绪余味 |
| 节拍6：新债务 | 建立下一集的新压迫 | 5-10% | 新悬念、新威胁或新问题出现 |

### 使用规则
- 普通推进集不需要完整6节拍，确保状态变化即可
- 关键集必须先完成6节拍规划，再开始写场景正文
- 节拍4（转折动作）占比不能过短——爽点需要足够的执行时间才能被观众感受到

## 情绪设计

> **权责说明**：情绪设计的完整规范（全剧情绪承诺、禁区情绪阈值、情绪峰值排列、单集情绪弧、调剂集设计）以 `references/emotion-design.md` 为唯一权威源。本节仅提供结构层面的对应关系，实际执行以 `emotion-design.md` 为准，不以本节为准。

常见高转化情绪链（结构参考）：

- 羞辱 -> 忍耐 -> 反击
- 心动 -> 误解 -> 逼近 -> 爆发
- 诱惑 -> 失控 -> 后悔 -> 更深绑定
- 希望 -> 夺走 -> 再争夺 -> 双倍回报

同一类情绪最多连续高强度使用 2-3 次，之后要换形态。具体禁区情绪阈值见 `emotion-design.md`。

## 节奏设计

每集建议具备：

1. 立即进入冲突
2. 30-50 秒内出现新信息或新动作
3. 结尾留下明确下一步压力

全剧节奏上：

- 前段快建承诺
- 中段快升筹码
- 后段快回收问题

## 爽点设计

爽点必须写清四件事：

1. 观众期待什么
2. 主角凭什么能做到
3. 兑现时谁被打脸或谁被治愈
4. 下一轮如何升级

可复用爽点类型：

- 身份曝光
- 公开反杀
- 地位逆转
- 关键选择偏向主角
- 被抛弃者成为唯一解
- 隐藏能力第一次被看见

### 爽点密度规范

全剧范围内，爽点密度必须满足以下基线：

| 爽点级别 | 定义 | 最低密度要求 | 连续间隔上限 |
|---------|------|-----------|-----------|
| 小爽点 | 让观众觉得"这集没白看"的即时回报 | 每集至少1次 | 不允许连续2集无小爽点 |
| 中爽点 | 阶段性满足或明显的关系推进 | 每5-8集至少1次 | 连续8集无中爽点视为中段疲劳 |
| 大爽点 | 身份翻转、终极反杀、重大情感兑现 | 每12-18集至少1次 | 连续18集无大爽点必须回退重做结构 |

密度过高同样是问题：
- 连续3集都有大爽点：观众不再信任接下来的压迫，爽感贬值
- 前段消耗大爽点过多：中后段无法升级，中段疲劳提前到来

### 爽点见证人设计

"谁来见证和放大爽感"必须提前设计，不能随机——见证人是爽点的放大器。

**见证人的情绪投入（优先级排序）：**
1. 之前参与过压迫的人（帮凶）——见证主角反杀时自带内疚和震惊
2. 旁观过羞辱但没有阻止的人——见证时有悔意和惊愕
3. 对主角有情绪判断的人（家人/同事/朋友）——见证时放大社会认可感
4. 反派本人——最强配置，反派当场被打脸时爽感倍率最高
5. 纯中立路人——最弱，无法放大爽感

**见证人的权力位置：**
- 见证人的社会地位越高，爽感倍率越高
- 大场合见证（多人在场）比私下见证爽感更强

**见证人数量与规模应逐级升级：**
- 小爽点：1-3人见证，私下化
- 中爽点：5-15人见证，半公开
- 大爽点：大场面，所有相关人见证，公开清算

**规则：** 见证人必须在大爽点前已出现在场景中，不能为放大爽感临时引入新见证人。

### 证据型/法庭型复仇的特殊规则

法律、商战、调查、证据链复仇题材天然容易滑成“一个编号接一个编号”的闯关结构。写这类项目时，证据不是爽点本身，**证据造成的公开羞辱、权力反转、关系破裂或现实代价**才是爽点。

硬规则：

- 每份核心证据必须绑定一个“受害旧伤 + 当前打脸对象 + 可见后果”，不能只绑定编号和专业名称。
- 法庭不能像随时可开启的自家会议室。除非是同一庭审内连续呈堂，否则不得写成“一份证据开一次庭”。需要分散推进时，优先使用董事会、律所、媒体直播、和解会议、预审动议、SEC 问询、内部听证等不同公共场合。
- 连续 3 集不得都以“提交/采纳/归档某 Exhibit”作为主要爽点；必须至少插入一次非程序型爽点，如反派当场失态、公开道歉/撤职、资产冻结、盟友倒戈、关系硬翻页、主角主动让对方付出代价。
- 专业术语必须被画面或一句人话翻译。可以保留 `Exhibit`、`chain of custody` 等风格词，但每个关键法律动作都要同时让观众知道“这意味着谁完了 / 谁赢了什么 / 谁失去了什么”。
- “法官一句 Admitted”只能作为程序结果，不能单独当高潮。高潮必须落在人身上：反派手抖、律师失语、董事会沉默、记者直播、家人站错队、主角一句短句反杀。

若一集的爽点描述只有“专业碾压 / 智性对抗 / 证据公开”，但没有明确被打脸者和可见代价，按弱爽点处理。

### 双时间线反杀要求

双时间线不是装饰性的闪回。每次过去线出现，都必须与现在线形成“伤害 -> 回击”的碰撞：

- 过去线给出一段具体迫害、羞辱、误导、背叛或失去。
- 现在线必须在同集或下一集让同一施害者、帮凶或见证人付出可见代价。
- 转场要有动作或道具铰链，例如“过去签字的笔 -> 现在同一支笔签撤职令”，“过去被泼酒的婚纱 -> 现在酒渍信纸成为 Exhibit 1”。
- 不能连续多次只用过去线做背景解释。连续 2 次“过去线只铺垫、现在线无即时回击” = P1；强卡点集这么写 = P0。

## 结构红旗

出现以下任一情况，结构阶段应直接回退重做：

- 前 3 集还在解释世界观，题材承诺没有落地
- 首个强卡点前没有明确兑现
- 中段只能换皮重复同类桥段
- 结尾大回收依赖临时新设定或新角色
- 爽点没有成本，纯靠主角突然开挂

## 节奏修复工具

质检发现中段节奏疲劳（连续几集状态不变、重复同类冲突）时，以下工具可在不推翻已写内容的前提下注入新节奏：

### 工具1：信息炸弹植入
让某个角色提前知道一件本来晚些才揭露的信息，立即改变该角色的行为动机。
- 适用：主角或反派知道了"不该现在知道"的事
- 效果：立即产生新信息差和行动压力
- 注意：必须同步更新悬念债务账本

### 工具2：短期功能性对手
引入只存在 3-5 集的新对立角色，制造阶段性新压迫，然后有理由退场。
- 适用：主要反派短暂失效或主冲突线进入真空期
- 效果：注入新类型冲突，打破重复感
- 规则：退场方式必须服务主线，不能随意消失

### 工具3：关系状态硬翻页
让主要关系发生超出预期的快速变化（信任崩塌、意外靠近、公开决裂）。
- 适用：感情线或对立线进入重复模式
- 效果：观众无法预测走向，重新投入注意力
- 规则：翻页必须有前因，不能是作者强推

### 工具4：反派视角集
用1集跟随反派视角，让观众看到反派计划，制造"观众知道但主角不知道"的新悬念。
- 适用：主角视角让观众产生压迫疲劳
- 效果：观众获得信息优势，对即将到来的危险产生预期恐惧（比重复羞辱更有效）
- 规则：必须揭示一个针对主角的新威胁

### 工具5：强制代价落地
让一个之前"还没付代价"的决定突然产生后果，打破安全感。
- 适用：有悬而未决的代价债务
- 效果：重建剧情因果可信度，观众重新相信"行动有后果"
