# `质检检查点.md` 固定模板

> 调用场景：任一 `/*质检` 命令或 `/整剧通读` 首次发现问题时创建；后续所有质检命令写入同一文件。

```markdown
# {剧名} 质检检查点

## 全局问题追踪表
| ID | 日期 | 质检命令 | 检查对象 | 等级 | 问题摘要 | 状态 | 必改时点 |
|----|------|---------|---------|------|---------|------|---------|
| QC-001 | 2026-03-12 | /结构质检 | 故事结构.md | P1 | 中段升级不足 | 新发现 | 进入分集大纲前 |

状态说明：
- `新发现`：本轮刚发现，未处理
- `处理中`：已开始修
- `已修复`：已修并复检通过
- `接受风险`：明确不修，但保留风险说明
- `延后处理`：不阻塞当前阶段，但需在指定时点回看

## 2026-03-12 /{质检命令}
- 检查对象：
- 结论：通过 / 需修改 / 阻塞
- 当前阶段：
- 检查区间：
- P0：
- P1：
- P2：
- 必改项：
- 可延后项：
- 下一步：
- 问题状态更新：
```

使用规则：

- 新问题必须登记到"全局问题追踪表"
- 同一个问题修复后，必须更新状态，不要重复新建
- 若选择 `接受风险`，必须写明原因和影响范围
- `/开始` 与 `/立项` 的内建自检不写入 `质检检查点.md`，只在 `选题分析.md` 推荐版/正式版末尾自述；如确需在后续阶段回看，可在 `/设定质检` 首次创建 `质检检查点.md` 时手工补记
- `/大纲质检` 额外补：`高风险集数`
- `/策划通读` 额外补：`六视角结论`、`综合吸引力评分`、`最大卖点`、`最大风险`、`建议试点批次`
- `/剧本质检` 和 `/衔接质检` 额外补：`检查区间`
- `/剧本质检` 额外补：`批次剧本通读结论`（编剧/导演/观众/平台/本地化五视角）与逐集表；若发现动作桥、重复情绪手段或批次承接问题，必须登记 QC ID
- `/衔接质检` 额外补：`断裂点`、`重复点`、`连续性问题`
- `/整剧通读` 额外补：`五视角通读结论`、`分段通读记录`、`可分镜阶段放大项`、`接受风险项`
- `/分镜质检` 额外补：`检查区间`、`分镜覆盖率`、`高风险镜头`、`prompt 完整度`
- `/总检` 额外补：`全局P0`、`全局P1`、`全局P2`、`导出前必改项`、`接受风险项`
