# `分集追踪.md` 固定模板

> 调用场景：`/分集大纲` 首次创建（预建全局追踪表）；`/分集剧本` 每集写完后回填 EP 块。

```markdown
# {剧名} 分集追踪

## 全局追踪表
| 集数 | 主要推进 | 本集爽点 | 本集兑现 | 新增悬念 | 下一集必须承接 |
|------|---------|---------|---------|---------|---------------|
| EP001 |         |         |         |         |               |

## EP{NNN}
- 登场角色与标准称谓：
- 本集推进：
- 开场钩子承接：
- 本集情绪曲线：
- 本集爽点：
- 本集爽点类型：（身份曝光/公开反杀/地位逆转/关键选择偏向/隐藏能力展示/关系确认/其他）
- 本集兑现：
- 尾钩：
- 尾钩类型：（信息钩/关系钩/结果钩/代价钩/身份钩/误会钩/威胁钩/选择钩/揭露钩/反杀钩；与 `分集大纲.md` 10 类枚举保持一致，禁止自造）
- 新增悬念：
- 已兑现悬念：
- 藏线进度：
  - 触及的藏线ID：
  - 本集新增发现证据：
  - 本集误读方向：
  - 最迟小回报/兑现责任：
- 角色状态变化：
- 关系温度变化：
- 新增功能性角色：
  - 是否必要：
  - 若必要，承担什么功能：
- 场景/道具新增或损坏：
- 伤势/造型连续性：
- 时间标记：（故事内时间，格式：故事第N天/周X/具体时段，需与分集大纲一致）
- 与大纲偏差：
- 下一集必须承接：
```

说明：

- `EP{NNN}` 为逐集复用块，每集复制一份继续往下追加
- 若某集无新增功能性角色，`新增功能性角色` 字段也保留，直接写"无"
