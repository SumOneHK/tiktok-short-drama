# 文档联动修订规则

## 目录

- 目标
- 一、标准修订顺序
- 二、问题类型与联动修订表
- 三、修订判定规则
- 四、修复闭环

## 目标

当问题被发现时，不能只改一处。三份核心文档分工不同：

- `故事设定.md`：稳定事实和全剧标准
- `分集追踪.md`：逐集变化和实际落地
- `质检检查点.md`：问题发现、修复状态和风险追踪

修订原则：

1. 先登记问题，再修事实，再修逐集记录，最后回填质检状态
2. 若问题只改剧本正文，不改三份文档，视为未修复
3. 同一个问题必须在 `质检检查点.md` 保留同一 ID 追踪到底

## 一、标准修订顺序

### 修复前：下游影响评估（不可跳过）

在执行任何修复操作前，先回答以下 4 个问题，并将评估结论写在 `质检检查点.md` 的问题记录中：

1. **影响范围**：这个问题最早从哪一集开始影响？最晚影响到哪一集？（写出集数区间）
2. **已写集数**：受影响区间内有多少集已经写完？（这些集都需要重检）
3. **级联风险**：修复这个问题后，是否可能引发新的连续性问题？（如：改了角色的某条信息，是否影响其他角色的反应）
4. **修复优先级**：是立即全部修复，还是先修当前集、后续集数延后处理？（延后处理需在 `质检检查点.md` 标记并设定必改时点）

只有完成上述评估，才能进入以下修复步骤。

发现问题后默认顺序：

1. 在 `质检检查点.md` 登记或更新问题
2. 判断该问题属于“稳定事实”还是“逐集落地”
3. 若影响稳定事实，先修 `故事设定.md`
4. 再修 `分集追踪.md` 的相关集记录
5. 再修 `分集剧本/*.md`、`分集大纲.md` 或 `故事结构.md`
6. 复检通过后，把 `质检检查点.md` 状态改为 `已修复`

## 二、问题类型与联动修订表

### 1. 命名 / 称谓问题

例子：

- 角色名字前后不一致
- 同一角色称谓混乱
- 英文展示名变动

必须同步修改：

- `故事设定.md`
  - `角色命名与称谓总表`
  - `角色档案`
- `分集追踪.md`
  - 涉及集数的 `登场角色与标准称谓`
- `质检检查点.md`
  - 更新问题状态和受影响集数

如已写剧本：

- 同步修 `分集剧本/*.md`

### 2. 角色功能问题

例子：

- 新增了冗余功能性角色
- 原本配角和新角色功能重叠

必须同步修改：

- `故事设定.md`
  - `角色档案`
  - `主要戏剧功能`
- `分集追踪.md`
  - `新增功能性角色`
  - 受影响集数的登场角色记录
- `质检检查点.md`
  - 记录是否删除、合并或降级该角色

### 3. 关系温度 / 人设问题

例子：

- 角色关系突然跳变
- 台词和人设冲突
- 角色知道了不该知道的信息

必须同步修改：

- `故事设定.md`
  - `关系网络`
  - `角色情绪引擎`
  - `角色台词声线卡`
- `分集追踪.md`
  - 相关集的 `关系温度变化`
  - `角色状态变化`
- `质检检查点.md`
  - 标记问题来源集和修复策略

### 4. 悬念 / 信息差问题

例子：

- 悬念埋了没收
- 提前泄露真相
- 兑现节点错位

必须同步修改：

- `故事设定.md`
  - `秘密与信息差`
- `分集追踪.md`
  - `新增悬念`
  - `已兑现悬念`
  - `下一集必须承接`
- `质检检查点.md`
  - 标记是“延后处理”还是“已修复”

如影响全局节奏：

- 同步修 `故事结构.md`
- 必要时修 `分集大纲.md`

### 5. 道具 / 伤势 / 资产连续性问题

例子：

- 道具已损坏却再次完好出现
- 角色伤势突然消失
- 造型不连续

必须同步修改：

- `故事设定.md`
  - `道具首登场清单`
  - `角色外观锚点`
- `分集追踪.md`
  - `场景/道具新增或损坏`
  - `伤势/造型连续性`
- `质检检查点.md`
  - 标记受影响集数和修复优先级

如涉及角色伤势或形象变化：
- `故事设定.md`
  - `角色外观变化记录`（更新受影响集数、预计持续集数、AI生成备注）
- `分集追踪.md`
  - 受影响集数的 `伤势/造型连续性` 字段

### 6. 结构 / 大纲问题

例子：

- 某集不该在这里揭露
- 大纲和剧本偏离
- 卡点前无回报

必须同步修改：

- `故事结构.md`
- `分集大纲.md`
- `分集追踪.md`
  - `与大纲偏差`
  - `下一集必须承接`
- `质检检查点.md`
  - 更新问题状态

如果结构被改动后影响既有事实：

- 回头同步修 `故事设定.md`

### 7. artStyle / 全局生成规则不一致问题

例子：

- `AI资产圣经.md` 写的是 `写实电影感`，但分镜稿 `AI画面提示` 注入的是 `国潮水墨` 关键词
- 剧本层面未确定 artStyle，分镜阶段各镜头风格关键词不一致
- 艺术风格切换后未同步回写 `AI资产圣经.md` 的 `artStyle` 字段和风格关键词模板
- `目标 AI 视频模型偏好` 与 `artStyle` 明显不匹配（例如强风格 artStyle 却选了风格兼容度差的模型）

必须同步修改：

- `剧本状态.json`
  - 确认 `artStyle`
- `故事设定.md`
  - `项目概览` 中的艺术风格
- `AI资产圣经.md`
  - 全局生成规则下的 `artStyle`、风格关键词模板、目标 AI 视频模型偏好
- `分镜脚本/*.md`
  - 受影响集数的 `AI画面提示` 字段和 `AI 视频生成附录` 中的 `当前 artStyle`
- `分集追踪.md`
  - 相关集的 `与大纲偏差`
- `质检检查点.md`
  - 记录 artStyle 不一致问题

### 8. 逻辑因果问题

例子：

- 事件发生缺乏前因
- 角色行动不合常理
- 代价消失

必须同步修改：

- 若是规则问题：先修 `故事设定.md`
- 若是结构问题：修 `故事结构.md` / `分集大纲.md`
- 若是单集落地问题：修 `分集追踪.md` 与相关剧本
- `质检检查点.md`
  - 记录问题层级和修复位置

## 三、修订判定规则

### 只改 `分集追踪.md` 的情况

- 只是某一集的实际写法微调
- 不影响全剧稳定事实
- 不影响角色命名、规则、长期关系

### 必须改 `故事设定.md` 的情况

- 影响稳定设定
- 影响角色标准信息
- 影响关系基线
- 影响道具或场景长期状态

### 必须改 `质检检查点.md` 的情况

- 任何被质检发现的问题
- 任何决定延后处理的问题
- 任何接受风险但不修的问题

## 四、修复闭环

一个问题只有同时满足以下条件，才算真正修完：

1. 相关源文档已同步更新
2. 相关剧本或大纲已同步更新
3. `质检检查点.md` 中原问题状态已改成 `已修复`
4. 最近一次对应质检命令已复检通过
5. 若项目已经跑过 `/总检` 或 `/合规`，且本轮修改影响剧情、角色、命名、关系、道具、台词或镜头表达，则 `总检报告.md` / `合规报告.md` 必须视为待重跑

否则一律视为“处理中”。
